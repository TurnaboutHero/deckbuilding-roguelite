# P8 전투 피드백·사운드·UI 레이어 안정화 설계

**상태:** 사용자 방향 승인 · 구현 전 정본  
**작성일:** 2026-07-13  
**대상:** `apps/ui` 전투 화면과 공용 오버레이, 합성 SFX  
**제품 단계:** engineering target / balance-neutral / experience-unverified

## 1. 결정 요약

P8은 신규 캐릭터·스킬·밸런스 확장이 아니라 **플레이어가 이미 구현된 규칙을 보고, 듣고, 즉시 이해하도록 만드는 제품 피드백 마감 단계**다.

구현 순서는 다음으로 고정한다.

1. 툴팁·팝오버 레이어 구조 복구
2. 전투 이벤트 기반 VFX 확장
3. 전투·UI 합성 SFX 확장
4. 모바일·접근성·성능·실브라우저 검증

밸런스 안정화와 사람 N≥5 플레이테스트 판정은 P8 완료 후 P9에서 수행한다. 연출이 불완전한 상태에서 손맛·가독성·난이도를 확정하지 않는다.

## 2. 현재 상태와 원인

### 2.1 툴팁 레이어 결함

현재 `Keyword`와 턴 버프 툴팁은 트리거 요소의 자식으로 `position: absolute` 렌더링된다. 툴팁 자신의 `z-index`가 높아도 다음 부모 경계를 벗어날 수 없다.

- `.combat-shell { overflow: hidden; }`
- `.skill-row { isolation: isolate; }`
- `.skill-card .card-effects { overflow: hidden; }`
- 전장·스킬 레일·HUD가 각각 같은 수준의 별도 스태킹 컨텍스트를 형성한다.

따라서 현상은 단순 숫자 부족이 아니다. `z-index: 9999` 같은 국소 수정은 클리핑과 부모 스태킹 컨텍스트를 해결하지 못하므로 금지한다.

### 2.2 전투 피드백 부족

현재 VFX는 피격, 방어, 화상, 위축, 코인 앞·뒷면 공개 중심이다. P6·P7에서 추가된 회복, 양면 속성, 과열, 쿨다운 감소, 임시 코인, 소환 장비 이벤트는 규칙은 동작하지만 전용 피드백이 부족하다.

### 2.3 사운드 부족

현재 합성 SFX는 다음 7종뿐이다.

- 앞면·뒷면 플립
- 피격
- 방어
- 구매
- 승리·패배

동전 장전·회수·소비, 스킬 계열, 속성 발동, 회복, 과열, 소환, 쿨다운 복귀를 청각적으로 구분할 수 없다.

## 3. 목표와 비목표

### 목표

- 모든 툴팁과 팝오버가 카드·HUD·스크롤·전장에 잘리지 않는다.
- 툴팁이 화면 밖으로 나가지 않고 상하 위치를 자동 전환한다.
- 전투 이벤트를 시각·청각 피드백으로 일관되게 매핑한다.
- 화염·마나·냉기·전기·혈액과 과열·소환을 서로 구분할 수 있다.
- 기존 결정론 코어와 전투 수치에는 영향을 주지 않는다.
- reduced-motion, 음소거, 키보드, 터치, 모바일 계약을 유지한다.
- 신규 런타임 의존성과 외부 오디오 파일을 추가하지 않는다.

### 비목표

- 캐릭터·스킬·몬스터 수치 조정
- 신규 캐릭터, 빌드 또는 노드 추가
- BGM 제작
- 캔버스/WebGL 렌더러 도입
- 화면 흔들림을 게임 규칙이나 명중 판정과 연결
- 기존 스프라이트 아틀라스 재생성

## 4. P8.1 공용 오버레이 레이어

### 4.1 구조

앱 최상위에 공용 `OverlayProvider`를 두고 `document.body` 아래의 전용 포털 루트에 오버레이를 렌더링한다. 앵커 요소 안에는 트리거와 접근성 연결 속성만 남긴다.

공용 레이어는 다음 이름과 순서만 사용한다.

| 레이어 | CSS 토큰 | 값 | 용도 |
|---|---|---:|---|
| world | `--z-world` | 0 | 배경·스프라이트 |
| stage | `--z-stage` | 10 | 전장·카드·HUD |
| controls | `--z-controls` | 100 | 런 메뉴 버튼·영속 제어 |
| popover | `--z-popover` | 200 | 더미 인스펙터 등 상호작용 팝오버 |
| tooltip | `--z-tooltip` | 300 | 키워드·상태·프리뷰 툴팁 |
| drag | `--z-drag` | 400 | 동전 드래그 프록시 |
| modal | `--z-modal` | 500 | 타이틀 확인·런 메뉴·결과 모달 |
| notice | `--z-notice` | 600 | 오류·저장 실패 등 최상위 알림 |

컴포넌트 CSS에 임의의 세 자리 이상 `z-index`를 새로 쓰지 않는다. 모든 공용 오버레이는 위 토큰 중 하나를 사용한다. 카드 내부 장식처럼 로컬 순서만 필요한 요소는 0~9 범위만 사용한다.

### 4.2 앵커 배치 규칙

공용 `AnchoredOverlay`는 다음 규칙을 따른다.

- `position: fixed`로 포털 루트에 렌더링한다.
- 기본 위치는 앵커 위쪽 중앙이다.
- 위 공간이 부족하면 아래로 전환한다.
- 좌우는 뷰포트 8px 패딩 안으로 클램프한다.
- 열린 동안 `resize`, 캡처 단계 `scroll`, 앵커/내용 크기 변경 시 위치를 다시 계산한다.
- 앵커가 DOM에서 사라지거나 화면에서 완전히 벗어나면 닫는다.
- 툴팁은 `pointer-events: none`, 팝오버는 `pointer-events: auto`다.
- 모달이 열리면 모달 바깥의 툴팁·팝오버를 모두 닫는다.

외부 위치 계산 라이브러리는 추가하지 않는다. `getBoundingClientRect`, `ResizeObserver`, 브라우저 이벤트만 사용한다.

### 4.3 이전 대상

다음 표면을 공용 레이어로 이전한다.

1. `Keyword`의 `.kw-tip`
2. `TurnBuffChip`의 `.turn-buff-tip`
3. 스킬 기대값 `.preview-tip`
4. 뽑기·버림·소모 영역 `PilePopover`
5. 동전 `.drag-proxy`
6. 런 메뉴와 파괴적 동작 확인 모달

`rejection-chip`, `pile-flow`, 플로팅 숫자는 입력을 받지 않는 짧은 피드백이므로 포털로 옮기지 않고 레이어 토큰만 적용한다.

### 4.4 툴팁 상호작용과 접근성

- 마우스: hover 진입 시 열고 이탈 시 닫는다.
- 키보드: focus 시 열고 blur 또는 Escape 시 닫는다.
- 터치: 탭으로 열고 같은 트리거 재탭·바깥 탭·Escape로 닫는다.
- 기존 `aria-describedby`와 `role="tooltip"` 연결을 유지한다.
- 팝오버는 `role="dialog"`, `aria-expanded`, `aria-controls`를 유지한다.
- Escape로 닫아도 트리거 포커스를 잃지 않는다.
- 모션 감소 설정에서는 위치 전환 애니메이션을 사용하지 않는다.

## 5. P8.2 전투 이벤트 VFX

### 5.1 아키텍처

코어 `CombatEvent`를 변경하지 않는다. UI에 순수 매퍼를 두어 `CombatEvent[]`를 짧은 `FeedbackCue[]`로 변환한다. 렌더러는 큐만 소비하며 전투 상태를 변경하지 않는다.

각 큐는 다음 중 하나의 목표만 가진다.

- unit: 플레이어 또는 적 패널/스프라이트
- coin: 손패·소켓·더미의 동전
- card: 스킬 카드
- summon: 소환 장비 슬롯
- stage: 전장 전체

동일 이벤트를 여러 UI에서 중복 해석하지 않는다. 기존 `playSfx`, VFX 집합, 플로팅 텍스트 분기를 매퍼의 결과로 단계적으로 통합한다.

### 5.2 이벤트 매핑

| 코어 이벤트 | 시각 피드백 |
|---|---|
| `coinsDrawn` | 손패 방향으로 짧은 금빛 이동·도착 펄스 |
| `coinPlaced` | 소켓 스냅·얇은 링 |
| `coinUnplaced` | 손패 복귀·짧은 역방향 펄스 |
| `coinFlipped` | 기존 앞·뒷면 공개 유지 |
| `skillUsed` | 사용 카드 테두리 펄스·공격/방어/지원 색조 구분 |
| `damageDealt` | 피격 반동·피해 숫자, `blocked > 0`이면 방패 흡수 링 추가 |
| `blockGained` | 파란 방패 팝·수치 상승 |
| `healed`, `enemyHealed` | 녹색 상승 입자·회복 숫자 |
| `statusApplied: burn` | 주황 불씨 펄스 |
| `statusApplied: frostbite` | 청백 결정 파편 펄스 |
| `statusApplied: shock` | 황청 지그재그 펄스 |
| `statusTicked` | 해당 속성의 약한 반복형 틱 |
| `coinCreated` | 생성 위치로 이동하는 작은 동전 실루엣 |
| `coinsConsumed` | 동전이 중앙으로 수축하며 사라짐 |
| `cooldownReduced` | 대상 카드의 쿨다운 배지 청록 플래시 |
| `overheatEntered` | 플레이어 주변 주황 열기 링·상태 칩 진입 펄스 |
| `overheatConsumed` | 사용 카드로 열기 수렴 후 짧은 폭발 |
| `summonAdded` | 빈 소환 슬롯에 마나 윤곽 생성 |
| `summonReplaced` | 기존 장비 퇴장 후 새 장비 진입 |
| `summonActed` | 해당 슬롯 점등·대상 방향 빔 또는 방패 파동 |
| `summonExpired` | 슬롯 입자가 위로 흩어지며 소멸 |
| `combatEnded` | 기존 결과 오버레이 전 짧은 승리/패배 전환 |

### 5.3 강도 규칙

- 일반 행동은 120~320ms 안에 끝난다.
- 과열 소비·소환 교체·보스 처치만 450ms까지 허용한다.
- 화면 전체 흔들림은 강한 플레이어 공격 또는 보스 공격에만 최대 3px, 160ms로 제한한다.
- 동일 프레임 다중 피해는 화면 흔들림을 한 번만 발생시킨다.
- 연출은 명령 입력 잠금 시간을 늘리지 않는다. 기존 해결 큐가 기다리는 구간만 사용한다.
- 이펙트는 텍스트와 실루엣 판독을 가리지 않는다.

신규 VFX는 CSS, pseudo-element, 작은 DOM 파티클로 구현한다. 이미지 자산 추가는 P8 범위에서 금지한다.

## 6. P8.3 합성 SFX 확장

### 6.1 원칙

- 기존 Web Audio 오실레이터·엔벨로프 방식과 무음 폴백을 유지한다.
- 외부 음원, 네트워크 로딩, 신규 패키지를 사용하지 않는다.
- 기본 음소거 정책은 브라우저 자동재생 안정성을 위해 유지한다.
- 첫 전투에는 음소거 토글 위치를 한 번만 알려주는 비차단 힌트를 표시한다.
- 음소거 토글은 전 화면에서 접근 가능하고 설정은 기존 키로 영속한다.

### 6.2 SFX 팔레트

| 범주 | 신규 구분음 |
|---|---|
| 동전 | 선택, 장전, 회수, 생성, 소비, 셔플 |
| 스킬 | 공격, 방어, 지원, 쿨다운 복귀 |
| 속성 | 화염, 마나, 냉기, 전기, 혈액 |
| 상태 | 과열 진입, 과열 소비 |
| 소환 | 생성, 행동, 교체, 소멸 |
| UI | 확인, 거절, 보상 획득, 오류 |
| 결과 | 승리, 패배, 보스 승리 강조 |

톤 설계는 다음 어휘를 사용한다.

- 화염: 짧은 sawtooth 하강 + 노이즈를 흉내 낸 빠른 다중 톤
- 마나: triangle 상승 아르페지오
- 냉기: 높은 sine/triangle의 짧은 결정음
- 전기: square의 빠른 교차 음정
- 혈액: 낮은 sine 두 박자
- 과열: 저음 상승 후 고음 스냅
- 소환: 중음 triangle 3단계 등장·행동 시 짧은 금속성 square

같은 SFX의 반복 피로를 줄이기 위해 재생 시 주파수에 결정론과 무관한 ±2% UI 전용 변주를 허용한다. 이 값은 전투 상태·리플레이·텔레메트리에 기록하지 않는다.

## 8. 테스트 계약

### 단위 테스트

- 레이어 이름은 허용된 8종 외 값을 받지 않는다.
- 앵커 배치 함수가 위/아래 전환과 좌우 8px 클램프를 결정론적으로 계산한다.
- `FeedbackCue` 매퍼가 모든 `CombatEvent` 변형을 누락 없이 처리한다.
- 음소거 중에는 `AudioContext`를 만들지 않는다.
- AudioContext 부재·생성 실패·재생 실패는 게임 진행을 막지 않는다.
- 각 SFX 이름은 비어 있지 않은 톤 배열을 가진다.

### 브라우저 테스트

- 1280×720, 1920×1080, 390×844에서 키워드 툴팁이 뷰포트 안에 완전히 들어온다.
- 카드 효과 행, 상단 상태 칩, 하단 HUD에서 연 툴팁이 각 부모의 경계에 잘리지 않는다.
- `document.elementsFromPoint` 기준 툴팁이 카드·HUD보다 위에 있다.
- 팝오버 위에 툴팁, 드래그 프록시 위에 모달이 그려진다.
- 모달 진입 시 기존 툴팁·팝오버가 닫힌다.
- hover, focus, 터치 탭, Escape, 바깥 탭 동작을 모두 검증한다.
- 화상·동상·감전·회복·과열·소환 이벤트마다 대응 VFX 클래스가 실제 DOM에 붙는다.
- 음소거 해제 후 대응 SFX 호출이 이벤트당 한 번 발생한다.
- 콘솔·페이지 오류와 관련 4xx 응답이 없다.

### 기존 게이트

- `pnpm typecheck`
- `pnpm lint`
- `pnpm test`
- `pnpm ci:sim`
- `pnpm build`
- `pnpm check:assets`
- `pnpm check:a11y`
- `pnpm check:perf`
- `pnpm check:mobile`

## 9. 성능·접근성 예산

- 신규 JS는 압축 전 기준 16KiB 이내를 목표로 하며 기존 차단 예산을 임의로 상향하지 않는다.
- 애니메이션 요소는 평상시 DOM에 누적하지 않고 종료 후 제거한다.
- 동시에 살아 있는 장식 파티클은 전장 전체 24개 이하로 제한한다.
- 레이아웃 측정은 열린 오버레이에만 수행하고 프레임당 한 번으로 합친다.
- `prefers-reduced-motion: reduce`에서는 위치 이동·흔들림·파티클을 제거하고 120ms 이하의 색·테두리 변화만 남긴다.
- 툴팁 텍스트 대비는 4.5:1 이상, 포커스 표시 대비는 3:1 이상을 유지한다.
- SFX는 시각 정보의 유일한 전달 수단이 될 수 없다.

## 10. 완료 기준

P8은 다음 조건을 모두 만족할 때만 완료로 기록한다.

1. 툴팁·팝오버 클리핑 재현 경로가 모두 회귀 테스트로 고정된다.
2. P6·P7 핵심 이벤트인 양면 속성, 회복, 과열, 쿨다운, 임시 코인, 소환에 전용 피드백이 있다.
3. 동전·속성·과열·소환의 합성 SFX가 실제 이벤트와 일대일로 연결된다.
4. reduced-motion과 음소거에서 정보 손실이나 진행 차단이 없다.
5. 데스크톱·모바일 브라우저 검수와 전체 릴리스 게이트가 통과한다.
6. CI 성공 SHA가 라이브 Pages에 배포되고 라이브 툴팁·전투 피드백 스모크가 통과한다.

수치 밸런스, 난이도, 음색 취향, 최종 손맛은 P8 완료로 자동 통과 처리하지 않는다. 해당 판정은 P9 사람 플레이테스트의 입력으로 남긴다.

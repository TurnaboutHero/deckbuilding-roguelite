> 마지막 동기화: 2026-07-16 · 기준: P13 / PRD v2.1 (스택 르미즈·과열 예약·갑주 반향·전속성 보상·몬스터 배치 A·수호자 삭제) · 콘텐츠 `1.7.0-revision` · 런 저장 v9

이 문서는 **현재 코드가 실제로 수행하는 동작**을 설명한다. 제품 의도와 게임 규칙의 단일 정본은 [`PRD.md`](./PRD.md)이며, 캐릭터별 결정 근거는 P10·P11·P12 동기화 기록에 남긴다. 이 문서는 구현을 읽기 위한 기술 안내다.

문서 우선순위와 역사 문서 구분은 [`docs/README.md`](./README.md)를 먼저 본다.

## 2. 모노레포와 의존 방향

```text
packages/core       @game/core     순수 TypeScript 전투·런 엔진
packages/content    @game/content  코인·스킬·적·캐릭터·이벤트 데이터
apps/ui             @game/ui       Vite + React UI
 tools/sim          @game/sim      동일 코어를 사용하는 CLI·대량 시뮬레이터
```

의존 방향은 다음과 같다.

```text
content → core
ui      → core + content
sim     → core + content
```

`core`는 React, DOM, 브라우저 저장소에 의존하지 않는다. UI와 시뮬레이터는 동일한 `ContentDb`, `CombatState`, `Command`, `CombatEvent` 계약을 사용한다.

## 3. 전투 상태와 코인 영역

전투의 단일 상태는 `CombatState`다. 코인은 `CoinUid`로 추적하며 항상 아래 다섯 영역 중 하나에 존재한다.

| 영역 | 의미 |
|---|---|
| `draw` | 뽑을 더미 |
| `hand` | 현재 손패 |
| `placed[slot]` | 장전했지만 아직 사용 선언하지 않은 코인 |
| `discard` | 버림 더미, 뽑을 더미가 비면 리셔플 |
| `exhausted` | 소비형 스킬의 연료, 이번 전투 리셔플에서 제외 |

영구 코인이 `exhausted`로 가도 런의 주머니에서 영구 삭제되지는 않는다. `permanent: false`인 임시 코인만 전투 종료 후 사라진다.

손 상한은 10이다. 드로우와 `addCoin(..., zone: 'hand')`가 같은 상한을 사용하며, 드로우 초과분은 뽑을 더미에 남고 손 생성 초과분은 버림 더미로 간다.

## 4. 현재 턴과 행동 규칙

### 4.1 플레이어 턴 시작

1. 각 슬롯의 `cooldownRemaining`을 1 감소시켜 0 아래로 내려가지 않게 한다.
2. 첫 턴을 제외하고 플레이어 방어를 0으로 초기화한다.
3. `turnStart` 특성·획득 패시브를 획득 순서대로 실행한다.
4. `5 - nextDrawPenalty + nextDrawBonus`개를 드로우하되 총 드로우를 0~8로 제한한다.
5. 드로우 보정치는 소비 후 0으로 초기화한다.

### 4.2 행동 단계

- **턴당 스킬 3회 같은 전역 캡은 없다.**
- 코인이 남고 가용 스킬이 있으면 계속 행동할 수 있다.
- 자동 실행을 시작하기 전에는 코인을 자유롭게 회수하거나 다른 합법 슬롯으로 옮길 수 있다. 이미 장전된 두 동전을 서로 드래그하면 합법성 검증 뒤 자리를 교환한다.
- 카드 제목은 정보 표면이며 발동하지 않는다. 완전 장전된 플립 스킬은 카드 하단 `스킬 사용`으로 언제든 수동 발동할 수 있고, 미사용 스킬의 실행 순번은 카드 배지와 실행 레일에 함께 표시한다. 소비형 스킬의 명시 선택·실행도 유지한다.
- 완전 장전된 플립 스킬의 기본 순서는 장전 완료 순서다. 실행 레일 드래그 또는 `앞`·`뒤` 버튼으로 턴 종료 전까지 바꿀 수 있다.
- UI와 봇은 `legalCommands(state, db)`가 열거한 행동만 제안한다.
- 최종 판정은 `step(state, command, db)`가 담당하며, 실패한 명령은 원래 상태를 변경하지 않는다.

### 4.3 쿨다운

`SkillDefBase.cooldown`은 `0 | 1 | 2 | 3 | 4`이다. 미지정 기본값은 1이다.

| 값 | 의미 |
|---|---|
| 0 | 반복. 코인과 조건이 남는 한 같은 턴에도 재사용 가능 |
| 1 | 사용 후 다음 플레이어 턴 시작에 0이 되어 즉시 가용 |
| 2 | 다음 한 턴 동안 봉인된 뒤 가용 |
| 3 | 다음 두 턴 동안 봉인된 뒤 가용 |
| 4 | 다음 세 턴 동안 봉인된 뒤 가용 |

`oncePerCombat`은 별도의 전투당 1회 잠금이며 활성화 중에는 쿨타임보다 우선한다. 쿨타임을 함께 선언한 스킬은 일회성 제거 후 그 재사용 주기로 복귀한다.

### 4.4 턴 종료

`endTurn` 명령 하나가 다음 플레이어 턴 시작 전까지를 한 번에 진행한다. 코어의 이 계약은 바뀌지 않았고, 직접 `endTurn`을 보내도 장전 스킬은 자동 발동하지 않는다.

UI 기본값은 수동 사용이다. 완전 장전된 플립 스킬은 카드 하단 `스킬 사용`으로 즉시 실행한다. 미사용 완전 장전 스킬이 있는 상태에서 턴 종료를 누를 때만 맥락 경고를 열고, `실행 후 턴 종료`·`사용하지 않고 턴 종료`·`돌아가기`를 제공한다. 경고의 `앞으로 장전 스킬 자동 실행`을 체크하면 설정을 영속하며, 전투 설정 패널에서 다시 끌 수 있다.

자동 실행을 선택했거나 설정이 켜져 있으면 별도 슬롯 ID 순서를 스냅샷하고 기존 `useFlipSkill`을 하나씩 제출한다. 각 스킬의 실제 이벤트 큐와 입력 잠금이 끝난 뒤 다음 스킬로 진행하며, 코인·장비 종류·기존 소환·적 대상 선택에서는 같은 실행 토큰을 유지한 채 멈췄다가 확정 후 한 번만 재개한다. 모든 예약이 해결된 뒤 냉기 도적의 보존 선택을 열고, 보존 확정 후에만 아래의 기존 `endTurn`을 제출한다.

자동 실행을 취소하면 이미 끝난 효과는 유지하고 남은 스킬의 장전은 보존한다. 다음 예약이 새 상태에서 불법이면 이유와 수정·재시도·남은 예약 건너뛰기 선택을 표시하며 조용히 소비하지 않는다. 승리 또는 패배가 확정되면 남은 예약과 적 턴을 중단한다. 부분 장전은 경고·예약에 포함하지 않고 최종 `endTurn`의 기존 회수·버림 규칙을 따른다.

1. 미선언 장전 코인을 손으로 반환한다.
2. 플레이어 화상과 지속 상태를 처리한다.
3. 턴 트리거와 코인 임시 속성 태그를 만료한다.
4. 남은 손패를 버림 더미로 보낸다.
5. 소환 장비가 자동 행동한다.
6. 살아 있는 적이 왼쪽부터 공개된 의도를 실행한다.
7. 적 화상·지속 상태를 처리한다.
8. 살아 있는 적의 다음 의도를 공개한다.
9. 턴을 증가시키고 다음 플레이어 턴을 시작한다.

적 턴 연출은 별도 입력이 아니라 이 과정에서 생성된 `CombatEvent[]`를 UI가 순서대로 재생한다.

## 5. 스킬 해결

### 5.1 장전형(`flip`)

아래는 코어에서 `useFlipSkill` 하나를 해결하는 순서다. UI의 턴 종료 자동 실행은 이 명령을 실행 레일 순서대로 반복할 뿐 여러 스킬을 하나의 코어 명령으로 묶지 않는다.

1. 슬롯, 쿨다운, 전투당 1회 잠금, 비용, 대상을 검증한다.
2. 슬롯 쿨다운과 전투당 사용 상태를 먼저 커밋한다.
3. 장전된 코인을 순서대로 플립하고 `coinFlipped` 이벤트를 만든다.
4. 기본 효과 → 일반 앞/뒷면 효과 → 속성·면 보너스 → 과열 보너스를 수집한다.
5. 스킬 효과를 텍스트 순서로 적용한다. 연속 피해 원자는 하나의 피해 패킷으로 합친다.
6. 장전된 각 속성 코인의 해당 면 고유 효과를 적용한다.
7. 공격 스킬 종료 트리거를 처리한다.
8. 사용한 코인을 버림 더미로 보내고, 과열 강화 분기를 사용했다면 과열을 소비한다.

각 효과 적용 뒤 승패를 검사한다. 전투가 끝나면 남은 효과와 훅은 실행하지 않는다.

번개 결투사의 `르미즈`(P13 스택 모델): `remiseCharges`는 최대 3, 플레이어 턴 시작마다 +1(첫 턴 포함, 전투 초기화 별도 지급 없음), 플레이어 턴 종료에 잔여 소멸. 스택이 있으면 공격 태그 원본 플립 스킬이 첫 코인 플립 전에 1을 확정 소비한다(뒷면도 소비). 첫 코인이 앞면이면 원본 해결 완료 후 같은 스킬을 **가상 코인 사본**으로 1회 반복한다 — 물리 코인 zone 이동 없음, `flip` RNG 스트림만 소비. 반복에서는 피해·방어·적 상태·속성 면 효과·`damageIfReused`만 유효하고, 드로우/생성/회수/덱 이동/`readyRemise`는 원본 전용이다(`remise.onRepeatFinish`와 반복 트리거 패시브만 예외). 반복은 별도 공격 해결로 취급되어 `onAttackSkillResolved`가 재발동하며, 반복이 다시 반복하지는 않는다. 이벤트: `remiseGained/remiseSpent/remiseRepeatResolved`.

면 모드는 데이터에 명시한다.

- `mode: 'any'`: 해당 면이 하나 이상이면 효과를 한 번 적용
- `mode: 'per'`: 해당 면 개수만큼 효과를 반복

### 5.2 소비형(`consume`)

- 손에 있는 지정 속성 코인을 정확한 수만큼 선택한다.
- 유효 속성은 코인의 기본 속성과 임시 `grants`의 합집합이다.
- 선택 코인은 `exhausted`로 이동한다.
- 소비는 플립하지 않으므로 일반 면 보너스와 코인 고유 면 효과가 모두 발동하지 않는다.
- 스킬의 확정 효과와 현재 과열 보너스만 적용한다.

## 6. 속성 코인과 상태이상

모든 속성 코인은 앞면과 뒷면 양쪽에 고유 효과가 있다.

| 코인 | 앞면 | 뒷면 |
|---|---|---|
| 화염 | 대상에게 화상 1 | 대상에게 피해 1 |
| 마나 | 플레이어 방어 1 | 플레이어 방어 2 |
| 냉기 | 대상에게 동상 1턴 | 플레이어 방어 1 |
| 전기(`lightning`) | 대상에게 감전 1턴 | 대상에게 피해 1 |
| 혈액 | 플레이어 회복 1 | 플레이어 방어 1 |

공격형 코인 효과는 단일 대상 스킬이면 그 적, 전체 대상 스킬이면 모든 살아 있는 적에게 적용된다. 자기·무대상 장전 스킬에 공격형 속성 코인이 들어가면 살아 있는 적을 명시적으로 선택해야 한다. 방어·회복 같은 우호 효과는 스킬 대상과 무관하게 플레이어에게 간다.

현재 상태이상은 다음과 같다.

- **화상**: 스택형. 대상의 턴 종료에 스택만큼 방어 무시 피해를 주고 1 감소한다.
- **동상**: 지속형. 동상에 걸린 공격자의 피해를 75%로 낮춘다.
- **감전**: 지속형. 감전 대상이 받는 피해를 150%로 높인다.

## 7. 과열

과열은 `PlayerState.overheat: boolean` + `pendingOverheat: boolean`(P13)으로 표현한다.

1. 즉시 진입: `enterOverheat` 효과(내면의 열정).
2. 예약 진입(P13): `scheduleOverheat` 원자는 비과열 상태에서만 예약을 만들고, 다음 플레이어 턴 시작(쿨다운 감소 직후)에 과열로 전환 후 소멸한다. 그 시점에 이미 과열이면 효과 없이 소멸(중첩 없음). 전투 종료 시 소멸. 패시브 기믹 `residualHeat`(잔열 축적)는 전투당 1회, 비과열 상태의 공격 스킬 화염 동전 사용 시 예약을 만든다.
3. 중첩되지 않고 턴을 넘어 유지된다.
4. `overheatBonus`가 없는 스킬은 과열을 소비하지 않는다.
5. 과열 강화 분기가 있는 스킬이 성공적으로 해결되면 보너스를 적용한 뒤 과열을 소비한다. 화격권의 그 사용은 새 예약을 만들지 않는다.
6. 취소되거나 불법인 명령은 과열을 소비하지 않는다. 이벤트: `overheatScheduled/overheatActivated/overheatEntered/overheatConsumed`.

## 7.0 갑주 반향 (P13)

- 적 턴 동안 `applyDamage`가 적 출처 피해에서 방어로 실제 흡수한 양만 `armorEchoAbsorbedThisEnemyTurn`에 누적한다(화상 등 방어 무시 피해 제외).
- 적 턴 종료에 기본 반향 = min(흡수, 6). 흡수 ≥1일 때만 반향 예열(`echoPreheat` 스택)과 정밀 방어(+4: `precisionDefenseArm` 예약 후 흡수 ≥1 && 잔여 방어 ≤2 충족) 보너스를 더하고 최종 12 상한. 이벤트 `echoComputed`.
- 반향 수치는 다음 플레이어 턴 동안 유지되고 턴 종료에 소멸. 반향 증폭 가용성은 턴당 1회 — `damagePlusEcho`/`aoeDamagePlusEcho`가 발동하면 소모(수치는 비소모, 이벤트 `echoSpent`).
- 어떤 원자도 현재 방어를 직접 소모하거나 피해로 변환하지 않는다. 구 원자(`damagePerBlock`/`blockFromCurrent`/`damagePlusBlock`)는 보상 은퇴된 저장 호환 스킬 정의에서만 살아 있다.

## 7.0.1 워리어 패시브 턴 플래그 (P13)

- `shieldMastery`(방패 숙련): 턴마다 첫 방어 스킬 해결에만 방어 +1 (`shieldMasteryUsedThisTurn`).
- `preparedStance`(빈틈없는 대비): 공격 태그 스킬을 사용한 턴에만 턴 종료 방어 1 (`attackSkillUsedThisTurn`). 잔여 동전 기반 구동작은 제거됨.

## 7.0.2 적 의도 원자 (P13 배치 A)

- `windup {turns, cancelOn.damageThreshold}`: 예고형 준비 — 시작 시 카운트다운·수치·취소 임계 공개, 준비 중 `vulnerableWhileWindup` 배수로 받는 피해 증가, 준비 시작 후 누적 HP 피해가 임계에 닿으면 취소.
- `conditionalAttack(playerHpBelowHalf)`, `phases {hpBelowFraction}`(1회 전환·비가역), `growOnUnblockedDamage(+healOnGrow)`(실피해 시 +1·완전 방어 시 −1, 공격력 가산), `healAlly(lowestHpAlly)`(공개 시 대상 바인딩, 대상 사망 시 실패).
- 전부 의도 수준 원자이며 `enemyTurnStart` 패시브의 자기 대상 한정 검증 벽은 유지된다. 이벤트: `enemyWindupStarted/Ticked/Cancelled`, `enemyPhaseChanged`, `enemyGrew`, `enemyHealFailed`.

## 7.1 P10 캐릭터 전투 상태

- 화염 전사는 첫 피해 감소, 체력 50% 진입 회복, 첫 화상 증가, 화상 턴 종료 방어를 전투·턴 플래그로 제한한다.
- 마도기사는 다음 공격 피해 보너스, 장비 행동 뒤 예약 광역 피해, 재구축 저장, 명령 지속 보존, 마나 소비 누적을 전투 상태로 추적한다.
- 병기 출력은 0~5이며 마나 검 피해와 마나 방패 방어에 공통으로 더한다.
- 마력 갑주 스킬이 현재 방어를 참조해도 방어는 감소하지 않는다.

## 7.2 P11 냉기 도적 전투 상태

- 고유 특성 `이중 주머니`는 턴 종료 시 손패에서 동전 1개를 골라 보존한다. 보존 동전은 버리지 않으며 실제 사용·소비 시 표식이 해제된다.
- 스킬로 얻는 추가 보존은 턴당 최대 2개, 동시에 유지할 수 있는 보존 동전은 최대 3개다.
- 냉기 도적의 보존 선택은 예약된 플립 스킬을 모두 해결한 뒤의 손패/장전 상태에서 열린다. 손패/장전 동전을 클릭하고 턴 종료 버튼 재클릭 또는 Enter로 확정한다. `자동 턴 종료 취소`나 Escape는 보존 선택과 자동 흐름을 함께 취소한다.
- 지정 뽑기는 기본·냉기 후보 중 선택한 실제 동전 정의를 뽑을 더미에서 찾는다. 해당 동전이 없으면 더미를 섞거나 대체 동전을 뽑지 않는다.
- 스킬이 선택하거나 지정 뽑기한 동전 자체를 보존해도 추가 보존 용량은 늘지 않는다. 자동 보존 동전과 `이중 주머니`의 신규 선택 1개는 서로 독립한다.
- 냉기로 취급되는 보존 기본 동전은 해당 스킬의 플립·속성 상호작용에만 냉기로 계산한다. 냉기 소비 비용은 실제 냉기 동전만 지불할 수 있다.
- `일부 소비`, `최소 수량 이상 전부 소비`와 소비량·대상 동상 비례 피해는 소비 커맨드가 확정한 실제 냉기 동전 수를 기준으로 계산한다.
- `차가운 손버릇`의 동상 부여를 먼저 처리한 뒤 `서리 복리`의 첫 공격 피해 +3을 판정한다. 플립과 소비 스킬은 같은 순서를 사용한다.
- 소비 선택 UI는 일부 소비의 `1~최대`, 전부 소비의 `최소 조건·실제 냉기 전부`를 구분하고 속성명을 직접 표시한다. 보존 보너스가 있는 1개 소비도 연료 선택을 열어 보존/일반 냉기를 고를 수 있다.
- 현재 UI는 턴 종료 보존을 직접 선택한다. 지정 뽑기만 별도 선택 화면이 없어 선언 후보 순서의 결정론 제안을 사용한다.

## 7.3 P12 혈액 마검사 전투 상태

- 고유 특성 `혈마검`은 전투 시작 시 치명적이지 않은 체력 1을 자동 투자하며, 런 전체 누적 투자량으로 0~5단계를 계산한다.
- 혈액 동전은 앞면 회복 1, 뒷면 방어 1을 적용한다. 소비형 스킬에서는 플립하지 않으므로 이 면 효과가 발동하지 않는다.
- 혈마검술은 현재 단계의 위력을 참조한다. 단계 2는 혈마검술 방어 2, 단계 3은 전투당 1회 생존, 단계 4는 턴당 첫 소비형 혈마검술 비용 1 감소, 단계 5는 `혈액 공양`을 `혈마해방`으로 전환한다.
- 체력 지불은 방어로 막을 수 없고 체력이 1 미만이 되는 사용을 거부한다. 흡혈은 해당 해결에서 실제 피해를 준 경우에만 회복한다.
- 혈마검 투자량은 `RunSave.bloodSwordInvestment`에 저장하며, 전투 시작·정산·복구에서도 유지한다.

## 8. 콘텐츠 데이터와 검증

콘텐츠는 함수가 아니라 TypeScript 리터럴과 `EffectAtom[]`으로 작성한다. 대표 원자는 다음과 같다.

```ts
{ kind: 'damage', amount: 10 }
{ kind: 'block', amount: 5 }
{ kind: 'heal', amount: 2 }
{ kind: 'draw', count: 1 }
{ kind: 'nextTurnDraw', count: 1 }
{ kind: 'reduceCooldown', amount: 1 }
{ kind: 'enterOverheat' }
{ kind: 'applyStatus', status: 'burn', stacks: 2, to: 'target' }
{ kind: 'addCoin', coin: 'fire', zone: 'draw', count: 1 }
{ kind: 'drawSpecific', coins: ['basic', 'frost'], count: 1 }
{ kind: 'preserveChosenCoin', count: 1 }
{ kind: 'damageByConsumed', base: 5, perCoin: 5 }
```

`validateContentDb()`가 중복 ID, 코스트, 쿨다운, 양면 코인 효과, 공격 스킬 대상, 트리거 재귀, 패시브·장비 안전 원자, 강화 패치 정합성을 검사한다. 신규 콘텐츠 작성 기준은 [`content-design-guide.md`](./content-design-guide.md)를 따른다.

## 9. 결정론 RNG와 프리뷰

런 문자열 시드에서 독립 스트림을 파생한다.

```text
runSeed
└─ combat(combatIndex, attempt)
   ├─ flip
   ├─ shuffle
   └─ ai
```

플립 소비량이 셔플이나 적 스트림을 바꾸지 않는다. 전투를 중도 이탈하고 다시 시작하면 `attempt`를 증가시켜 새 전투 스트림을 만든다.

`previewFlip()`은 별도 피해 공식을 사용하지 않는다. 장전 코인 수 `n <= 5`에 대해 가능한 `2^n` 면 조합을 전수 생성하고, 스크립트 RNG를 주입해 실제 `resolveFlip()`을 실행한다. 그 이벤트를 집계해 피해·방어·자해·회복·화상·생성 코인의 최소, 최대, 기대값을 계산한다.

## 10. 런 구조와 보상

런 그래프는 3막이며 막마다 10번 방문한다.

- 방문 9: 휴식 고정
- 방문 10: 막 보스 고정
- 나머지: 전투, 엘리트, 상점, 이벤트, 보물 후보
- 1막 첫 방문: 일반 전투 고정
- 1막 첫 두 방문 후보: 엘리트 제외

일반 전투 보상은 코인 선택, 엘리트는 추가 스킬 제안, 비최종 보스는 추가 패시브 3택이다. 동전 후보는 P13부터 전속성 가중 비복원 추출(대표 4/기본 3/보유 2/미보유 1, 3개 제시, `weightedCoinOptions`가 보상·상점 공용 정본, 난수 소비 횟수 고정). 상점은 코인·스킬·패시브 구매와 코인 제거를 지원하며 스킬 카드에 전투와 동일한 효과 행·키워드 툴팁을 표시한다. 휴식은 최대 HP의 30% 회복 또는 스킬 강화 중 하나를 선택한다. 막 보스 승리 뒤에는 다음 막 시작 전에 전체 회복한다.

그래프, 보상, 상점, 이벤트 롤은 모두 런 시드에서 이름이 분리된 결정론 스트림으로 만든다.

## 11. 저장

저장은 전투 상태가 아니라 `RunSave`를 전투 경계에서 기록한다.

- 현재 저장 버전: v9
- v1~v8 저장을 순차 마이그레이션 (v8→v9는 비수호자 필드 보존 승격)
- `character === 'guardian'` 저장은 로드하지 않고 `retired-character`로 분류 — 원문을 격리 키에 보존하고 UI가 "수호자는 로스터에서 제외되어 이 런을 이어갈 수 없습니다" 안내와 새 런 시작을 유도한다
- 구정책(기본+대표) 시절의 확정 전 pending 동전 offer는 정의된 coin id면 수용한다(스킬 offer는 계속 캐릭터 경계 검증)
- 6슬롯 저장은 8칸으로 `null` / `false` 패딩
- v7 혈액 마검사 저장은 혈마검 투자량 0으로 이관
- 주 저장과 백업에 이중 쓰기
- 주 저장 손상 시 백업 복구
- 손상 원문은 격리 키에 보존
- 파싱 불가와 미래·미지원 버전을 `corrupt` / `unsupported`로 구분

전투 도중 이탈하면 전투 스냅샷을 복원하지 않고 `attempt + 1`로 해당 전투를 새로 구성한다.

## 12. UI 경계와 이벤트

React UI는 게임 규칙을 직접 판정하지 않는다.

- `legalCommands()`로 가능한 입력을 얻는다.
- 드래그·버튼·키보드 입력을 `Command`로 변환한다.
- `step()` 결과의 새 상태를 렌더링한다.
- `CombatEvent[]`로 플립, 피해 숫자, 상태 변화, 효과음, 결과 티켓을 재생한다.
- 실행 순서는 물리 슬롯과 분리한 슬롯 ID 배열로 관리한다. 완전 장전된 카드의 순번 배지와 실행 레일이 대기·실행·완료·확인 필요 상태를 표시하고, 자동 실행 중에는 편집을 잠근다.
- 현재 턴 요약은 사용·장전·실행 대기·버림 예정 수를 표시한다. 실행 대기는 수동으로 아직 사용하지 않은 완전 장전 스킬이며, 버림 예정은 현재 영역에서 예약·기존 보존 동전을 제외한 추정이다. 이후 스킬의 드로우·생성·보존 효과까지 미리 시뮬레이션하지 않는다.
- `추천 장전`은 `legalCommands()`와 `step()`으로 왼쪽부터 배치만 제안·재검증하고, 미리보기 확인 전에는 상태를 바꾸지 않는다. 대상·장비·소환·보존 선택과 스킬 실행은 대신하지 않는다.
- 최근 해결 결과 20개는 접을 수 있는 전투 기록에 남고, 기존 단기 결과 티켓은 즉시 피드백으로 유지한다.
- 상시 `?` 도움말에서 장전, 실행 순서, 자동 턴 종료, 대상, 버림·보존, 쿨다운, 상태를 설명한다.
- 스킬 상세 프리뷰는 장전 완료만으로 고정하지 않는다. 데스크톱은 카드 hover·행동 바 focus, 터치는 동전 미선택 상태의 카드 본문 탭으로 열며 바깥 입력·Escape·카드 레일 스크롤·해결 시작에 닫는다.
- 다중 적 대상 선택은 전장 상단 안내, 유효 적 강조, 카드 행동 바의 선택 상태와 취소 버튼을 함께 사용한다.
- 모바일 카드·동전 레일은 자체 가로 스크롤, scroll snap과 다음 항목 peek를 사용하며 페이지 스크롤과 분리한다. 동전 레일은 이전·다음 버튼, 끝 상태 비활성화와 현재 위치/전체 수를 함께 제공한다.

전투 설정은 하나의 로컬 저장 계약으로 통합돼 있다. 동전 던지기 `기본`(600ms)·`빠르게`(240ms)·`즉시`(애니메이션 없음), 화면 흔들림, 피해 숫자 크기, 도움말 크기, 고대비, 배경 효과, 모션 줄이기, 소리를 제어한다. 기존 플립 속도·음소거 키를 읽고 쓰는 호환 경로를 유지하며, `prefers-reduced-motion`은 항상 애니메이션 없는 경로를 강제한다.

코인의 마지막 면은 코어 상태에 남지 않는다. UI가 `coinFlipped` 이벤트로 연출용 면을 기억하고, 해당 코인이 다시 드로우되면 기록을 지운다.

## 14. 동기화 규칙

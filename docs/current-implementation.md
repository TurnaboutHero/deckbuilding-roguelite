# 현재 구현 스냅샷

> 마지막 동기화: 2026-07-18 · 기준: P13 / PRD v2.1 + Drive v1.2 + 캐릭터 데이터 테이블 revision 148의 확정 행 + Fable Directive 9~19 · 콘텐츠 `1.7.0-revision` · 런 저장 v10

이 문서는 **현재 코드가 실제로 수행하는 동작**을 설명한다. 제품 의도와 게임 규칙의 단일 정본은 [`PRD.md`](./PRD.md)이며, 캐릭터별 결정 근거는 P10·P11·P12 동기화 기록에 남긴다. 이 문서는 구현을 읽기 위한 기술 안내다.

문서 우선순위와 역사 문서 구분은 [`docs/README.md`](./README.md)를 먼저 본다.

## 2. 모노레포와 의존 방향

```text
packages/core       @game/core     순수 TypeScript 전투·런 엔진
packages/content    @game/content  코인·스킬·적·캐릭터·이벤트 데이터
apps/ui             @game/ui       Vite + React UI
 tools/sim          @game/sim      동일 코어를 사용하는 CLI·대량 시뮬레이터
```

의존 방향은 다음과 같다.

```text
content → core
ui      → core + content
sim     → core + content
```

`core`는 React, DOM, 브라우저 저장소에 의존하지 않는다. UI와 시뮬레이터는 동일한 `ContentDb`, `CombatState`, `Command`, `CombatEvent` 계약을 사용한다.

## 3. 전투 상태와 코인 영역

전투의 단일 상태는 `CombatState`다. 코인은 `CoinUid`로 추적하며 항상 아래 다섯 영역 중 하나에 존재한다.

| 영역 | 의미 |
|---|---|
| `draw` | 뽑을 더미 |
| `hand` | 현재 손패 |
| `placed[slot]` | 장전했지만 아직 사용 선언하지 않은 코인 |
| `discard` | 버림 더미, 뽑을 더미가 비면 리셔플 |
| `exhausted` | 소비형 스킬의 연료, 이번 전투 리셔플에서 제외 |

영구 코인이 `exhausted`로 가도 런의 주머니에서 영구 삭제되지는 않는다. `permanent: false`인 임시 코인만 전투 종료 후 사라진다.

손 상한은 10이다. 드로우와 `addCoin(..., zone: 'hand')`가 같은 상한을 사용하며, 드로우 초과분은 뽑을 더미에 남고 손 생성 초과분은 버림 더미로 간다.

## 4. 현재 턴과 행동 규칙

### 4.1 플레이어 턴 시작

1. 각 슬롯의 `cooldownRemaining`을 1 감소시켜 0 아래로 내려가지 않게 한다.
2. 첫 턴을 제외하고 플레이어 방어를 0으로 초기화한다.
3. `turnStart` 특성·획득 패시브를 획득 순서대로 실행한다.
4. `3 - nextDrawPenalty + nextDrawBonus`개를 드로우하되 총 드로우를 0~8로 제한한다.
5. 드로우 보정치는 소비 후 0으로 초기화한다.

기본 드로우 3개는 v1.2 전투 경제를 먼저 검증하기 위한 과도기 수치다. 기존 드로우 보정과 손 상한 10은 유지하며, 시뮬레이션·플레이테스트 결과로 별도 조정한다.

### 4.2 행동 단계

- **턴당 스킬 3회 같은 전역 캡은 없다.**
- 코인이 남고 가용 스킬이 있으면 계속 행동할 수 있다.
- 행동을 확정하기 전에는 코인을 자유롭게 회수하거나 다른 합법 슬롯으로 옮길 수 있다. 이미 장전된 두 동전을 서로 드래그하면 합법성 검증 뒤 자리를 교환한다.
- v1.2 성공 단계형 플립 스킬은 기본 코인을 자유롭게 장전할 수 있다. 속성 코인은 속성 스킬에만 장전할 수 있으며, 스킬과 코인의 속성이 달라도 허용한다. 무속성 기본기에는 속성 코인을 장전할 수 없다. 아직 성공 단계형으로 이관하지 않은 레거시 스킬은 기존 `requiredCoin`·`requiredElement` 제한을 유지한다.
- 카드 제목과 일러스트는 정보 표면이며 플립 스킬을 개별 발동하지 않는다. 완전 장전된 플립 스킬의 실행 순번은 카드 배지와 실행 레일에 함께 표시한다. 소비형 스킬의 명시 선택·실행은 유지한다.
- 완전 장전된 플립 스킬의 기본 순서는 장전 완료 순서다. 실행 레일 드래그 또는 `앞`·`뒤` 버튼으로 행동 확정 전까지 바꿀 수 있다.
- 화면 아래 `행동 확정`을 한 번 누르면 그 시점의 완전 장전 스킬과 실행 순서를 잠그고 순차 판정을 시작한다. 확정 뒤에는 판정 결과를 보고 코인을 재배치할 수 없다.
- UI와 봇은 `legalCommands(state, db)`가 열거한 행동만 제안한다.
- 최종 판정은 `step(state, command, db)`가 담당하며, 실패한 명령은 원래 상태를 변경하지 않는다.

### 4.3 쿨다운

`SkillDefBase.cooldown`은 `0 | 1 | 2 | 3 | 4`이다. 미지정 기본값은 1이다.

| 값 | 의미 |
|---|---|
| 0 | 반복. 코인과 조건이 남는 한 같은 턴에도 재사용 가능 |
| 1 | 사용 후 다음 플레이어 턴 시작에 0이 되어 즉시 가용 |
| 2 | 다음 한 턴 동안 봉인된 뒤 가용 |
| 3 | 다음 두 턴 동안 봉인된 뒤 가용 |
| 4 | 다음 세 턴 동안 봉인된 뒤 가용 |

`oncePerCombat`은 별도의 전투당 1회 잠금이며 활성화 중에는 쿨타임보다 우선한다. 쿨타임을 함께 선언한 스킬은 일회성 제거 후 그 재사용 주기로 복귀한다.

### 4.4 턴 종료

`endTurn` 명령 하나가 다음 플레이어 턴 시작 전까지를 한 번에 진행한다. 코어의 이 계약은 바뀌지 않았고, 직접 `endTurn`을 보내도 장전 스킬은 자동 발동하지 않는다.

UI는 스킬별 수동 발동과 선택적 자동 실행 설정을 제공하지 않는다. 플레이어는 코인을 배분하고 실행 순서를 정한 뒤 `행동 확정`을 한 번 누른다. 완전 장전된 스킬이 없더라도 같은 버튼으로 보존 및 `endTurn` 흐름을 시작한다. 과거 저장 데이터의 `autoExecuteLoadedSkills` 키는 호환을 위해 읽을 수 있으나 동작에는 반영하지 않고 다시 저장하지 않는다.

행동 확정 시 별도 슬롯 ID 순서를 스냅샷하고 기존 `useFlipSkill`을 하나씩 제출한다. 각 스킬의 실제 이벤트 큐와 입력 잠금이 끝난 뒤 다음 스킬로 진행하며, 코인·장비 종류·기존 소환·적 대상 선택에서는 같은 실행 토큰을 유지한 채 멈췄다가 확정 후 한 번만 재개한다. 모든 예약이 해결된 뒤 냉기 도적의 보존 선택을 열고, 보존 확정 후에만 아래의 기존 `endTurn`을 제출한다.

확정 실행을 취소하면 이미 끝난 효과는 유지하고 남은 스킬의 장전은 보존한다. 다음 예약이 새 상태에서 불법이면 이유와 수정·재시도·남은 예약 건너뛰기 선택을 표시하며 조용히 소비하지 않는다. 승리 또는 패배가 확정되면 남은 예약과 적 턴을 중단한다. 부분 장전은 예약에 포함하지 않고 최종 `endTurn`의 기존 회수·버림 규칙을 따른다.

1. 미선언 장전 코인을 손으로 반환한다.
2. 플레이어 화상·중독과 지속 상태를 처리한다.
3. 반환된 손패의 미사용 속성 동전을 세어 살아 있는 적의 `playerTurnEndPunishment`를 처리한다.
4. 턴 트리거와 코인 임시 속성 태그를 만료한다.
5. 남은 손패를 버림 더미로 보낸다.
6. 소환 장비가 자동 행동한다.
7. 살아 있는 적이 왼쪽부터 공개된 의도를 실행한다. 나이테 재생은 각 적 행동 시작에 먼저 처리한다.
8. 적 화상·중독·지속 상태를 처리하고 나이테 피해 임계 제거와 라운드 성장을 정산한다.
9. 살아 있는 적의 다음 의도를 공개한다.
10. 턴을 증가시키고 다음 플레이어 턴을 시작한다.

적 턴 연출은 별도 입력이 아니라 이 과정에서 생성된 `CombatEvent[]`를 UI가 순서대로 재생한다.

## 5. 스킬 해결

### 5.1 장전형(`flip`)

아래는 코어에서 `useFlipSkill` 하나를 해결하는 순서다. UI의 확정 실행은 이 명령을 실행 레일 순서대로 반복할 뿐 여러 스킬을 하나의 코어 명령으로 묶지 않는다.

1. 슬롯, 쿨다운, 전투당 1회 잠금, 비용, 대상을 검증한다.
2. 슬롯 쿨다운과 전투당 사용 상태를 먼저 커밋한다.
3. 장전된 코인을 순서대로 플립하고 `coinFlipped` 이벤트를 만든다.
4. v1.2 성공 단계형은 성공면 개수를 세어 `successLadder`의 정확히 한 단계만 선택한다. 레거시 스킬은 기본 효과 → 일반 앞/뒷면 효과 → 속성·면 보너스 → 과열 보너스를 수집한다.
5. 선택한 스킬 효과를 텍스트 순서로 적용한다. 연속 피해 원자는 하나의 피해 패킷으로 합친다.
6. 장전된 각 속성 코인의 해당 면 고유 효과를 적용한다.
7. 성공면인 동일 속성 동전이 하나 이상이면 공명을 한 번만 발동한다. `resonanceTriggered`를 먼저 기록하고 공명 효과를 적용한다.
8. 공격 스킬 종료 트리거를 처리한다.
9. 사용한 코인을 버림 더미로 보내고, 과열 강화 분기를 사용했다면 과열을 소비한다.

각 효과 적용 뒤 승패를 검사한다. 전투가 끝나면 남은 효과와 훅은 실행하지 않는다.

#### D9 영구 코인 인챈트

인챈트는 런 주머니의 **영구 코인 사본 하나**에만 붙는 불변 속성이다. 한 사본은 인챈트 하나만 가질 수 있고, 전투 중 생성되는 임시 코인은 인챈트를 가질 수 없으며 인챈트가 든 임시 코인은 해결 전에 거부한다.

| ID | 효과 |
|---|---|
| `sharpness` | 공격 스킬의 성공 단계형에서 이 코인이 성공면이면 피해 +1 |
| `heads-polish` | 이 코인의 앞면 확률을 60%로 변경 |
| `tails-polish` | 이 코인의 뒷면 확률을 60%로 변경 |
| `echo` | 매 전투 첫 사용 뒤 이 코인을 손패로 돌리고, 그 전투에서는 다시 발동하지 않음 |
| `pendulum` | 매 전투 첫 사용에 현재 성공 단계형 스킬의 성공면으로 이 코인만 확정 판정하고, 그 전투에서는 다시 발동하지 않음 |

`pendulum`은 해당 코인의 일반 플립 난수 한 번을 그대로 소비해 뒤의 전투 RNG 순서를 바꾸지 않는다. `echo`는 플립과 소비형 스킬 모두에서 비용 이동보다 우선해 손패로 되돌린다.

플립 후 해결 순서는 다음과 같다. 스킬 효과, 속성 코인 면 효과, 공명 뒤에 **인챈트 해석 단계 4**가 오며, 그 뒤에 캐릭터 패시브와 후속 상태·공격 종료 처리를 한다. 현재 단계 4에서 피해를 더하는 인챈트는 `sharpness`뿐이며, `heads-polish`·`tails-polish`·`pendulum`은 플립면을 정할 때, `echo`는 사용 코인 이동을 정할 때 처리한다.

번개 결투사의 `르미즈`(P13 스택 모델): `remiseCharges`는 최대 3, 플레이어 턴 시작마다 +1(첫 턴 포함, 전투 초기화 별도 지급 없음), 플레이어 턴 종료에 잔여 소멸. 스택이 있으면 공격 태그 원본 플립 스킬이 첫 코인 플립 전에 1을 확정 소비한다(뒷면도 소비). 첫 코인이 앞면이면 원본 해결 완료 후 같은 스킬을 **가상 코인 사본**으로 1회 반복한다 — 물리 코인 zone 이동 없음, `flip` RNG 스트림만 소비. 반복에서는 피해·방어·적 상태·속성 면 효과·`damageIfReused`만 유효하고, 드로우/생성/회수/덱 이동/`readyRemise`는 원본 전용이다(`remise.onRepeatFinish`와 반복 트리거 패시브만 예외). 반복은 별도 공격 해결로 취급되어 `onAttackSkillResolved`가 재발동하며, 반복이 다시 반복하지는 않는다. 이벤트: `remiseGained/remiseSpent/remiseRepeatResolved`.

면 모드는 데이터에 명시한다.

- `mode: 'any'`: 해당 면이 하나 이상이면 효과를 한 번 적용
- `mode: 'per'`: 해당 면 개수만큼 효과를 반복

현재 확정된 기본기는 `공격`(`slash`/화염 격투가 `jab`: 앞면 성공, 0개 효과 없음 / 1개 피해 4, 강화 6)과 `방어`(`guard`/화염 격투가 `fist-guard`: 뒷면 성공, 0개 효과 없음 / 1개 방어 4, 강화 6)다. 네 정의 모두 쿨다운 0으로 반복 사용하며, 레거시 기본 효과·반대면 보너스는 없다. 강화 6은 `캐릭터 데이터 테이블` revision 148의 `COMMON_ATTACK`/`COMMON_DEFENSE` 확정 행을 따른다.

자동 시뮬레이터의 Aggro·Turtle·GreedyEV 정책은 성공 단계 기대값과 생성 코인 기대값을 평가한다. Turtle은 공개된 적 의도에 대한 방어 기대값과 반격 가치를 함께 보며, 효과 가치가 0인 행동은 정본 명령 순서에 따라 턴 종료와 동률 처리한다. D7 전환기 기준선은 500/500 종료, 크래시·불변식 위반 0, 786/2500 전투 완주(31.44%), 평균 10.174턴이다. 이는 v1.2 전사와 v1.1 적의 교차 버전 관측값이며 최종 캐릭터 데이터 갱신 때 재검토한다.

화염 격투가의 v1.2 시작 전용기는 `화염권`(`fire-fist`: 화염/앞면/코스트 2, 성공 단계 피해 2 / 피해 4·화상 1 / 피해 7·화상 2, 화염 공명 화상 1)과 `직격타`(`direct-hit`: 화염/앞면/코스트 2, 성공 단계 피해 1 / 피해 4·다음 드로우 맨 위 임시 화염 1 / 피해 6·같은 생성)다. 화염권 강화는 2개 성공 단계를 피해 9·화상 3으로, 직격타 강화는 0개 성공 단계를 피해 2로 바꾼다. 직격타의 임시 코인은 생성된 경우 다음 턴 손으로 들어오며 전투 종료 후 런 동전 주머니에는 남지 않는다.

### 5.2 소비형(`consume`)

- 손에 있는 지정 속성 코인을 정확한 수만큼 선택한다.
- 유효 속성은 코인의 기본 속성과 임시 `grants`의 합집합이다.
- 선택 코인은 `exhausted`로 이동한다.
- 소비는 플립하지 않으므로 일반 면 보너스와 코인 고유 면 효과가 모두 발동하지 않는다.
- 스킬의 확정 효과와 현재 과열 보너스만 적용한다.

## 6. 속성 코인과 상태이상

모든 속성 코인은 앞면과 뒷면 양쪽에 고유 효과가 있다.

| 코인 | 앞면 | 뒷면 |
|---|---|---|
| 화염 | 대상에게 화상 1 | 대상에게 피해 1 |
| 마나 | 플레이어 방어 1 | 플레이어 방어 2 |
| 냉기 | 대상에게 동상 1턴 | 플레이어 방어 1 |
| 전기(`lightning`) | 대상에게 감전 1턴 | 대상에게 피해 1 |
| 혈액 | 지정한 적에게 코인 피해 1 | 체력 1 상실 후 지정한 적에게 코인 피해 2 |

일반 공격형 코인 효과는 단일 대상 스킬이면 그 적, 전체 대상 스킬이면 모든 살아 있는 적에게 적용된다. 혈액의 `coinDamage`는 예외로 언제나 지정한 적 한 명만 맞힌다. 혈액 코인을 전체·자기·무대상 장전 스킬에 넣으면 기존 적 선택 UI로 대상을 명시해야 한다. 혈액 뒷면의 `loseHp`는 방어를 무시하며, 현재 체력이 1이면 체력 상실과 코인 피해가 한 단위로 함께 불발한다. 소비형 비용으로 쓴 혈액 코인은 플립하지 않아 어느 면 효과도 발생하지 않는다.

현재 상태이상은 다음과 같다.

- **화상**: 스택형. 대상의 턴 종료에 스택만큼 방어 무시 피해를 주고 1 감소한다.
- **중독**: 스택형. 대상의 턴 종료에 스택만큼 방어 무시 피해를 주며 정화 전까지 감소하지 않는다.
- **동상**: 지속형. 동상에 걸린 공격자의 피해를 75%로 낮춘다.
- **감전**: 지속형. 감전 대상이 받는 피해를 150%로 높인다.
- **치유 차단**: 지속형. 플레이어의 직접 회복과 흡혈을 모두 막고 `healPrevented` 이벤트로 실패 이유를 공개한다. 적 치유에는 적용하지 않는다.

## 7. 과열 (저장 호환용 휴면 시스템)

Drive v1.2의 화염 격투가 범위에는 과열이 없으므로 신규 보상에서는 사용하지 않는다. `내면의 열정`, `과열권`, `배기 폭발`, `잔열 축적`은 `retiredFromRewards`로 스킬·패시브 보상 및 상점에서 제외했다. 구 저장과 리플레이를 깨지 않도록 정의와 `PlayerState.overheat: boolean` + `pendingOverheat: boolean` 상태, 이벤트, 턴 플래그는 아래 레거시 계약으로 유지한다.

1. 즉시 진입: `enterOverheat` 효과(내면의 열정).
2. 예약 진입(P13 레거시): `scheduleOverheat` 원자는 비과열 상태에서만 예약을 만들고, 다음 플레이어 턴 시작(쿨다운 감소 직후)에 과열로 전환 후 소멸한다. 그 시점에 이미 과열이면 효과 없이 소멸(중첩 없음). 전투 종료 시 소멸. 퇴역 패시브 기믹 `residualHeat`(잔열 축적)는 구 저장에서 전투당 1회, 비과열 상태의 공격 스킬 화염 동전 사용 시 예약을 만든다.
3. 중첩되지 않고 턴을 넘어 유지된다.
4. `overheatBonus`가 없는 스킬은 과열을 소비하지 않는다.
5. 과열 강화 분기가 있는 레거시 스킬이 성공적으로 해결되면 보너스를 적용한 뒤 과열을 소비한다. v1.2 `화염권`과 `직격타`에는 과열 분기나 과열 예약이 없다.
6. 취소되거나 불법인 명령은 과열을 소비하지 않는다. 이벤트: `overheatScheduled/overheatActivated/overheatEntered/overheatConsumed`.

## 7.0 갑주 반향 (P13)

- 적 턴 동안 `applyDamage`가 적 출처 피해에서 방어로 실제 흡수한 양만 `armorEchoAbsorbedThisEnemyTurn`에 누적한다(화상 등 방어 무시 피해 제외).
- 적 턴 종료에 기본 반향 = min(흡수, 6). 흡수 ≥1일 때만 반향 예열(`echoPreheat` 스택)과 정밀 방어(+4: `precisionDefenseArm` 예약 후 흡수 ≥1 && 잔여 방어 ≤2 충족) 보너스를 더하고 최종 12 상한. 이벤트 `echoComputed`.
- 반향 수치는 다음 플레이어 턴 동안 유지되고 턴 종료에 소멸. 반향 증폭 가용성은 턴당 1회 — `damagePlusEcho`/`aoeDamagePlusEcho`가 발동하면 소모(수치는 비소모, 이벤트 `echoSpent`).
- 어떤 원자도 현재 방어를 직접 소모하거나 피해로 변환하지 않는다. 구 원자(`damagePerBlock`/`blockFromCurrent`/`damagePlusBlock`)는 보상 은퇴된 저장 호환 스킬 정의에서만 살아 있다.

## 7.0.1 워리어 패시브 턴 플래그 (P13)

- `shieldMastery`(방패 숙련): 턴마다 첫 방어 스킬 해결에만 방어 +1 (`shieldMasteryUsedThisTurn`).
- `preparedStance`(빈틈없는 대비): 공격 태그 스킬을 사용한 턴에만 턴 종료 방어 1 (`attackSkillUsedThisTurn`). 잔여 동전 기반 구동작은 제거됨.
- `residualHeat` 관련 예약 플래그는 퇴역 패시브를 가진 구 저장에서만 동작한다. 신규 패시브 후보에는 나타나지 않는다.

## 7.0.2 적 의도 원자 (P13 배치 A)

- `windup {turns, cancelOn}`: 예고형 준비 — 취소 조건은 `{ kind: 'skillDamage', threshold }` 또는 자원 조건 `{ kind: 'enemyResourceAtMost', resource: 'furnaceTemperature', value }`로 구분한다. 시작 시 카운트다운·수치·취소 임계를 공개하고, 준비 중 `vulnerableWhileWindup` 배수로 받는 피해가 증가한다. 스킬 피해 누적이 `threshold`에 닿거나 자원 조건이 충족되면 취소한다.
- `conditionalAttack(playerHpBelowHalf)`, `phases {hpBelowFraction, damageTakenMultiplier}`(1회 전환·비가역), `growOnUnblockedDamage(+healOnGrow/maxStacks/minHpDamageFraction/loseOnFullBlock)`(실피해 비율·상한·완전 방어 감소를 데이터로 지정), `healAlly(lowestHpAlly, cleanse)`(공개 시 대상 바인딩, 대상 사망 시 치유·정화 모두 실패), `growthBranch(atLeast)`(스택 임계 도달 시 같은 패턴 칸의 예고 의도로 교체).
- 배치 A 정련값: 붉은기병은 돌진 피해가 50%보다 많이 HP에 들어갈 때만 기세 +1(최대 3), 기세당 이후 돌진 +15%. 광전사는 광란 페이즈 전체에서 받는 피해 ×1.25. 치유사는 바인딩 대상 회복 12 후 상태 2개를 정화. 흡혈귀 시종은 만찬 최대 5, 4 이상이면 성배의 입맞춤 대신 1턴 예고 후 `7×3` 진홍의 향연을 사용한다.
- 전부 의도 수준 원자이며 `enemyTurnStart` 패시브의 자기 대상 한정 검증 벽은 유지된다. 이벤트: `enemyWindupStarted/Ticked/Cancelled`, `enemyPhaseChanged`, `enemyGrew`, `enemyCleansed`, `enemyHealFailed`.

## 7.0.3 상태·턴 종료·라운드 성장 원자 (P13 배치 B)

- M-11 `재부리 역병의사`: 독액 주사는 피해 7 뒤 실제 HP 피해가 있을 때만 중독 2를 부여한다. `역병 연무`는 1턴 예고 후 중독 1을 먼저 더하고, 갱신된 중독이 5 이상이면 치유 차단 2턴을 부여한다. 중독은 턴 종료에 스택만큼 방어 무시 피해를 주고 정화 전까지 유지된다.
- M-12 `서리묘지 백색망령`: 플레이어 턴 종료에 미사용 속성 동전이 4개 이상이면 동상 1턴을 부여한다. 장전에서 돌아온 동전, 임시·보존 동전, 임시 속성 부여를 받은 기본 동전도 손에 남아 미사용이면 각각 한 번만 센다. 이 검사는 기존 지속 상태 감소 뒤, 손패 버림 전에 수행한다. 기본 순환은 피해 7+동상 1과 1턴 예고형 동상 2다.
- M-14 `흑가시 숲 고목정령`: 나이테는 최대 5, 라운드 종료마다 1을 얻는다. 스택당 받는 피해 8% 감소, 행동 시작 시 최대 HP의 3% 회복을 제공한다. 한 라운드의 실제 HP 피해가 최대 HP의 15% 이상이면 1개, 25% 이상이면 2개를 제거한 뒤 의무 성장 1을 적용하고 피해 누계를 초기화한다. 방어에 막힌 양은 누계하지 않으며 다단·소환·지속 피해의 실제 HP 피해는 합산한다.
- 배치 B는 `EnemyDef.playerTurnEndPunishment`와 `EnemyDef.roundGrowth`, 조건부 `applyStatus`를 추가했다. 봉인·동전 압수·적 소환은 후속 배치로 남기고 `enemyTurnStart` 패시브의 자기 대상 한정 검증 벽은 유지한다. 이벤트: `healPrevented`, `playerTurnEndPunished`, `enemyGrowthReduced`, 기존 `enemyGrew`.
- 근거: `PRD/P13_REVISION_DESIGN_SYNC.md` §6.2·§6.4 및 소유자 승인 20종 상세 명세. 수치는 balance-provisional이다.

## 7.0.4 보호·석화·전쟁기 원자 (P13 배치 C)

- `EnemyDef.threat`, `protectionLink { target: 'highestThreatAlly', redirectFraction, durability, brokenTurns, damageTakenMultiplierWhileBroken }`, `petrify { damageReduction, shatterRawDamageFraction, crackedTurns, crackedDamageTakenMultiplier, cancelWindupIntentId }`, `warBanner { attackAuraPercent, march { attackPercent, turns, shieldMaxHpFraction } }`는 모두 선택적 데이터 필드이며 콘텐츠 검증기가 범위·대상·정수 제약을 확인한다.
- M-05 `아이젠발 성채수호병`은 전투 시작 시 살아 있는 최고 위협 아군을 연결하고, 그 대상 피해의 40%를 수호병으로 재지정한다. 수호병을 대상으로 한 공격 스킬은 다단 여부와 무관하게 내구도를 한 번만 깎는다. 내구도 0이면 링크는 2 적 턴 동안 끊기고, 수호병은 받는 피해 ×1.2가 된 뒤 내구도 2로 복구한다. 재지정된 두 피해 조각은 각자의 방어를 따로 사용한다.
- M-06 `성 오델리아의 봉헌 가고일`은 `petrify` 의도 해결 뒤 석화 상태가 된다. 석화 중 받는 피해는 70% 감쇄되지만 감쇄 전 피해 누적이 최대 HP의 20%에 닿으면 다음 `falling-assault` 준비를 취소하고, 석화를 해제한 뒤 1 적 턴의 균열(받는 피해 ×1.3)을 얻는다.
- M-08 `그리폰 왕가의 전쟁기수`가 살아 있는 동안 다른 적의 공격은 +10% 오라를 받는다. `royal-march`는 1턴 예고 후 전쟁기수 자신을 포함한 모든 살아 있는 적에게 2 적 턴 동안 공격 +20%와 최대 HP의 8% 보호막을 부여한다. 보호막은 원천 소유량으로 추적한다.
- 공통 사망 정리는 같은 피해 해결 안에서 보호 링크와 전쟁기수 원천의 행군 보호막·공격 버프를 제거한다. 이벤트는 `damageRedirected`, `protectionLinkBroken/Removed`, `petrifyProgressed/Shattered`, `enemyAuraApplied/Removed`, `enemyMarchRemoved`로 원인과 정리 결과를 남긴다.
- 배치 C 몬스터는 2·3막 조우 풀에 편입되어 있으며 라이브 조우는 최대 3적을 유지한다. 수치와 실제 체감은 `balance-provisional` / `experience-unverified`다.

## 7.0.5 동전 압수·스킬 봉인 원자 (P13 배치 D 기반)

- `EnemyDef.coinSeizure`와 적 행동 `seizeCustody`는 예고 시점 손패에서 최다 공개 속성·대상 동전 ID·상한을 동결한다. 해결 때 아직 손에 남은 지정 동전만 최대 2개, 예고 시 손패 절반의 내림값까지만 전투 한정 `custody` 영역으로 옮긴다. 다른 동전으로 재지정하지 않으며 원래 ID·속성·임시 여부·인챈트를 보존한다.
- M-09 `검은주머니 동전도둑`이 사망하면 자신이 압수한 동전을 압수 순서대로 같은 해결 안에서 버린 더미로 반환한다. 압수 영역은 드로우·셔플에서 제외되고 전투가 런의 영구 동전 가방을 변경하지 않는다.
- `EnemyDef.skillSeal`과 적 행동 `sealRecentSkill`은 최근 플레이어 2턴의 실제 스킬 사용을 기록해 현재 합법적으로 사용할 수 있는 최다 반복 스킬을 고른다. 봉인은 플레이어 턴 2회 동안 해당 스킬의 배치·플립·소비 명령을 코어에서 거부하지만 `턴 종료`는 항상 유지한다.
- 사용 가능한 스킬이 하나뿐이면 봉인 대신 다음 플레이어 턴 1회 동안 해당 스킬 고유 피해·방어·회복·상태이상 스택에 ×0.75(내림)를 적용한다. 코인 고유 피해·속성 효과는 감소시키지 않는다. 이미 같은 M-10의 봉인이 남아 있을 때 재시전하면 봉인을 갱신하지 않고 피해 6만 준다.
- 이벤트 `coinSeizureTelegraphed`, `coinsSeized/Returned`, `skillSealed`, `skillSealFallbackReduced`, `placedCoinsReturned`, `skillSealRepeatStruck`가 예고·발동·방어 정리의 원인을 남긴다. UI는 압수 더미와 소유 적, 봉인/감소 종류와 남은 플레이어 턴을 텍스트·배지로 표시한다.
- M-09/M-10은 2·3막 조우 풀에만 편입되며 라이브 조우는 최대 3적이다. 수치와 실제 체감은 `balance-provisional` / `experience-unverified`다.

## 7.0.6 D15 엘리트와 D16 소환 수명주기

- M-17 `검은가시 심문관 로데릭`은 최근 반복 스킬을 열성으로 추적하고 3중첩에서 예고형 처형과 봉인을 결합한다. M-18 `무너진 왕의 재무관 마르셀`은 생성 가능한 속성만 세금 대상으로 지정하고 미납 시 위조 동전·보호막·예고형 압류를 적용한다.
- M-15 `모르트벨 뼈종 사령술사`와 M-16 `펜마르시 알지기 마녀`는 빈 전투 슬롯에만 소환한다. 해골 시종은 독립 시작 조우에 들어가지 않고, 진흙 알은 수명주기와 1회 지연을 거쳐 늪지 부화체로 변환된다.
- 엘리트·소환물까지 포함한 라이브 적 상한은 3이다. 축약 테스트 DB에서는 선택적 후보를 필터링해 기존 RNG 소비를 보존한다.

## 7.0.7 D17 재의 공작 발데마르와 source-UID 가드

- M-19 `재의 공작 발데마르`(HP 180)는 3막 10번째 방문의 보스로 `ember-archmage`를 대체한다. 잿불 마도왕 정의는 저장·테스트 호환을 위해 DB에 남으며, 최종 보스 보상과 막 전환 보상은 바꾸지 않는다.
- 용광로는 0에서 시작해 최대 6이다. 보스 행동마다 +1, 플레이어 화상 HP 피해마다 한 번 +1, 화상 제거마다 한 번 -2이며, 각 페이즈의 실제 HP 피해 한계 도달은 온도 -1이다(P1 27, P2 19, P3 10). 한계는 전투 시작/페이즈 진입에 고정한 HP의 15%를 올림해 계산한다. 1페이즈는 피해 10 뒤 실제 HP 피해가 있었을 때만 화상 1을 거는 `burning-slash`와 1턴 예고 화상 2 `ember-brand`를 사용한다.
- 용광로가 최대일 때 일반 의도 순환 대신 `coronation`을 1턴 예고하고, 이후 피해 24·화상 3·용광로를 3으로 설정해 해결한다. 용광로가 5 이하이면 해결 대신 취소되어 온도 3으로 설정되며, 3페이즈에서는 성장 2도 제거한다. HP 70% 미만 진입은 플레이어 화상 1 제거·온도 2·최대 두 `ash-vassal` 소환, HP 35% 미만 진입은 온도 2와 최대 5중첩의 행동당 공격 +8% `final ember`를 적용한다.
- `ash-vassal`(HP 24)은 피해 6과 피해 4+화상 1을 순환한다. 소환 때 발데마르의 실제 source UID를 고정해 그 source에서 온 피해에만 가드 하나당 15%를 제공하며 최대 두 가드(총 30%)만 합산한다.
- 가드 합성 순서: 공유 대상 피해 수정 단계에서 살아 있는 가드 중 저장된 `sourceEnemyUid`가 대상 적 UID와 같은 것만 선언한 감소율을 기여한다. `maxSources`까지만 합산한 뒤 기존 수정 피해에 `max(0, 1 - 합산 감소율)`을 곱해 내림하고, 그 다음 기존 방어/보호 재지정 분할을 적용한다. 따라서 두 15% 가드는 70% 피해가 되며, 가드가 사망하면 즉시 기여가 사라진다. UID 결속은 슬롯 이동·변환 뒤에도 유지된다.
- 수치와 체감은 `balance-provisional` / `experience-unverified`이며, 소환 가신은 일반 그래프 조우 풀에 독립적으로 들어가지 않는다.

## 7.0.8 D18 무관의 주화왕 아우렐

- M-20 `무관의 주화왕 아우렐`은 2막 보스로 속성세, 최대 6개의 왕실 금고, 납화폐 칙령, 예고형 압류·왕관 몰수를 사용한다. 압수한 실제 코인 사본은 ID·속성·임시 여부·인챈트를 보존하고 같은 속성을 사용하거나 보스 정리 조건을 만족하면 반환된다.
- 생산 빌드에서는 D18 테스트 픽스처와 센티넬 문자열을 제거한다. 개발 픽스처는 localhost와 비생산 빌드에서만 열리며, 전체 모바일 회귀는 실전 조우·페이즈·복구 경로를 별도로 검증한다.
- D15~D18 완료로 P13의 중세 판타지 몬스터 카탈로그 20종이 모두 구현됐다. 세부 수치와 사람 체감은 계속 `balance-provisional` / `experience-unverified`다.

## 7.1 P10 캐릭터 전투 상태

- 화염 전사는 첫 피해 감소, 체력 50% 진입 회복, 첫 화상 증가, 화상 턴 종료 방어를 전투·턴 플래그로 제한한다.
- 마도기사는 다음 공격 피해 보너스, 장비 행동 뒤 예약 광역 피해, 재구축 저장, 명령 지속 보존, 마나 소비 누적을 전투 상태로 추적한다.
- 병기 출력은 0~5이며 마나 검 피해와 마나 방패 방어에 공통으로 더한다.
- 마력 갑주 스킬이 현재 방어를 참조해도 방어는 감소하지 않는다.

## 7.2 P11 냉기 도적 전투 상태

- 고유 특성 `이중 주머니`는 턴 종료 시 손패에서 동전 1개를 골라 보존한다. 보존 동전은 버리지 않으며 실제 사용·소비 시 표식이 해제된다.
- 스킬로 얻는 추가 보존은 턴당 최대 2개, 동시에 유지할 수 있는 보존 동전은 최대 3개다.
- 냉기 도적의 보존 선택은 예약된 플립 스킬을 모두 해결한 뒤의 손패/장전 상태에서 열린다. 손패/장전 동전을 클릭하고 행동 확정 버튼 재클릭 또는 Enter로 확정한다. `확정 흐름 취소`나 Escape는 보존 선택과 확정 흐름을 함께 취소한다.
- 지정 뽑기는 기본·냉기 후보 중 선택한 실제 동전 정의를 뽑을 더미에서 찾는다. 해당 동전이 없으면 더미를 섞거나 대체 동전을 뽑지 않는다.
- 스킬이 선택하거나 지정 뽑기한 동전 자체를 보존해도 추가 보존 용량은 늘지 않는다. 자동 보존 동전과 `이중 주머니`의 신규 선택 1개는 서로 독립한다.
- 냉기로 취급되는 보존 기본 동전은 해당 스킬의 플립·속성 상호작용에만 냉기로 계산한다. 냉기 소비 비용은 실제 냉기 동전만 지불할 수 있다.
- `일부 소비`, `최소 수량 이상 전부 소비`와 소비량·대상 동상 비례 피해는 소비 커맨드가 확정한 실제 냉기 동전 수를 기준으로 계산한다.
- `차가운 손버릇`의 동상 부여를 먼저 처리한 뒤 `서리 복리`의 첫 공격 피해 +3을 판정한다. 플립과 소비 스킬은 같은 순서를 사용한다.
- 소비 선택 UI는 일부 소비의 `1~최대`, 전부 소비의 `최소 조건·실제 냉기 전부`를 구분하고 속성명을 직접 표시한다. 보존 보너스가 있는 1개 소비도 연료 선택을 열어 보존/일반 냉기를 고를 수 있다.
- 현재 UI는 턴 종료 보존을 직접 선택한다. 지정 뽑기만 별도 선택 화면이 없어 선언 후보 순서의 결정론 제안을 사용한다.

## 7.3 P12 혈액 마검사 전투 상태

- 고유 특성 `혈마검`은 전투 시작 시 치명적이지 않은 체력 1을 자동 투자하며, 런 전체 누적 투자량으로 0~5단계를 계산한다.
- 혈액 동전은 육참골단형 공격 코인이다. 앞면은 지정 적에게 코인 피해 1, 뒷면은 방어를 무시하고 체력 1을 잃은 뒤 같은 적에게 코인 피해 2를 준다. 체력 1에서는 뒷면의 두 효과가 함께 불발하고 `bloodCoinFizzle` 이벤트를 남긴다.
- 코인 피해는 `source: 'coin'`인 별도 피해 패킷이므로 스킬 피해 기반 흡혈·턴 트리거를 발동하지 않는다. 흡혈은 혈액 마검사 스킬 원자에서만 제공한다.
- 혈마검술은 현재 단계의 위력을 참조한다. 단계 2는 턴의 첫 혈마검술 사용 후 방어 2, 단계 3은 턴의 첫 혈마검술 처치 시 임시 혈액 동전 1개를 버림 더미에 추가, 단계 4는 턴당 첫 소비형 혈마검술 비용 1 감소, 단계 5는 `혈액 공양`을 `혈마해방`으로 전환한다.
- 체력 지불은 방어로 막을 수 없고 체력이 1 미만이 되는 사용을 거부한다. 흡혈은 해당 해결에서 실제 피해를 준 경우에만 회복한다.
- 혈마검 투자량은 `RunSave.bloodSwordInvestment`에 저장하며, 전투 시작·정산·복구에서도 유지한다.

## 8. 콘텐츠 데이터와 검증

콘텐츠는 함수가 아니라 TypeScript 리터럴과 `EffectAtom[]`으로 작성한다. 대표 원자는 다음과 같다.

```ts
{ kind: 'damage', amount: 10 }
{ kind: 'coinDamage', amount: 2 }
{ kind: 'loseHp', amount: 1 }
{ kind: 'block', amount: 5 }
{ kind: 'heal', amount: 2 }
{ kind: 'draw', count: 1 }
{ kind: 'nextTurnDraw', count: 1 }
{ kind: 'reduceCooldown', amount: 1 }
{ kind: 'enterOverheat' }
{ kind: 'applyStatus', status: 'burn', stacks: 2, to: 'target' }
{ kind: 'addCoin', coin: 'fire', zone: 'draw', count: 1 }
{ kind: 'drawSpecific', coins: ['basic', 'frost'], count: 1 }
{ kind: 'preserveChosenCoin', count: 1 }
{ kind: 'damageByConsumed', base: 5, perCoin: 5 }
```

`validateContentDb()`가 중복 ID, 코스트, 쿨다운, 양면 코인 효과, 공격 스킬 대상, 트리거 재귀, 패시브·장비 안전 원자, 강화 패치 정합성을 검사한다. 신규 콘텐츠 작성 기준은 [`content-design-guide.md`](./content-design-guide.md)를 따른다.

## 9. 결정론 RNG와 프리뷰

런 문자열 시드에서 독립 스트림을 파생한다.

```text
runSeed
└─ combat(combatIndex, attempt)
   ├─ flip
   ├─ shuffle
   └─ ai
```

플립 소비량이 셔플이나 적 스트림을 바꾸지 않는다. 전투를 중도 이탈하고 다시 시작하면 `attempt`를 증가시켜 새 전투 스트림을 만든다.

`previewFlip()`은 별도 피해 공식을 사용하지 않는다. 장전 코인 수 `n <= 5`에 대해 가능한 `2^n` 면 조합을 전수 생성하고, 스크립트 RNG를 주입해 실제 `resolveFlip()`을 실행한다. 그 이벤트를 집계해 피해·방어·자해·회복·화상·생성 코인의 최소, 최대, 기대값을 계산한다.

## 10. 런 구조와 보상

런 그래프는 3막이며 막마다 10번 방문한다.

- 방문 9: 휴식 고정
- 방문 10: 막 보스 고정
- 나머지: 전투, 엘리트, 상점, 이벤트, 보물 후보
- 1막 첫 방문: 일반 전투 고정
- 1막 첫 두 방문 후보: 엘리트 제외

일반 전투 보상은 코인 선택, 엘리트는 추가 스킬 제안, 비최종 보스는 추가 패시브 3택이다. 동전 후보는 P13부터 전속성 가중 비복원 추출(대표 4/기본 3/보유 2/미보유 1, 3개 제시, `weightedCoinOptions`가 보상·상점 공용 정본, 난수 소비 횟수 고정). 엘리트와 보스는 같은 3개 동전 후보에 각각 다른 인챈트를 짝지어 제시한다. 거절을 명시적으로 선택할 수 있으며, 선택한 짝만 새 영구 코인 사본에 기록된다. 인챈트 제시는 `reward-enchant` 독립 RNG 스트림을 써 일반 보상 RNG를 바꾸지 않는다. 상점은 코인·스킬·패시브 구매와 코인 제거를 지원하며 스킬 카드에 전투와 동일한 효과 행·키워드 툴팁을 표시한다. 휴식은 최대 HP의 30% 회복 또는 스킬 강화 중 하나를 선택한다. 막 보스 승리 뒤에는 다음 막 시작 전에 전체 회복한다.

그래프, 보상, 상점, 이벤트 롤은 모두 런 시드에서 이름이 분리된 결정론 스트림으로 만든다.

저장은 전투 상태가 아니라 `RunSave`를 전투 경계에서 기록한다.

- 현재 저장 버전: v10
- v1~v9 저장을 순차 마이그레이션 (v8→v9는 비수호자 필드 보존 승격, v9→v10은 주머니 순서에 맞춘 영구 코인 사본 ID를 가산)
- `permanentCoins`는 주머니와 인덱스 정렬된 `{ uid, defId, enchant? }` 원장이다. 새 보상은 `nextUid`로 사본 ID를 추가하고, 제거는 선택한 사본만 지운다. 인챈트가 없는 기존 코인의 정의·순서·수치는 v9→v10에서 바꾸지 않는다.
- `character === 'guardian'` 저장은 로드하지 않고 `retired-character`로 분류 — 원문을 격리 키에 보존하고 UI가 "수호자는 로스터에서 제외되어 이 런을 이어갈 수 없습니다" 안내와 새 런 시작을 유도한다
- 구정책(기본+대표) 시절의 확정 전 pending 동전 offer는 정의된 coin id면 수용한다(스킬 offer는 계속 캐릭터 경계 검증)
- 6슬롯 저장은 8칸으로 `null` / `false` 패딩
- v7 혈액 마검사 저장은 혈마검 투자량 0으로 이관
- 주 저장과 백업에 이중 쓰기
- 주 저장 손상 시 백업 복구
- 손상 원문은 격리 키에 보존
- 파싱 불가와 미래·미지원 버전을 `corrupt` / `unsupported`로 구분

전투 도중 이탈하면 전투 스냅샷을 복원하지 않고 `attempt + 1`로 해당 전투를 새로 구성한다.

## 12. UI 경계와 이벤트

React UI는 게임 규칙을 직접 판정하지 않는다.

- `legalCommands()`로 가능한 입력을 얻는다.
- 드래그·버튼·키보드 입력을 `Command`로 변환한다.
- `step()` 결과의 새 상태를 렌더링한다.
- `CombatEvent[]`로 플립, 피해 숫자, 상태 변화, 효과음, 결과 티켓을 재생한다. 성공 단계형 공명은 `resonanceTriggered` 뒤에 실제 공명 효과 이벤트가 이어져 해결 결과가 원인을 정확히 표시한다.
- 실행 순서는 물리 슬롯과 분리한 슬롯 ID 배열로 관리한다. 완전 장전된 카드의 순번 배지와 실행 레일이 대기·실행·완료·확인 필요 상태를 표시하고, 확정 실행 중에는 편집을 잠근다.
- 현재 턴 요약은 사용·장전·실행 대기·버림 예정 수를 표시한다. 실행 대기는 행동 확정 시 순차 판정될 완전 장전 스킬이며, 버림 예정은 현재 영역에서 예약·기존 보존 동전을 제외한 추정이다. 이후 스킬의 드로우·생성·보존 효과까지 미리 시뮬레이션하지 않는다.
- `추천 장전`은 `legalCommands()`와 `step()`으로 왼쪽부터 배치만 제안·재검증하고, 미리보기 확인 전에는 상태를 바꾸지 않는다. 대상·장비·소환·보존 선택과 스킬 실행은 대신하지 않는다.
- 최근 해결 결과 20개는 접을 수 있는 전투 기록에 남고, 기존 단기 결과 티켓은 즉시 피드백으로 유지한다.
- 상시 `?` 도움말에서 장전, 실행 순서, 행동 확정, 대상, 버림·보존, 쿨다운, 상태를 설명한다.
- 스킬 상세 프리뷰는 장전 완료만으로 고정하지 않는다. 데스크톱은 카드 hover·행동 바 focus, 터치는 동전 미선택 상태의 카드 본문 탭으로 열며 바깥 입력·Escape·카드 레일 스크롤·해결 시작에 닫는다.
- 다중 적 대상 선택은 전장 상단 안내, 유효 적 강조, 카드 행동 바의 선택 상태와 취소 버튼을 함께 사용한다.
- 모바일 카드·동전 레일은 자체 가로 스크롤, scroll snap과 다음 항목 peek를 사용하며 페이지 스크롤과 분리한다. 동전 레일은 이전·다음 버튼, 끝 상태 비활성화와 현재 위치/전체 수를 함께 제공한다.

전투 설정은 하나의 로컬 저장 계약으로 통합돼 있다. 동전 던지기 `기본`(600ms)·`빠르게`(240ms)·`즉시`(애니메이션 없음), 화면 흔들림, 피해 숫자 크기, 도움말 크기, 고대비, 배경 효과, 모션 줄이기, 소리를 제어한다. 기존 플립 속도·음소거 키를 읽고 쓰는 호환 경로를 유지하며, `prefers-reduced-motion`은 항상 애니메이션 없는 경로를 강제한다.

코인의 마지막 면은 코어 상태에 남지 않는다. UI가 `coinFlipped` 이벤트로 연출용 면을 기억하고, 해당 코인이 다시 드로우되면 기록을 지운다.

## 13. 검증과 릴리스 명령

루트 스크립트가 현재 게이트의 정본이다.

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm ci:sim
pnpm build
pnpm check:content
pnpm check:assets
pnpm check:perf
pnpm check:a11y
pnpm check:feedback-regression
pnpm check:mobile
pnpm release:verify
```

GitHub Actions는 타입·린트·테스트·시뮬레이션·빌드·콘텐츠·에셋·번들 예산을 검사하고, 별도 플레이테스트 작업에서 브라우저 탐색, 모바일 플레이, 접근성 대비, 성능 게이트를 실행한다.

## 14. 동기화 규칙

- 제품 규칙이 바뀌면 `docs/PRD.md`와 해당 결정 로그를 먼저 갱신한다.
- 전투 상태, 명령, 효과 원자, 해결 순서가 바뀌면 이 문서를 함께 갱신한다.
- 콘텐츠 제한이나 템플릿이 바뀌면 `content-design-guide.md`를 함께 갱신한다.
- 저장 스키마가 바뀌면 저장 버전, 마이그레이션, 이 문서의 저장 절을 같은 PR에서 갱신한다.
- 테스트 개수나 번들 실제 바이트처럼 자주 변하는 값은 문서에 복사하지 않고 실행 스크립트와 임계값 소스를 링크한다.

# 현재 구현 스냅샷

> 마지막 동기화: 2026-07-22 · 기준: PRD v4.5.1 + D23 즉시 전투 롤백 · 콘텐츠 `1.7.0-revision` · 런 저장 v10

이 문서는 현재 코드가 실제로 수행하는 동작을 설명한다. 제품 규칙의 단일 정본은 [`docs/PRD.md`](./PRD.md), 이번 전투 전환의 승인 기준은 [`PRD/D23_V45_IMMEDIATE_COMBAT_ROLLBACK.md`](../PRD/D23_V45_IMMEDIATE_COMBAT_ROLLBACK.md)다.

## 1. 모노레포와 정본

| 영역 | 패키지 | 책임 |
|---|---|---|
| 전투·런 엔진 | `packages/core` | 상태, 합법 명령, RNG, 피해·상태 해결, 저장 계약 |
| 콘텐츠 | `packages/content` | 동전·스킬·패시브·캐릭터·적·조우 데이터 |
| 게임 UI | `apps/ui` | 타이틀, 튜토리얼, 선택·맵·전투·보상 화면 |
| 시뮬레이터 | `tools/sim` | 동일 코어를 사용하는 결정론 회귀와 밸런스 사전 검증 |

밸런스 수치와 재미 평가는 `balance-provisional` / `experience-unverified`다. 자동 시뮬레이션은 회귀를 막지만 실제 플레이테스트를 대체하지 않는다.

## 2. 현재 전투 계약

플레이어 턴은 다음 순서로 진행한다.

```text
동전 3개 드로우
→ 스킬·대상 선택
→ 손패 동전을 스킬에 걸기
→ 즉시 사용
→ RNG 플립
→ 스킬의 고정 성공면과 일치한 개수로 단계 판정
→ 동전·스킬·상태 효과를 끝까지 해결
→ 남은 동전으로 다음 행동 또는 명시적 턴 종료
```

- 행동 예약, 실행 순서 레일, 전역 행동 확정, 자동 동전 배치, 자동 스킬 실행, 자동 턴 종료는 제품 UI에 없다.
- 반복 스킬은 코스트와 합법적인 동전이 남는 동안 같은 턴에 다시 사용할 수 있다.
- 쿨다운, 턴당 1회, 전투당 1회, 일회성, 명시적 반복 불가 스킬은 해결 직후 잠긴다.
- 일반 플립형 스킬은 플레이어가 면을 선언하지 않는다. 선택한 동전은 RNG로 플립하고 실제 결과를 사용한다.
- 무속성 플립형과 별도 제한이 없는 플립형에는 기본·속성 동전을 자유롭게 걸 수 있다.
- 소비형 스킬은 명시된 속성 동전만 선택해 플립 없이 `exhausted` 영역으로 보낸다.
- 턴 종료 뒤 소환 장비가 행동하고, 적이 예고한 의도를 실행한다.
- 과거 저장과 테스트 마이그레이션을 위한 예약 데이터 타입은 코어 호환 경계에 남아 있을 수 있지만 새 UI와 새 명령 경로에서는 생성하거나 실행하지 않는다.

## 3. 동전과 판정

| 동전 | 앞면 | 뒷면 |
|---|---|---|
| 기본 | 피해 4 | 방어 4 |
| 화염 | 피해 3, 화상 2 | 방어 3, 대상이 화상이면 피해 2 |
| 마나 | 버림 더미에 임시 기본 동전 1 | 다음 턴 드로우 +1 |
| 전기 | 고정 피해 3 | 감전 2 |
| 냉기 | 동상 2 | 방어 3, 다음 턴 방어 2 |
| 혈액 | 체력 2 지불, 피해 7 | 출혈 2 |

- 혈액 앞면은 현재 체력이 2 이하이면 동전 고유 효과만 불발한다. 스킬 성공 단계와 스킬 효과는 정상 해결한다.
- 속성 동전 효과는 스킬 효과와 별도 원천이지만 v4.5 피해 묶음 순서에 따라 같은 패킷 또는 별도 고정 피해로 처리한다.
- 임시 동전은 영역을 이동해도 임시 상태를 유지하고 전투 종료에 제거한다.
- 소모된 영구 동전은 전투 종료 뒤 주머니로 복귀한다.

## 4. 상태이상과 피해

- **화상**: 대상 턴 시작에 수치만큼 체력을 잃고 1 감소한다. 방어를 우회하며 출혈·감전을 발동하지 않는다.
- **동상**: 대상이 가하는 공격 피해를 25% 낮추고 대상 턴 종료에 1 감소한다.
- **출혈**: 대상이 받는 공격 피해를 50% 높이고 대상 턴 종료에 1 감소한다. 고정 피해·방전·화상·자해에는 적용하지 않는다.
- **감전**: 직접 피해를 받을 때 현재 수치만큼 방전 피해를 주고 1 감소한다. 방전은 방어에 막히며 감전·출혈을 다시 발동하지 않는다.
- 같은 대상을 향한 동전·스킬·크리티컬·의존 일반 피해는 하나의 피해 패킷으로 묶고, 명시적 다단 공격은 타격별로 해결한다.
- 고정 피해는 방어를 우회하고 직접 피해이므로 감전을 발동한다.

## 5. 제품 화면 흐름

```text
타이틀
→ 새 시작 또는 이어서 하기
→ 캐릭터 선택
→ 노드 선택형 맵
→ 조우
→ 보상·성장
→ 다음 노드
```

### 타이틀과 설정

- 타이틀에는 `이어서 하기`, `새 런 시작`, `튜토리얼`, `설정`이 있다.
- 저장된 런이 없으면 이어서 하기는 비활성화된다.
- 설정은 동전 연출 속도, 소리, 모션 줄이기, 고대비를 로컬에 저장한다.

### 캐릭터 선택

- 다섯 캐릭터 초상화를 가로 카드로 크게 표시한다.
- 선택한 캐릭터의 HP, 시작 동전, 시작 스킬, 고유 패시브를 한 화면에서 확인한 뒤 별도 시작 버튼으로 확정한다.
- 수호자 캐릭터는 선택 목록과 콘텐츠 풀에서 제거됐다.

### 노드 맵

- 기존 `RunGraph.layers`를 사용해 현재 갈 수 있는 노드를 활성화한다.
- 일반 전투, 엘리트, 보물, 상점, 휴식, 이벤트, 보스가 기존 런 진행 규칙으로 연결된다.

### 튜토리얼

1. 게임 목표와 노드 진행
2. 기본·속성 동전
3. 스킬 코스트·유형·효과
4. 동전 선택·걸기·즉시 사용
5. 뽑을 더미·손패·버림 더미
6. 고정 시드 연습 전투

튜토리얼은 일반 런 저장과 보상에서 분리한다.

### 패시브 인벤토리

- 런 HUD에서 현재 보유 패시브 수를 확인하고 목록을 열 수 있다.
- 이름, 등급, 속성, 발동 시점, 전체 효과를 `ContentDb.passives`에서 읽어 표시한다.

## 6. 전투 UI

- 1280×720 논리 좌표의 16:9 프레임을 기준으로 한다.
- PC는 레터박스, 모바일 가로는 동적 뷰포트 맞춤, 모바일 세로는 회전 안내를 사용한다.
- 카드·동전·주요 버튼은 최소 44×44 CSS px 조작 영역을 보장한다.
- 스킬 카드에는 코스트, 고정 성공면, 성공 개수별 단계, 쿨다운·반복·일회성, 대상을 표시한다.
- 손패 동전을 선택한 뒤 카드 소켓의 `걸기`를 누르고 `즉시 사용`으로 해결한다.
- 방금 해결한 행동은 플립 결과, 피해·방어·상태 변화만 짧게 보여준다.
- 누적 전투 기록 UI는 렌더링하지 않는다. 코어 이벤트 스트림은 애니메이션, 테스트, 텔레메트리를 위해 유지한다.

## 7. 콘텐츠 상태

- 캐릭터: 화염 격투가, 번개 결투사, 냉기 도적, 마도기사, 혈액 마검사.
- 시작 기본기는 `공격`과 `방어`로 통일했다.
- 성장·상점의 동전 보상은 전속성 풀을 가중치로 개방한다. 다른 캐릭터의 전용 스킬은 섞지 않는다.
- 상점은 구매 전 스킬의 코스트, 유형, 효과, 강화 내용을 표시한다.
- 마도기사 갑주 빌드는 현재 방어 누적형이 아니라 실제로 막은 피해를 다음 턴 `갑주 반향`으로 되돌리는 반격형이다.
- 혈액 동전은 회복·방어 동전이 아니라 체력을 대가로 공격하는 육참골단형이며, 흡혈은 혈액 마검사의 전용 스킬 빌드에만 있다.
- 20종 몬스터 카탈로그가 콘텐츠와 조우 풀에 연결돼 있다. 라이브 적 상한은 소환물을 포함해 3이다.

## 8. 저장과 호환

- 타이틀 저장은 기존 런 저장 마이그레이션과 백업 복구 경계를 유지한다.
- 진행 중 전투는 새 규칙으로 직렬화하지 않는다. 구형 전투 중 저장은 안전한 런 경계에서 재개한다.
- 저장 스키마에 남은 구형 자동 실행·예약 키는 읽기 호환만 허용하고 새 저장에는 쓰지 않는다.

## 9. 검증 기준

배포 전 다음을 모두 통과해야 한다.

- TypeScript 타입 검사, ESLint, 단위·통합 테스트
- 결정론 시뮬레이션과 콘텐츠 검증
- 프로덕션 빌드, 에셋 출처, 성능 예산, 명도 대비
- 1600×900 데스크톱과 932×430 모바일 가로 브라우저 플레이테스트
- 타이틀 → 선택 → 맵 → 전투의 실제 클릭 흐름
- 같은 반복 스킬의 한 턴 다중 즉시 사용
- 전투 기록·실행 레일·자동 배치·자동 실행 UI 부재
- 콘솔·페이지·네트워크 오류 0건

검증 증거가 없는 항목은 완료로 표시하지 않는다.

## 1. 제품 개요

본 게임은 카드가 아닌 **동전으로 덱을 구성하는 로그라이크 덱빌딩 전투 게임**이다.

플레이어는 매 턴 코인 주머니에서 동전을 뽑고, 뽑은 동전을 스킬에 배치한 뒤 플립한다. 스킬은 기본 효과를 반드시 가지고 있으며, 동전의 앞면/뒷면 결과에 따라 추가 효과가 붙는다.

기존 기획 자료처럼 코인 결과를 먼저 확인한 뒤 카드를 사용하는 방식이 아니라, 현재 기획에서는 **스킬에 동전을 먼저 배치한 뒤 결과를 확인하는 방식**을 사용한다.

핵심 경험은 다음과 같다.

> 플레이어는 매 턴 제한된 동전을 어떤 스킬에 걸지 판단하고, 결과를 감수하며, 전투 후 보상을 통해 자신의 코인 주머니를 원하는 확률 구조로 성장시킨다.

## 2. 핵심 방향성

### 핵심 문장

**스킬은 실패하지 않는다.**
스킬은 기본 효과를 발동하고, 동전의 면 결과에 따라 추가 효과가 붙는다.

### 왜 족보 조합이 아니라 스킬 장전인가

동전의 앞/뒤와 속성을 조합해 포커식 족보를 만드는 방식도 가능하다. 하지만 족보 방식은 결과적으로 **좋은 조합을 뽑는 확률적 재미**에 집중되며, 플레이어가 전투 중 직접 판단하고 개입하는 전략적 선택은 제한된다.

본 게임의 목표는 높은 족보를 만드는 것이 아니라, **한정된 자원을 어떻게 쓸지 결정하는 선택과 결과의 재미**다. 따라서 동전을 독립적인 족보 시스템으로 쓰는 대신, 플레이어가 동전을 스킬에 직접 장전하고 결과를 확인하는 스킬 시스템을 채택한다. 플레이어는 매 순간 다음의 주도적 판단을 내린다.

- 이 동전을 지금 사용할 것인가?
- 높은 효과(고점)를 노릴 것인가, 안정적인 효과(저점)를 선택할 것인가?

### 게임의 정체성

카드가 아닌 동전으로 덱을 짜는 로그라이크. **고정된 스킬 셋(상수)에 매 턴 무작위로 주어지는 동전(변수)을 조합해 최적 효율을 찾는 퍼즐형 전투**를 지향한다.

- 기본 동전: 스킬 코스트의 뼈대. 덱의 굴림과 안정성을 보장하는 범용 자원
- 속성 동전: 스킬의 성질을 바꾸거나 강력한 특수기(소비형)를 발동시키는 희귀 자원

### 주요 설계 방향

- 코인은 기본 코인과 속성 코인만 사용한다.
- 특수 코인은 사용하지 않는다.
- 캐릭터마다 시작 속성이 존재한다.
- 시작 덱은 단순하게 제공한다.
- 보상을 통해 다른 속성 코인을 추가하며 자유 빌딩을 열어준다.
- 스킬은 크게 장전형 스킬과 소비형 스킬로 나뉜다.
- 일반 스킬은 기본 전투 문법을 담당한다.
- 고급/희귀 스킬은 코인 더미 조작, 손패 생성, 속성 소비 등 빌드 핵심 역할을 담당한다.

플레이어가 매 턴 다음과 같은 고민을 하게 만든다.

- 이 코인을 어떤 스킬에 넣을 것인가?
- 앞면 고점을 노릴 것인가?
- 뒷면 안정성을 선택할 것인가?

## 6. 코인 시스템

### 6.1 코인 종류

코인은 두 종류만 존재한다.

| 코인 종류 | 설명 |
|---|---|
| 기본 코인 | 속성이 없는 범용 코인. 모든 스킬 코스트로 사용 가능 |
| 속성 코인 | 기본 코인처럼 사용 가능하며, 특정 면이 나왔을 때 추가 속성 효과 발동 |

특수 코인은 존재하지 않는다.

### 6.2 기본 코인

기본 코인은 모든 캐릭터가 공통으로 사용하는 범용 자원이다.

```
기본 코인
앞면: 스킬의 앞면 효과 발동
뒷면: 스킬의 뒷면 효과 발동
속성 효과 없음
```

### 6.3 속성 코인

속성 코인은 기본 코인처럼 스킬에 배치할 수 있다.
다만 플립 결과가 특정 면일 경우, 속성 고유 효과가 추가로 발동한다.

예시:

```
화염 코인
앞면: 화상 +1
뒷면: 피해 +1
```

속성 코인은 "기본 코인보다 무조건 강한 코인"이 아니라,
기본 코인 역할을 하면서 특정 조건에서 추가 가치를 만드는 코인이다.

### 6.4 속성 코인 예시

| 속성 | 앞면 효과 | 뒷면 효과 | 컨셉 |
|---|---|---|---|
| 화염 | 현재 대상에게 화상 1 | 현재 대상에게 피해 1 | 앞면 고점형 공격 |
| 마나 | 방어 1 | 방어 2 | 뒷면 안정형 방어 |
| 냉기 | 현재 대상에게 동상 1 | 방어 1 | 방어적 제어 |
| 전기 | 현재 대상에게 감전 1 | 현재 대상에게 피해 1 | 폭딜 보조 |
| 혈액 | 체력 1 회복 | 방어 1 | 생존 보조 |

상태이상 문법은 두 갈래로 확정한다.

- **스택형** (화상): 스택 수치가 곧 효과 크기. 대상 턴 종료 시 발동 후 스택 1 감소
- **지속형** (동상·감전): 효과 강도는 고정(25% / 50%), 부여 수치는 **지속 턴 수**. 재부여 시 턴 수 가산. 대상 턴 종료 시 잔여 턴 1 감소

현재 프로토타입은 다섯 속성을 모두 구현한다. 비핵심 속성도 동전 자체의 양면 효과만으로 선택 가치가 있어야 한다.

- 단일 적 스킬의 공격형 효과는 선택한 적에게 적용한다.
- 전체 적 스킬의 본체·면 효과와 공격형 속성 효과는 모든 생존 적에게 적용한다.
- 자기·무대상 스킬에 공격형 속성 동전을 장전하면 살아 있는 적을 명시 선택해야 한다.
- 방어·회복처럼 이로운 속성 효과는 스킬 대상과 관계없이 플레이어에게 적용한다.
- 속성 동전을 플립하지 않고 소비하면 면 보너스와 속성 고유 효과는 발동하지 않는다.

### 6.5 코인 더미 구조

전투 중 코인은 다음 영역을 이동한다.

| 영역 | 설명 |
|---|---|
| 코인 주머니 / 뽑을 더미 | 이번 전투에서 드로우 가능한 코인 |
| 손패 | 이번 턴 뽑은 코인 |
| 스킬 슬롯 | 스킬에 배치된 코인 |
| 버림 더미 | 사용되었거나 턴 종료 시 폐기된 코인 |
| 소모 영역 | 소비형 스킬로 사용되어 이번 전투 흐름에서 제거된 코인 |

기본 흐름:

```
뽑을 더미 → 손패 → 스킬 슬롯 → 버림 더미
```

뽑을 더미가 비면 버림 더미를 섞어 다시 뽑을 더미로 만든다.

### 6.6 임시 코인과 영구 코인

코인은 크게 두 상태를 가진다.

| 구분 | 설명 |
|---|---|
| 영구 코인 | 전투 후에도 코인 주머니에 남는 코인 |
| 임시 코인 | 전투 중 생성되며, 전투 종료 시 제거되는 코인 |

원칙:

- 스킬로 생성되는 코인은 기본적으로 임시 코인이다.
- 전투 보상, 상점, 이벤트로 얻는 코인은 영구 코인이다.
- 스킬을 통한 영구 변환은 기본적으로 사용하지 않는다.

이렇게 해야 전투 중 폭발력과 로그라이크 성장 구조가 분리된다.

## 7. 캐릭터 시스템

캐릭터마다 시작 코인 구성과 대표 속성이 다르다.

기본 구조:

```
캐릭터
- 기본 코인 8개
- 대표 속성 코인 2개
- 시작 스킬 4개(반복 기본 공격·방어 + 캐릭터 스킬 2개), 최대 장착 슬롯 8개
- 고유 특성(패시브) 1개
```

### 캐릭터 고유 특성 (v0.4 확정)

캐릭터는 **패시브 고유 특성 1개**를 가진다. 설계 원칙:

- 별도 버튼(액티브)이 아니라 전투 시작·턴 시작·속성 코인 사용 같은 **자동 조건**으로 발동한다 (이미 판단 요소가 많은 게임이라 초반 복잡도를 억제)
- 단순 수치 버프(예: 공격 피해 +2)가 아니라 **캐릭터의 코인 규칙을 살짝 바꾸는 효과**여야 한다
- 대표 속성 빌드를 보조한다

확정: **전사 — 불씨 주머니**: 전투 시작 시 임시 화염 코인 1개를 뽑을 더미에 추가한다(전투 종료 시 제거). 첫 손패에 화염 코인이 잡힐 확률이 77.8% → 87.9%, 2개 이상은 22.2% → 42.4%로 올라 정체성이 강해지되 확정 공급은 아니다. 캐릭터 기획 양식은 [content-design-guide.md](./content-design-guide.md) §3 참조.

예시: 전사

```
전사
시작 코인:
- 기본 코인 8개
- 화염 코인 2개

전투 컨셉:
- 공격 중심
- 앞면 보너스 선호
- 화상 누적
- 전투 중 화염 코인 생성
```

캐릭터별 컨셉 예시:

| 캐릭터 | 시작 코인 | 대표 속성 | 플레이 컨셉 |
|---|---|---|---|
| 전사 | 기본 8 / 화염 2 | 화염 | 공격, 화상, 앞면 고점 |
| 수호자 | 기본 8 / 마나 2 | 마나 | 방어, 안정성, 뒷면 활용 |
| 혈액 캐릭터 (미확정) | 기본 8 / 혈액 2 | 혈액 | 전용 캐릭터 기획 확인 전까지 구현 보류 |
| 술사 | 기본 8 / 전기 2 | 전기 | 감전(받는 피해 증가), 연계 폭딜 |
| 냉기 도적 | 기본 8 / 냉기 2 | 냉기 | 동전 보존, 지정 뽑기, 동상 기반 암살 |

초반에는 캐릭터별 대표 속성을 중심으로 플레이하게 하고,
중후반 보상에서 다른 속성 코인을 얻으며 자유 빌딩을 가능하게 한다.

### 성장과 빌드 구축 (런 진행의 두 축)

- **동전 덱 깎기**: 동전의 추가·제거·변환을 통해 주머니 속 확률 구조를 플레이어에게 유리하게 조작한다. (영구 변환·제거는 전투 보상/이벤트에서만 — §10)
- **시너지 스킬 세팅**: 육성 중인 속성 동전·특정 메커니즘과 궁합이 맞는 스킬을 파밍·장착해 빌드를 완성한다.

### 상점과 이벤트 (v0.4 스펙 확정 — 구현은 MVP 이후)

적 처치·이벤트로 얻는 재화(**골드**)로 상점에서 세 가지 행동이 가능하다:

1. **원하는 동전 제거** — 기준가 75골드, 구매할 때마다 +25 (덱 압축이 너무 빨리 끝나지 않게 누진)
2. **무작위 5종 스킬 구매** — 일반 3 + 고급 1 + 무작위 1 구성, 후반일수록 희귀 확률 상승
3. **무작위 3종 동전 구매** — 완전 랜덤이 아니라 캐릭터 대표 속성 가중 + 보유 속성 보정("빌드를 따라오는 랜덤")

**이벤트 원칙**: 스킬·동전을 주는 이벤트에는 **반드시 위험요소**(전투 / 체력 감소 / 재화 소모 / 동전·스킬 소모)가 있다 — 전투 보상과 상점의 가치를 보호하기 위함. 특수 코인이 비목표(§4)이므로 저주류 페널티도 코인이 아닌 위 위험요소로 표현한다. 가격표·수급 초안은 [content-design-guide.md](./content-design-guide.md) §6 참조.

## 8. 스킬 시스템

### 8.1 스킬 기본 표기 규칙

스킬은 다음 문법으로 표기한다.

```
스킬명
코스트: 필요한 코인 수
효과: 무조건 발동하는 기본 효과
앞면: 앞면이 나온 코인에 따라 추가되는 효과
뒷면: 뒷면이 나온 코인에 따라 추가되는 효과
속성: 특정 속성 코인 조건에 따라 추가되는 효과
```

핵심 원칙:

- 효과 = 항상 발동
- 앞면/뒷면 = 추가 효과
- 속성 효과 = 속성 코인 자체의 추가 효과

판정 단위 (확정):

- `앞면: X` / `뒷면: X` → 해당 면이 **1개 이상**이면 X를 1회 발동
- `앞면이 나올 때마다 X` / `앞면 1개마다 X` → 해당 면 코인 **1개당** X를 반복 발동
- 속성 코인의 속성 효과는 항상 **코인 개별 판정** (화염 코인 각각이 앞면이면 각각 화상 +1)
- 일회성: 일부 희귀 스킬에는 **전투당 1회만 사용 가능한 일회성 조건**이 붙을 수 있다

행동 제한 규칙 (v1.3 확정 — 구 v0.4 턴당 3회 캡 폐지):

- 턴당 사용 횟수 카운터는 없다 — **코인이 남고 스킬이 가용하면 계속 사용**할 수 있다
- 스킬별 **쿨다운**이 사용 케이던스를 정한다: 반복(쿨 0)=같은 턴 무제한, 쿨 1=다음 턴 가용, 쿨 2=한 턴 봉인, 쿨 3=두 턴 봉인. 플레이어 턴 시작에 1씩 감소
- **전투당 1회**는 쿨다운과 별개의 단일 메커니즘 — 다음 전투에서 리셋
- 일반 스킬의 최대 코스트는 **4코인**이다
- **5코스트 스킬은 희귀·일회성·궁극형 전용**으로만 허용한다
- **6코스트 이상의 스킬은 만들지 않는다** — 큰 기술은 코스트가 아니라 조건/충전/소비형으로 표현한다
- 소비형 스킬의 속성 코인 조건은 **1~3개**까지만 사용한다 (1=일반/고급, 2=고급/희귀, 3=희귀/궁극)
- 4+코스트 스킬은 강한 기본치에 더해 드로우/임시 코인/다음 턴 이득/의미 있는 상태 중 1개 이상을 동반한다

> 근거: 캡 대신 쿨다운이 "코스트를 관리한다"는 감각을 잇는다 — 반복 기본기(공격 4+3/방어 4+3)는 코인 소진용 바닥이되 유료 스킬 기대값(코인당 5.5 vs 6.5+)을 지배하지 못한다. 상세는 [PRD/P7_NEW_DESIGN_DECISIONS.md](../PRD/P7_NEW_DESIGN_DECISIONS.md) D1·D2.

### 8.2 장전형 스킬

장전형 스킬은 기본 전투 문법이다. 동전을 슬롯에 장전하고 굴려 앞/뒤 판정 결과를 기반으로 효과를 계산한다. 희귀한 스킬 중에는 전투 중 단 한 번만 발동하는 일회성 조건이 붙기도 한다.

처리 순서:

```
1. 스킬에 코인 배치
2. 스킬 사용
3. 배치된 코인 플립
4. 스킬 기본 효과 적용
5. 앞면/뒷면 추가 효과 적용
6. 속성 코인 효과 적용
7. 최종 효과 반영
```

예시: 베기

```
베기
코스트: 코인 1개
효과: 피해 6
앞면: 피해 +4
뒷면: 추가 효과 없음
```

| 결과 | 최종 효과 |
|---|---|
| 앞면 | 피해 10 |
| 뒷면 | 피해 6 |

예시: 방어

```
방어
코스트: 코인 1개
효과: 방어 5
앞면: 추가 효과 없음
뒷면: 방어 +3
```

| 결과 | 최종 효과 |
|---|---|
| 앞면 | 방어 5 |
| 뒷면 | 방어 8 |

예시: 화염 코인을 베기에 사용

```
베기
효과: 피해 6
앞면: 피해 +4

화염 코인
앞면: 화상 +1
```

화염 코인 앞면 결과: 피해 10 + 화상 1
화염 코인 뒷면 결과: 피해 6

예시: 강타 — 다중 코인 장전형의 표준형 (판정 단위 "1개마다" 사용)

```
강타
코스트: 코인 2개
효과: 피해 8
앞면: 앞면이 나올 때마다 피해 +5
뒷면: 추가 효과 없음
```

장전 예시: [기본 코인]과 [화염 코인]을 하나씩 장전 → 판정: 기본(앞) / 화염(앞)

최종 효과: 피해 18 (기본 8 + 앞면 2개 × 5) + 화상 1 (화염 코인 앞면의 속성 효과)

### 8.3 소비형 스킬

소비형 스킬은 특정 속성 코인을 플립하지 않고 즉시 소비해 발동하는 스킬이다.

규칙:

```
1. 손패에 특정 속성 코인이 있어야 한다.
2. 조건을 만족하면 해당 속성 코인을 소비한다.
3. 소비된 코인은 플립하지 않는다.
4. 소비된 코인의 앞면/뒷면 효과는 발동하지 않는다.
5. 대신 스킬의 확정 효과가 발동한다.
```

소비형 스킬의 목적:

| 목적 | 설명 |
|---|---|
| 확정 효과 제공 | 플립 운 없이 강한 효과 발동 |
| 속성 코인 배출구 | 쌓인 속성 코인을 의미 있게 사용 |
| 빌드 보상 | 특정 속성을 많이 모은 플레이어에게 강한 선택지 제공 |
| 선택지 강화 | 플립할지, 소비할지 고민하게 만듦 |

예시: 화염검

```
화염검
조건: 손패의 화염 코인 1개 소비
효과: 이번 턴 피해를 줄 때마다 화상 +1
```

앞/뒤 결과를 확인하지 않고 조건 충족 시 즉시 소모·발동한다. 소비형은 이렇게 **확정적인 고점**을 만들어 준다.

> v0.4 재조정: 턴당 스킬 3회 제한 도입으로 화염검 발동 후 공격 기회가 최대 2회 → 조건을 3개에서 1개(고급)로 하향. 3개 소비 계단은 불의 심장(§9.3, 공격 스킬마다 화상 +2)이 담당한다.

예시: 폭염참

```
폭염참
조건: 손패의 화염 코인 2개 소비
효과: 피해 14, 화상 +3
```

## 9. 스킬 등급

스킬 등급은 단순 수치 차이가 아니라, **코인 시스템에 개입하는 강도**로 구분한다.

| 등급 | 역할 |
|---|---|
| 일반 | 기본 전투 문법, 단순 장전형 스킬 |
| 고급 | 코인 더미 조작, 속성 코인 생성, 소비형 스킬 |
| 희귀 | 손패 생성, 전체 변환, 강력한 소비형 효과, 빌드 핵심 엔진 |

### 9.1 일반 스킬

일반 스킬은 단순하고 직관적이어야 한다.

```
베기
코스트: 코인 1개
효과: 피해 6
앞면: 피해 +4
뒷면: 추가 효과 없음
```

```
방어
코스트: 코인 1개
효과: 방어 5
앞면: 추가 효과 없음
뒷면: 방어 +3
```

```
불타는 일격
코스트: 코인 2개
효과: 피해 8, 임시 화염 코인 1개를 버림 더미에 추가
앞면: 앞면 1개마다 피해 +3
뒷면: 추가 효과 없음
```

### 9.2 고급 스킬

고급 스킬은 코인 더미를 조작하거나 속성 소비를 시작한다.

```
화염 주입
코스트: 코인 1개
효과: 임시 화염 코인 1개를 뽑을 더미에 추가
앞면: 피해 +4
뒷면: 방어 +4
```

```
점화 검술
조건: 손패의 화염 코인 1개 소비
효과: 피해 10, 화상 +2
```

```
용광로
코스트: 코인 1개
효과: 손패의 기본 코인 1개를 이번 턴 동안 화염 코인으로 취급
앞면: 피해 +4
뒷면: 추가 효과 없음
```

### 9.3 희귀 스킬

희귀 스킬은 한 턴 폭발력 또는 빌드 핵심 엔진 역할을 한다. 일부 희귀 스킬에는 **전투당 1회만 사용 가능한 일회성 조건**을 붙여 폭발력을 허용하면서 반복 악용을 차단한다.

```
화염 폭주
코스트: 코인 1개
사용 제한: 일회성 (전투당 1회)
효과: 이번 턴 동안 손패의 기본 코인을 모두 화염 코인으로 취급
앞면: 임시 화염 코인 1개를 손패에 추가
뒷면: 자신에게 피해 2
```

```
불의 심장
조건: 손패의 화염 코인 3개 소비
효과: 이번 턴 공격 스킬을 사용할 때마다 화상 +2
```

> 화염검(§8.3)과의 구분: 화염검은 **모든 피해**(향후 다단 히트·비공격 피해 포함)에 반응하고, 불의 심장은 **공격 스킬 사용** 단위로 더 큰 수치를 준다. 두 스킬 모두 비MVP이며 수치는 시뮬레이터 튜닝 대상.

```
용의 숨결
조건: 손패의 화염 코인 3개 소비
효과: 모든 적에게 피해 18, 화상 +4
```

## 10. 코인 조작 효과 기준

코인 조작 효과는 강도에 따라 등급을 제한한다.

| 효과 | 강도 | 추천 등급 |
|---|---|---|
| 임시 속성 코인을 버림 더미에 추가 | 낮음 | 일반 / 고급 |
| 임시 속성 코인을 뽑을 더미에 추가 | 중간 | 고급 |
| 임시 속성 코인을 손패에 추가 | 높음 | 희귀 |
| 손패의 기본 코인 1개를 속성 코인으로 취급 | 중간 | 고급 |
| 이번 턴 손패의 기본 코인을 전부 속성 코인으로 취급 | 매우 높음 | 희귀 |
| 전투 동안 기본 코인을 속성 코인으로 변환 | 매우 높음 | 희귀 / 제한적 |
| 영구적으로 기본 코인을 속성 코인으로 변환 | 매우 높음 | 전투 보상 / 이벤트 전용 |

기본 원칙:

- 스킬로 발생하는 변환은 대부분 **이번 턴 동안 취급**으로 처리한다.
- 영구 변환은 전투 보상이나 이벤트에서만 제공한다.

## 11. 화염 격투가 시작 세트

### 11.1 시작 코인

- 기본 코인 8개
- 화염 코인 2개
- 총 10개

### 11.2 시작 스킬

```
정권
코스트: 코인 1개
쿨다운: 0(반복 가능)
효과: 피해 4
앞면: 피해 +3
```

## 12. 확률 및 밸런스 기준

공정한 동전 기준 확률은 다음과 같다.

| 조건 | 확률 |
|---|---|
| 코인 1개 앞면 | 50% |
| 코인 1개 뒷면 | 50% |
| 코인 2개 같은 면 | 50% |
| 코인 2개 서로 다른 면 | 50% |
| 코인 2개 모두 앞면 | 25% |
| 코인 3개 같은 면 | 25% |
| 코인 3개 모두 앞면 | 12.5% |

코스트별 위상 (v0.4 확정 — 기대값 기준은 [content-design-guide.md](./content-design-guide.md) §1):

| 코스트 | 의미 | 추천 등급 |
|---|---|---|
| 1 | 가벼운 기본 행동 | 일반 |
| 2 | 핵심 주력기 | 일반/고급 |
| 3 | 강한 행동 | 고급/희귀 |
| 4 | 턴의 핵심 행동 (일반 상한) | 희귀 |
| 5 | 손패를 거는 올인기 | 희귀 + 일회성/궁극 전용 |
| 6+ | **금지** — 조건/충전/소비형으로 표현 | — |

밸런스 원칙:

- 1코인 스킬은 안정적이어야 한다.
- 2코인 스킬은 선택의 재미를 줘야 한다.
- 3코인 스킬은 강한 행동, 4코인 스킬이 턴의 핵심 행동이다 (위 코스트별 위상 표 기준).
- 앞면 3개 같은 낮은 확률 조건은 희귀 스킬이나 고점 보상으로 제한한다.
- 조건 실패로 아무 일도 안 일어나는 구조는 피한다.
- 속성 코인은 장전형과 소비형 양쪽에서 의미가 있어야 한다.
- 손패 생성 효과는 희귀 등급 이상에서 제한적으로 사용한다.

## 13. 주요 리스크와 해결 방향

### 13.1 운빨 게임처럼 느껴질 위험

문제: 코인 결과가 좋지 않을 때 플레이어가 무력감을 느낄 수 있다.

해결:

- 모든 장전형 스킬에 기본 효과 제공
- 앞면/뒷면은 추가 효과로만 사용
- 뒷면 전용 이점 제공
- 소비형 스킬로 확정 선택지 제공

### 13.2 속성 코인이 너무 안 터질 위험

문제: 속성 코인이 있어도 앞면이 나오지 않으면 빌드 체감이 약할 수 있다.

해결:

- 속성 코인을 소비형 스킬의 비용으로 사용
- 속성 코인 생성 스킬 제공
- 고급/희귀 스킬에서 손패나 뽑을 더미 조작 허용
- 후반에는 기본 코인을 속성 코인으로 취급하는 스킬 제공

### 13.3 손패 생성 + 소비형 스킬의 폭주 위험

문제: 손패에 속성 코인을 생성하고 소비형 스킬을 반복하면 과도한 연계가 발생할 수 있다.

해결 후보:

- 희귀 소비형 스킬은 턴당 1회 사용 제한
- 손패 생성 스킬 수 제한
- 소비형 스킬의 조건 코인 수 증가
- 생성 코인은 임시 코인으로만 처리
- 전투 종료 시 임시 코인 제거

### 13.4 캐릭터 컨셉이 흐려질 위험

문제: 초반부터 여러 속성을 너무 많이 제공하면 캐릭터별 정체성이 약해진다.

해결:

- 시작 코인은 기본 8개 + 대표 속성 2개로 고정
- 초반 보상은 대표 속성 위주로 제공
- 중반 이후 다른 속성 보상 개방
- 희귀 보상에서 다속성 빌드 허용

### 15.1 전투 화면 레이아웃 (확정)

```
┌────────────────────────────────────────────────────────┐
│ [HP바]                                        [HP바]    │  ← 유닛별 HP·방어·상태이상 HUD
│   전사 스프라이트            적 스프라이트 ⚔의도 …       │  ← 필드에 서 있는 캐릭터
│                              (적은 다수 존재 가능)       │     (v0.3.1: 캡슐 초상화 → 필드 스프라이트)
│              [ 스킬 카드 ][ 스킬 카드 ][ 스킬 카드 ] …   │  ← 캐릭터 스킬 슬롯 (카드 열)
│                ●●           ●            ●●●           │     노란 원 = 코스트 소켓
│                                                        │
│  (회색 원)      〔 손패 동전 트레이 〕      (버림/소모)   │  ← 하단
│  코인 주머니                               카운트        │
└────────────────────────────────────────────────────────┘
```

- **좌측 하단 회색 원** = 코인 주머니(뽑을 더미). 매 턴 동전 5개가 여기서 나온다. 잔여 매수 표시.
- **스킬 카드의 노란 원 영역** = 스킬 코스트이자 장전 소켓. 스킬은 코스트만큼 동전을 사용하며, 뽑힌 동전을 원하는 스킬의 노란 원 안에 **드래그해 장전**한다. 소켓이 모두 채워지면 사용 가능.
- **카드 본문** = 제목 바 + 일러스트 패널 + 효과 텍스트(기본 / 앞면·뒷면 / 속성 / 소비 조건). **일회성 스킬은 배지로 구분**하고 사용 후 전투 종료까지 비활성 표시.
- **유닛 표현 (v0.3.1 확정)** = 플레이어(좌)와 적(우, 다수 가능)은 캡슐 초상화가 아니라 **필드에 서 있는 스프라이트**. HP·방어·상태이상은 유닛별 HUD 바, 적의 다음 행동 의도는 적 머리 위 아이콘+수치.
- **아트 디렉션 (v0.3.1 확정)**: 밝은 전체 톤 + 어두운 금속 프레임의 HUD/카드, Die in the Dungeon풍 통통한 SD 픽셀 비율(2.5등신), 32px 스프라이트 + Neo둥근모 폰트.
- (보완) **우측 하단** = 버림 더미·소모 영역 카운트. 손패 트레이는 주머니와 카드 열 사이.

## 18. 최종 정리

현재 기획의 핵심은 다음과 같다.

- 코인은 기본 코인과 속성 코인만 존재한다.
- 캐릭터는 기본 코인 8개와 대표 속성 코인 2개로 시작한다.
- 스킬은 기본 효과를 반드시 가진다.
- 앞면과 뒷면은 성공/실패가 아니라 추가 효과 조건이다.
- 속성 코인은 플립하면 속성 효과를 노릴 수 있고, 소비형 스킬에 사용하면 확정 효과의 연료가 된다.
- 일반 스킬은 단순한 장전형 중심, 고급 스킬은 코인 더미 조작과 소비형 중심, 희귀 스킬은 손패 생성, 전체 변환, 빌드 엔진 중심으로 설계한다.
- 스킬 사용은 쿨다운이 관리한다(반복 0/쿨 1~3, v1.3 — 턴당 캡 없음). 코스트는 일반 4가 상한이고, 5는 희귀·일회성·궁극 전용이며, 6 이상은 만들지 않는다. 소비형 조건은 속성 코인 1~3개다.
- 캐릭터는 코인 규칙에 개입하는 패시브 고유 특성 1개를 가진다.

이 방향이면 시스템이 과하게 퍼지지 않으면서도,
동전 덱빌딩만의 정체성과 속성 빌드의 확장성을 둘 다 가져갈 수 있다.

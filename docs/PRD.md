# Coin Combat PRD v2.1

> 제품 규칙 정본 · 2026-07-16 (P13 설계 개정 반영)
> 상태: engineering-safe / balance-provisional / experience-unverified
> P13 결정 근거: [`PRD/P13_REVISION_DESIGN_SYNC.md`](../PRD/P13_REVISION_DESIGN_SYNC.md)

이 문서는 제품과 게임 규칙의 단일 정본이다. P7~P12 문서는 결정 과정과 구현 근거를 보존하는 기록이며, 이 문서와 충돌할 때는 본문을 직접 갱신한다. 더 이상 “최신 변경 이력이 구본문을 덮어쓴다”는 오버라이드 방식은 사용하지 않는다.

## 1. 브랜드 코어

### 한 줄 소개

> 동전을 배치하고 뒤집어, 불확실한 결과를 빌드로 통제하는 턴제 전략 로그라이크.

### 핵심 플레이 감각

> 배치로 계획하고, 플립으로 긴장하며, 빌드로 확률을 통제한다.

### 플레이어 판타지

매 턴 손에 들어온 동전을 어느 스킬에 배치할지 선택하고, 앞면과 뒷면의 불확실성을 캐릭터 고유 메커니즘과 성장 선택으로 유리하게 바꾼다.

### 디자인 원칙

1. 동전 하나마다 선택 가치가 있어야 한다.
2. 앞면과 뒷면은 성공과 실패가 아니라 서로 다른 추가 효과다.
3. 운이 결과를 흔들지만 판단과 빌드가 승패를 결정해야 한다.
4. 같은 동전도 캐릭터마다 다른 전투 리듬을 만들어야 한다.
5. 카드 문구는 짧고 직관적으로 쓰며 세부 규칙은 툴팁으로 설명한다.
6. 수치의 정본은 콘텐츠 데이터와 밸런스 시트에 두고 설명 문서에 중복하지 않는다.

## 2. 제품 범위와 비목표

### 현재 제품 범위

- 데스크톱과 모바일 웹
- 결정론 시드 기반의 3막 런
- 동전 주머니와 장착 스킬을 함께 성장시키는 덱빌딩
- 캐릭터 전용 스킬 풀과 모든 캐릭터가 쓰는 공용 무속성 풀
- 전투, 엘리트, 보물, 상점, 휴식, 이벤트, 보스 노드
- 런 저장, 마이그레이션, 백업 복구

### 비목표

- 별도 행동 횟수 제한
- 포커 족보식 조합
- 플립 결과를 먼저 확인한 뒤 스킬을 고르는 구조
- 6코스트 이상의 일반 스킬
- 일반 보상에서 다른 캐릭터의 **전용 스킬**을 섞는 구조 (동전은 P13부터 전속성 가중 개방 — §7.2)
- 속성 코인과 별개인 특수 코인 계층
- 플레이어 대신 장전·대상·빌드를 결정하는 완전 자동 전투 또는 방치형 진행

## 3. 핵심 플레이 루프

```text
런 시작
→ 연결된 노드 선택
→ 전투·이벤트·상점·휴식 등 조우
→ 동전·스킬·패시브·재화로 빌드 성장
→ 막 보스 도전
→ 다음 막 진입
→ 3막 보스 처치 또는 사망으로 런 종료
```

전투 안에서는 다음 순서를 반복한다.

```text
플레이어 턴 시작 → 동전 드로우 → 스킬 선택 → 동전 장전
→ 필요하면 카드 하단에서 수동 사용·남은 스킬의 실행 순서 확인
→ 턴 종료(미사용 완전 장전이 있으면 실행·버림·돌아가기 선택)
→ 선택한 자동 실행 또는 소환 장비 행동 → 적 의도 실행 → 다음 플레이어 턴
```

## 4. 기본 전투 규칙

### 4.1 턴과 손패

- 플레이어 턴이 시작되면 기본 3개를 드로우한다. 이는 v1.2 전투 경제 검증을 위한 과도기 수치이며, 전투 시뮬레이션과 플레이테스트 뒤 확정한다.
- 사용 가능한 동전이나 스킬이 남아 있어도 언제든 턴 종료 흐름을 시작할 수 있다.
- 스킬에 사용한 동전과 턴 종료 시 손에 남은 동전은 버림 더미로 이동한다.
- 코스트를 모두 채운 플립 스킬은 카드 하단 `스킬 사용`으로 즉시 수동 발동할 수 있다. 사용하지 않고 턴 종료를 누르면 경고에서 순서대로 실행하거나, 사용하지 않고 종료하거나, 전투로 돌아갈 수 있다.
- 부분 장전은 실행 순서에 포함하지 않는다. 최종 `endTurn`에서 손으로 회수된 뒤 다른 미사용 동전과 같은 버림 규칙을 따른다.
- 뽑을 더미가 부족하면 버림 더미를 섞어 새 뽑을 더미로 만들고 부족한 수만큼 계속 뽑는다.
- 손 상한은 10개다.
- 별도의 턴당 행동 횟수 제한은 없다.

### 4.2 스킬 슬롯과 코스트

| 항목 | 규칙 |
|---|---|
| 시작 스킬 | 기본 4개 |
| 기본 장착 슬롯 | 8칸 |
| 빈 슬롯 | 허용 |
| 일반 코스트 | 1~4개 |
| 예외 코스트 | 5개, 희귀·궁극기급에 한함 |
| 6코스트 이상 | 금지 |

### 4.3 플립형 스킬

1. 스킬 코스트와 정확히 같은 수의 동전을 배치한다.
2. v1.2 성공 단계형 플립 스킬에는 기본 동전을 자유롭게 배치할 수 있다. 속성 동전은 속성 스킬에만 배치할 수 있으며 스킬과 다른 속성도 허용한다. 무속성 기본기에는 속성 동전을 배치할 수 없다. 아직 성공 단계형으로 이관하지 않은 레거시 스킬은 기존 동전·속성 요구 조건을 유지한다.
3. 완전 장전된 스킬은 카드 하단 `스킬 사용`으로 즉시 발동할 수 있으며, 동시에 미사용 실행 순서에도 등록된다. 기본 순서는 장전 완료 순서이며 자동 실행 전까지 바꿀 수 있다.
4. 플레이어가 수동 사용하거나 턴 종료 경고·자동 모드가 해당 스킬을 실행하면 배치한 각 동전을 개별적으로 플립한다.
5. 성공 단계형은 성공면 개수에 대응하는 단계 하나만 적용한다. 레거시 스킬은 기존 기본·앞면·뒷면 효과를 적용한다.
6. 실제 면에 따른 속성 동전 고유 효과를 적용한다.
7. 성공한 동전 중 스킬과 같은 속성이 하나 이상이면 공명 효과를 한 번 적용하고 `resonanceTriggered` 이벤트로 원인을 남긴다.
8. 사용한 동전을 버림 더미로 보낸다.

면 효과는 데이터에서 다음 중 하나로 명시한다.

- `any`: 해당 면이 하나 이상이면 한 번 적용
- `per`: 해당 면이 나온 개수만큼 적용

### 4.4 소비형 스킬

- 손패의 요구 속성 동전을 선택해 플립 없이 소비한다.
- 소비한 동전은 이번 전투의 `exhausted` 영역으로 이동해 리셔플에서 제외된다.
- 소비에서는 스킬 면 효과와 속성 동전 면 효과가 발동하지 않는다.
- 영구 동전은 다음 전투의 주머니에 복귀하고, 임시 동전은 전투 종료 시 사라진다.

### 4.5 쿨다운과 일회성

| 표기 | 의미 |
|---|---|
| 반복 | 코스트와 조건이 남으면 같은 턴에도 재사용 가능 |
| 쿨다운 1 | 다음 플레이어 턴 시작에 다시 사용 가능 |
| 쿨다운 N | 플레이어 턴 시작마다 1 감소하고 0에서 재사용 가능 |
| 일회성 | 이번 전투에서 한 번만 사용하며 다음 전투에 초기화 |

일회성과 쿨다운은 별도 규칙이다. 강화로 일회성이 제거되는 스킬은 명시된 쿨다운으로 재사용한다.

### 4.6 대상과 적 의도

- 적은 다음 행동을 의도로 미리 표시한다.
- 플레이어가 턴을 종료하면 살아 있는 적이 표시한 의도를 실행한다.
- 적이 한 명이면 단일 대상 스킬의 대상을 자동 지정한다.
- 적이 여러 명이면 현재 대상을 클릭으로 선택한다. 선택이 없으면 가장 왼쪽의 살아 있는 적을 사용한다.
- 모든 적 대상 스킬은 현재 대상을 무시한다.
- 자기 대상 스킬에 공격형 속성 동전을 사용하면 적용할 적을 함께 지정해야 한다.
- 방어·회복 같은 우호 속성 효과는 스킬 대상과 관계없이 플레이어에게 적용한다.

### 4.7 수동 사용과 선택적 턴 종료 자동 실행

- 완전 장전된 플립 스킬의 카드 하단 행동 바는 `스킬 사용`으로 활성화된다. 카드 제목과 일러스트는 정보·선택 표면이며 발동 버튼이 아니다.
- 기본 설정은 수동 모드다. 미사용 완전 장전 스킬이 없으면 턴 종료 버튼은 곧바로 기존 `endTurn` 흐름으로 진행한다.
- 수동 모드에서 미사용 완전 장전 스킬이 있는 상태로 턴 종료를 누를 때만 맥락 경고를 연다. 경고는 `실행 후 턴 종료`, `사용하지 않고 턴 종료`, `돌아가기`를 제공한다.
- 경고의 `앞으로 장전 스킬 자동 실행`을 체크하고 실행하면 이후 전투는 자동 모드로 저장한다. 전투 설정의 `턴 종료 시 장전 스킬 자동 실행`에서 언제든 다시 끌 수 있다.
- 자동 모드에서는 턴 종료 버튼이 완전 장전 수를 포함해 `스킬 N개 자동 실행 후 턴 종료`로 표시되고 경고 없이 실행한다.
- UI는 물리 슬롯 배열과 별개인 슬롯 ID 순서만 관리한다. 카드의 순번 배지와 실행 레일이 같은 순서를 표시한다.
- 데스크톱은 실행 레일 드래그, 키보드·터치는 `앞`·`뒤` 제어로 순서를 바꾼다. 자동 실행을 시작하면 순서를 고정한다.
- 자동 실행은 기존 `useFlipSkill` 명령을 한 번에 하나씩 제출하고 실제 이벤트 큐와 입력 잠금이 끝난 뒤 다음 스킬로 진행한다. 코어의 직접 `endTurn` 명령은 장전 스킬을 발동하지 않는다.
- 코인, 장비 종류, 기존 소환, 적 대상 선택이 필요하면 같은 실행 항목에서 멈추고, 선택을 확정한 뒤 정확히 한 번 재개한다.
- 냉기 도적의 보존 선택은 실행하기로 한 예약 스킬이 모두 해결된 뒤의 상태로 연다. 보존을 확정해야 기존 `endTurn`을 제출한다.
- 자동 실행 중 취소하면 이미 끝난 효과는 유지하고 아직 실행하지 않은 스킬의 장전은 보존한다.
- 다음 스킬이 새 상태에서 불법이 되면 자동으로 건너뛰지 않는다. 이유와 `순서·장전 수정`, `다시 시도`, `남은 스킬 건너뛰고 종료`를 표시한다.
- 실행 중 승리 또는 패배가 확정되면 남은 예약과 적 턴을 즉시 중단한다.
- 부분 장전은 경고·자동 실행 대상이 아니며 최종 `endTurn`의 기존 회수·버림 규칙을 따른다.
- 소비형 스킬은 장전형 예약 대상이 아니며 기존의 명시 선택·실행 흐름을 유지한다.
## 5. 동전 시스템

### 5.1 동전 종류

- **기본 동전**: 모든 캐릭터와 플립형 스킬에 쓸 수 있는 범용 자원
- **속성 동전**: 기본 동전처럼 배치할 수 있고 앞면·뒷면에 각각 고유 효과가 있는 자원
- **임시 동전**: 전투 중에만 존재하며 어느 더미에 있든 전투 종료 시 제거되는 동전

### 5.2 현재 속성 동전 효과

| 동전 | 앞면 | 뒷면 |
|---|---|---|
| 화염 | 대상에게 화상 1 | 대상에게 피해 1 |
| 마나 | 플레이어 방어 1 | 플레이어 방어 2 |
| 냉기 | 대상에게 동상 1턴 | 플레이어 방어 1 |
| 전기 | 대상에게 감전 1턴 | 대상에게 피해 1 |
| 혈액 | 플레이어 체력 1 회복 | 플레이어 방어 1 |

### 5.3 드로우·보존·탐색

- **즉시 드로우**: 뽑을 더미에서 이번 턴 손패로 가져온다.
- **다음 턴 드로우**: 다음 플레이어 턴의 기본 드로우에 더한다.
- **보존**: 선택한 동전을 턴 종료에 버리지 않고 다음 턴 손패에 남긴다.
- **탐색/지정 드로우**: 뽑을 더미에서 조건에 맞는 실제 동전을 골라 손패로 가져온다.
- 지정한 동전이 없으면 대체 동전을 뽑지 않는다. 사용 전 `대상 없음`으로 표시한다.

기본 보존 상한은 턴당 1개다. 캐릭터 특성·스킬·패시브가 명시적으로 확장할 수 있다.

## 6. 상태이상과 방어

- **화상**: 대상의 턴 종료에 스택만큼 방어 무시 피해를 주고 1 감소한다.
- **동상**: 해당 적이 주는 공격 피해를 25% 낮추고 지속 시간이 감소한다.
- **감전**: 해당 적이 받는 피해를 50% 높이고 지속 시간이 감소한다.
- **방어**: 일반 공격 피해를 막고 다음 자기 턴 시작에 초기화된다.
- 체력 지불과 화상 피해는 방어로 막을 수 없다.

화상 수치를 참조하는 공격은 별도 문구가 없으면 화상을 소모하지 않는다.

## 7. 스킬·패시브 보상 풀

### 7.1 전역 원칙

```text
현재 캐릭터의 보상 후보 = Common + 현재 캐릭터 exclusiveTo
```

- 캐릭터 전용 스킬은 해당 캐릭터를 플레이할 때만 등장한다.
- 공용 무속성 스킬은 모든 캐릭터가 온전히 사용할 수 있어야 한다.
- 속성 코인을 직접 요구하지 않아도 특정 캐릭터 메커니즘을 참조하면 캐릭터 전용으로 분류한다.
- 시작 전용·고유·저장 호환용 레거시 콘텐츠는 일반 보상에서 제외할 수 있어야 한다.
- 같은 원칙을 신규 캐릭터에도 자동 적용한다.

### 7.2 동전 보상 풀 (P13 전속성 가중 개방)

일반 동전 보상과 상점 동전 후보는 **기본 동전 + 모든 속성 동전**에서 가중 비복원 추출로 서로 다른 정의 3개를 제시한다.

```text
가중치: 대표 속성 4 · 기본 동전 3 · 보유 비대표 속성 2 · 미보유 속성 1
```

- "보유"는 현재 런 주머니에 그 속성 동전이 1개 이상 존재함을 뜻한다.
- 결정론: 같은 시드·같은 주머니면 같은 후보. 추출의 난수 소비 횟수는 주머니 내용과 무관하게 고정된다.
- 대표/보유 속성이 우대되지만 어떤 속성도 배제되지 않는다. 비대표 동전의 가치는 면 효과+범용 코스트 지불이다(소비형 비용은 각 캐릭터 속성만 — 의도된 비대칭).

### 7.3 노드별 보상 주기

| 노드 | 보상 |
|---|---|
| 일반 전투 | 재화 소량 + 동전 후보 중 1개 |
| 엘리트 | 재화 중량 + 동전 후보 중 1개 + 스킬 1개 |
| 보물 | 재화 대량 + 패시브 1개 |
| 보스 | 재화 대량 + 동전 후보 중 1개 + 패시브 3개 중 1개 |

일반 전투마다 스킬 3택을 주지 않는다. 캐릭터 전용 2개+공용 1개의 구성은 별도 3택 보상이나 이벤트에 적용하고, 엘리트의 단일 스킬은 현재 캐릭터에게 유효한 전체 후보에서 뽑는다.

### 7.4 상점

- 스킬 5개: 캐릭터 전용 3개 + 공용 무속성 2개를 우선 구성하고, 후보 부족 시 유효한 나머지 풀로 채운다. 각 스킬 카드는 전투 카드와 동일한 효과 행·키워드 툴팁을 표시한다(포인터·키보드 포커스·터치 모두).
- 동전: §7.2의 전속성 가중 후보를 판매한다. 가격은 기본 25 / 대표 속성 50 / 비대표 속성 70.
- 패시브 구매와 동전 영구 제거를 제공한다.
- 구매한 스킬은 빈 슬롯에 장착하거나 기존 스킬과 교체한다.

## 8. 런 진행

### 8.1 막 구조

- 한 런은 최대 3막이다.
- 막마다 10개의 노드를 방문한다.
- 1~8번째는 분기형 후보, 9번째는 휴식, 10번째는 보스다.
- 1막 첫 방문은 일반 전투로 고정하고 첫 두 방문 후보에서는 엘리트를 제외한다.
- 이전 노드로 돌아갈 수 없다.
- 1·2막 보스 처치 후 다음 막 진입 전에 체력을 전부 회복한다.

### 8.2 확률형 노드

| 노드 | 가중치 |
|---|---:|
| 일반 전투 | 50% |
| 엘리트 | 10% |
| 보물 | 2% |
| 상점 | 15% |
| 휴식 | 3% |
| 이벤트 | 20% |

고정 휴식과 보스는 확률 계산에서 제외한다.

### 8.3 휴식과 이벤트

- 휴식에서는 최대 체력의 30% 회복 또는 패시브를 제외한 스킬 1개 강화를 선택한다.
- 이벤트는 리스크와 리턴을 함께 제공한다. 체력·재화·동전·스킬을 지불하거나 전투를 거쳐 보상을 얻을 수 있다.

## 9. 캐릭터 로스터

`exclusiveTo`는 저장 호환을 위해 안정적인 내부 ID를 사용한다. 표시명과 내부 ID는 다를 수 있다.

| 표시명 | 내부 ID | 대표 속성 | 핵심 메커니즘 |
|---|---|---|---|
| 화염 격투가 | `warrior` | 화염 | 화상 누적·화상 비례 마무리, 과열(즉시+예약 진입) |
| 번개 결투사 | `sorcerer` | 전기 | 스택형 르미즈(최대 3), 감전 |
| 냉기 도적 | `frost-knight` | 냉기 | 이중 주머니, 보존, 지정 드로우, 동상 |
| 마도기사 | `arcanist` | 마나 | 소환 장비, 병기 출력, 갑주 반향 |
| 혈액 마검사 | `blood-spellblade` | 혈액 | 체력 지불, 흡혈, 혈마검 투자·런 성장 |

수호자(`guardian`)는 P13에서 로스터에서 삭제되었다. 구저장은 크래시 없이 `retired-character`로 분류·격리 보존되며 새 런 시작을 안내한다(런 저장 v9).

### 9.1 공용 기본기

- **공격**(ID `slash`): 코스트 1, 반복, 성공면 앞면. 성공 0개는 효과 없음, 1개는 피해 4, 강화 시 피해 5
- **방어**(ID `guard`): 코스트 1, 반복, 성공면 뒷면. 성공 0개는 효과 없음, 1개는 방어 4, 강화 시 방어 5

기본기 표시명은 모든 캐릭터에서 **공격/방어**로 통일한다(P13). 화염 격투가는 같은 수치 계약의 전용 기본기(ID `jab`/`fist-guard`)를 쓰지만 표시명은 동일하게 공격/방어다. 내부 ID는 저장 호환을 위해 불변. 고유 스킬이 있는 캐릭터는 교체·제거 규칙을 별도로 명시한다.

### 9.2 화염 격투가

- 시작 전용: 공격(`jab`), 방어(`fist-guard`), 불꽃 스트레이트, 불씨권
- 전투 시작 특성 불씨 주머니가 임시 화염 동전 1개를 뽑을 더미에 추가한다.
- 화상 빌드: 화염 동전 생성 → 화상 누적 → 화상 비례 마무리
- 과열 빌드(P13 개정): 진입 경로 3종 — 내면의 열정(즉시), 화격권(비과열 상태에서 화염 동전 사용 시 **다음 턴 과열 예약**; 과열 중이면 강화 피해로 소비하고 새 예약 없음), 패시브 잔열 축적(전투당 1회 최후 진입 수단). 예약은 다음 플레이어 턴 시작에 활성화되고 중첩되지 않는다.
- 안티터틀(P13): 잿불 갑주는 방어 4+뒷면 +2(동전당 방어 제거, 생성 동전은 버림 더미로), 방패 숙련은 턴당 첫 방어 스킬만 +1, 빈틈없는 대비는 공격 스킬을 사용한 턴에만 턴 종료 방어 1.
- 소비형 스킬은 실제 화염 동전만 소비한다.

### 9.3 번개 결투사

- 르미즈는 스택형 전투 자원이다(최대 3, 플레이어 턴 시작 +1, 턴 종료 소멸). 공격 태그 원본 플립 스킬이 첫 코인 플립 전에 스택 1을 확정 소비하고(뒷면도 소비), 첫 코인이 앞면이면 원본 해결 후 같은 스킬을 가상 사본으로 1회 반복한다.
- 반복은 피해·방어·상태·속성 면 효과만 정상 해결하고, 드로우/생성/회수/덱 이동/스택 생성은 원본 전용이다(반복 트리거로 명시된 효과·패시브만 예외). 반복은 다시 반복하지 않는다.
- 레두블망·아타크 콩포제가 스택을 생성·반환해 연격을 잇고, 감전으로 적이 받는 피해를 높인다.
- 전용 스킬과 패시브는 `sorcerer` 풀에만 등장한다.

### 9.4 냉기 도적

- 고유 특성 이중 주머니로 턴 종료 시 동전 1개를 직접 보존한다.
- 보존 동전 연계 빌드와 냉기 동전 다량 소비 빌드를 사용한다.
- 냉기로 취급되는 기본 동전은 명시한 플립 상호작용에만 유효하며 냉기 소비 비용이 될 수 없다.

### 9.5 마도기사

- 마나 검과 마나 방패는 턴 종료에 자동 행동하고 소환 수치가 0이면 소멸한다.
- 병기 출력은 전투 동안 장비의 피해·방어 수치를 높인다.
- 갑주 반향 빌드(P13 개정): 적 턴에 방어로 **실제 흡수한 피해**만 반향이 된다 — 기본 반향 = min(흡수, 6), 흡수 1 이상일 때만 예열(갑주 축압)·정밀 방어(마력 증폭막) 보너스, 최대 12. 반향은 다음 플레이어 턴 동안 유지되고 미사용분은 소멸한다. 반향 증폭은 턴당 1회만 피해에 더해진다(갑주 강타/마도 갑주 해방 중 하나). 어떤 스킬도 현재 방어를 직접 소모하거나 피해로 변환하지 않는다 — 과방어는 더 안전할 뿐 피해를 더하지 않는다.
- 마나 병기 빌드는 장비 확보·출력 성장·광역화·복제·유지로 고점을 만든다.

### 9.6 혈액 마검사

- 혈액 동전은 플립하면 회복 또는 방어를 제공하고, 소비하면 면 효과가 없다.
- 체력 지불은 방어로 막을 수 없고 플레이어를 1 미만으로 만들 수 없다.
- 흡혈은 스킬이 실제 피해를 1 이상 줬을 때 명시한 수치만큼 한 번 회복한다.
- 혈마검 투자량은 런 동안 유지되며 단계 0~5와 혈마검 위력을 결정한다.
- 흡혈 순환 빌드는 혈액 동전을 현재 생존에 사용하고, 혈마검 각성 빌드는 혈액 동전을 장기 성장에 투자한다.
- 세부 단계·스킬·패시브 계약은 `PRD/P12_BLOOD_SPELLBLADE_DESIGN_SYNC.md`를 근거로 구현한다.

## 10. 몬스터와 조우

몬스터는 피해/체력 인플레가 아니라 전투 퍼즐을 만든다(P13). 주 역할 1개 + 기믹 태그를 가지며, 강한 효과는 예고되고 현실적 카운터가 2개 이상 있다.

- 주요 행동(준비형)은 해결 전에 카운트다운·대상·수치·취소/약화 조건을 표시한다.
- 하드 카운터·회피 불가 랜덤 셧다운 금지. 랜덤 대상/결과는 준비 시점에 공개한다.
- 일반 조우의 핵심 위협은 1~2개. 교란 축 3개 중첩 금지.
- 일반 몬스터: 패시브 0~1개 / 엘리트: 1개 권장 / 보스: 1~2개와 페이즈 가능. `enemyTurnStart` 패시브는 자기 대상 한정.
- 현재 구현 기믹(배치 A): 예고 준비(windup)·준비 중 취약·플레이어 HP 조건 분기·HP 분기점 페이즈·성장 스택(완전 방어 시 상실)·아군 치유(대상 사망 시 실패). 20종 정본 카탈로그와 배치 B~F 진입 조건은 `PRD/P13_REVISION_DESIGN_SYNC.md` §6이 정본이며, 구현된 것은 배치 A 6종뿐이다.

적 의도에는 가능한 한 피해량, 횟수, 방어·상태 효과를 표시한다.

## 11. UX·튜토리얼

### 11.1 전투 화면 필수 정보

- 손패, 동전 속성과 보존 여부
- 뽑을 더미·버림 더미·소비 영역의 수와 구성
- 스킬 코스트·쿨다운·일회성·대상
- 카드별 장전 수, 실행 대기 순번, 쿨다운·사용 완료·선택·불가 사유를 함께 표시하는 단일 하단 행동 바
- 예약 스킬의 순서와 대기·실행·완료·확인 필요 상태를 보여주는 실행 레일
- 현재 사용·장전·실행 대기·버림 예정 동전 수. 버림 예정은 현재 영역 기준 추정이며 이후 스킬의 드로우·생성·보존 효과로 달라질 수 있다.
- 현재 대상과 적 의도
- 플레이어·적 상태이상과 패시브
- 플립 결과와 효과 해결 순서
- 최근 해결 결과를 보존하는 전투 기록과 상시 접근 가능한 전투 도움말

### 11.2 점진 튜토리얼 순서

1. 턴 시작 3개 드로우와 코스트만큼 장전
2. 완전 장전된 스킬은 카드 하단 `스킬 사용`으로 수동 실행하거나, 순서를 확인한 뒤 턴 종료 경고·자동 모드로 실행
3. 사용·미사용 동전의 버림과 더미 재구성
4. 쿨다운과 일회성
5. 속성 동전의 양면 효과
6. 스킬 대상에 따른 속성 효과 대상
7. 코인 보존
8. 플립 없는 속성 동전 소비
9. 첫 보상, 상점, 휴식은 해당 화면에서 별도 문맥 팁으로 안내

처음 필요한 상황에는 한 항목씩 점진 안내를 보여준다. 전투 중 `?` 도움말에서는 장전, 실행 순서, 자동 턴 종료, 대상, 버림·보존, 쿨다운과 상태를 언제든 다시 확인할 수 있다.

모바일의 카드와 동전은 크기를 과도하게 축소하지 않고 자체 가로 스크롤·스냅 레일로 제공한다. 다음 항목 일부를 노출해 스와이프 가능성을 알리고, 동전 레일은 이전·다음 버튼과 현재 위치/전체 수를 함께 제공한다.

장전한 동전은 발동 전까지 회수하거나 다른 합법 슬롯으로 옮길 수 있고, 이미 장전된 두 동전을 서로 드래그해 교환할 수 있다. `추천 장전`은 왼쪽부터 합법 배치 제안만 미리 보여준 뒤 확인을 받으며, 스킬 발동이나 대상·선택 효과를 대신 결정하지 않는다.

전투 설정은 동전 던지기 `기본`·`빠르게`·`즉시`, 화면 흔들림, 피해 숫자 크기, 도움말 크기, 고대비, 배경 효과 줄이기, 모션 줄이기, 소리를 제공하고 로컬에 저장한다. `즉시`와 모션 줄이기는 플립 애니메이션 없이 결과를 표시하며, 운영체제의 `prefers-reduced-motion`은 로컬 설정보다 우선한다.

### 12.3 저장 계약

- 런 경계 상태만 저장하고 전투는 시드와 재도전 번호로 재구성한다.
- 저장 스키마 변경은 버전 증가, 순차 마이그레이션, 검증, 왕복 테스트를 함께 제출한다.
- 주 저장과 백업을 함께 쓰고 손상 원문은 격리한다.
- 제거된 콘텐츠 ID가 있는 구저장은 명시적 매핑으로 새 ID에 마이그레이션하거나, 이어갈 수 없는 런(예: 삭제된 캐릭터)은 크래시 없이 명시 상태(`retired-character`)로 분류해 원문을 격리 보존하고 사용자에게 안내한다.
- 혈마검처럼 런 동안 성장하는 캐릭터 자원은 전투 임시 상태가 아니라 런 저장에 포함한다.

## 13. 검증과 성공 기준

### 자동 검증

- 콘텐츠·타입·린트·단위·속성 기반 테스트
- 같은 시드와 명령열의 결정론 재현
- 전투와 런 불변식
- 보상 풀 경계와 저장 마이그레이션
- UI 빌드, 접근성, 성능·번들 예산, 모바일 플레이테스트

### 사람 검증

- 동전을 어디에 배치할지 실제로 고민하는가
- 앞·뒷면이 모두 선택 가치가 있는가
- 고코스트·소비형 스킬을 준비할 수단이 충분한가
- 캐릭터마다 보상과 전투 리듬이 분명히 다른가
- 공용 스킬이 빌드를 대체하지 않고 약점을 보완하는가

밸런스와 손맛은 자동 검증 통과만으로 확정하지 않는다. `PRD/PLAYTEST_KIT.md`의 사람 표본을 통과하기 전까지 `balance-provisional`, `experience-unverified`로 유지한다.

## 14. 현재 후속 과제

- 사람 N≥5 플레이테스트와 캐릭터별 빌드 선택률 검증

# 콘텐츠 기획 가이드 v2.2 — P10 현재 규칙

> 마지막 동기화: 2026-07-13 · PRD v1.3 · 콘텐츠 `1.4.0-p10`
>
> 신규 코인·스킬·캐릭터·몬스터·패시브를 작성할 때 사용하는 **활성 가이드**다. 제품 규칙은 [`PRD.md`](./PRD.md)와 최신 [`../PRD/P10_CHARACTER_DESIGN_SYNC.md`](../PRD/P10_CHARACTER_DESIGN_SYNC.md)가 정본이며, 기반 전투 규칙은 P7·P9 결정 로그에 남긴다. 실제 스키마와 검증은 `packages/core/src/content-types.ts`, 현재 수치는 `packages/content/src/index.ts`가 정본이다.

과거 가이드의 **턴당 스킬 3회**, **스킬별 턴당 1회**, **6슬롯**, **단면 속성 코인**, **전사 4스킬 구구성**은 폐기됐다. 역사적 근거가 필요하면 [`implementation-plan.md`](./implementation-plan.md)를 보되 신규 콘텐츠 규칙으로 사용하지 않는다.

## 0. 공통 전제

- 매 턴 기본 드로우는 5개다. 드로우 보정 후 총량은 0~8, 손 상한은 10이다.
- 전역 스킬 사용 횟수 제한은 없다. 코인과 가용 스킬이 남으면 계속 행동한다.
- 장착 슬롯은 8칸이며 시작 스킬은 4개다. 빈 슬롯은 `null`이다.
- 스킬은 장전형(`flip`) 또는 소비형(`consume`)이다.
- 기본 효과는 항상 존재해야 한다. 플립은 실패/성공이 아니라 저점과 고점을 만든다.
- 사용하지 않은 손패 코인은 턴 종료 시 버림 더미로 간다. 턴 간 이월하지 않는다.
- 생성 코인은 항상 임시다. 영구 코인 변경은 런 보상·상점·이벤트에서만 수행한다.
- 모든 수치는 사람 데이터 전까지 `balance-provisional`이다.

## 1. 행동 모델과 쿨다운

### 1.1 쿨다운 값

`cooldown?: 0 | 1 | 2 | 3 | 4`이며 미지정 기본값은 1이다.

| 값 | 사용 감각 | 권장 역할 |
|---|---|---|
| 0 | 같은 턴 반복 가능 | 공용 기본 공격·방어처럼 코인을 소진하는 바닥 행동 |
| 1 | 다음 턴 바로 다시 사용 | 일반적인 주력기 |
| 2 | 한 턴 쉬고 재사용 | 강한 엔진·광역·고효율 행동 |
| 3 | 두 턴 쉬고 재사용 | 빌드 전환, 강한 셋업, 고효율 지원 |
| 4 | 세 턴 쉬고 재사용 | 강화로 일회성이 제거된 희귀 엔진 |

`oncePerCombat: true`는 전투당 1회 메커니즘이다. 전투당 1회 스킬에는 `cooldown >= 1`을 함께 선언하지 않는다. 강화로 전투당 1회가 제거되면 기본 쿨다운 1이 적용된다.

### 1.2 반복 스킬 가드레일

반복 스킬은 손 코인이 자연 상한이므로 별도 사용 횟수 카운터를 만들지 않는다. 다만 다음을 지킨다.

- 반복 기본기는 유용하되 유료·고급 스킬의 효율을 지배하지 않아야 한다.
- 반복 행동에 무료 드로우나 무료 코인 생성처럼 순환을 늘리는 효과를 넣지 않는다.
- 현재 기준 기본기는 비용 1, 기본 4 + 유리한 면 3(`any`), 기대값 5.5다.

## 2. 코인 설계

### 2.1 스키마

```ts
interface CoinDef {
  id: CoinDefId;
  element: Element | null;
  procs?: {
    heads?: EffectAtom[];
    tails?: EffectAtom[];
  };
}
```

- 기본 코인은 `element: null`이고 `procs`를 가질 수 없다.
- 속성 코인은 앞면과 뒷면에 각각 1개 이상의 효과를 선언해야 한다.
- 코인 면 효과에는 `damage`, `block`, `heal`, `applyStatus`만 사용한다.
- 소비형 스킬은 플립하지 않으므로 코인 면 효과가 발동하지 않는다.

### 2.2 현재 양면 기준

| 속성 | 앞면 | 뒷면 | 역할 |
|---|---|---|---|
| 화염 | 화상 1 | 피해 1 | 누적 피해·공격 보조 |
| 마나 | 방어 1 | 방어 2 | 양면 안정 방어 |
| 냉기 | 동상 1턴 | 방어 1 | 공격 약화·생존 |
| 전기 | 감전 1턴 | 피해 1 | 피해 증폭·공격 보조 |
| 혈액 | 회복 1 | 방어 1 | 유지력 |

새 속성 코인을 추가할 때는 양면 모두 단독 가치가 있어야 한다. 한쪽이 사실상 꽝이거나 다른 속성의 완전 상위 호환이 되면 안 된다.

### 2.3 대상 규칙

- 공격형 코인 효과는 단일 대상 스킬이면 그 적에게 적용한다.
- 전체 대상 스킬이면 모든 살아 있는 적 각각에 적용한다.
- 자기·무대상 장전 스킬에 공격형 속성 코인이 들어가면 살아 있는 적 선택을 요구한다.
- 방어·회복 같은 우호형 효과는 스킬 대상과 무관하게 플레이어에게 적용한다.

### 2.4 코인 템플릿

```ts
const newCoin = {
  id: coin('new-element'),
  element: 'new-element',
  procs: {
    heads: [{ kind: 'applyStatus', status: '...', stacks: 1, to: 'target' }],
    tails: [{ kind: 'block', amount: 1 }]
  }
} satisfies CoinDef;
```

검토 질문:

- 앞면과 뒷면이 모두 유의미한가?
- 비시그니처 캐릭터가 보상으로 집어도 단독 가치가 있는가?
- 공격형·우호형 대상 규칙이 툴팁 한 줄로 설명 가능한가?
- 소비 연료로 쓸 때 면 효과가 발동하지 않아도 선택 갈등이 생기는가?

## 3. 스킬 설계

### 3.1 공통 스키마

```ts
interface SkillDefBase {
  id: SkillId;
  name: string;
  rarity: 'common' | 'advanced' | 'rare';
  tags: readonly ('attack' | 'defense' | 'utility' | 'ultimate')[];
  targetType: 'single-enemy' | 'all-enemies' | 'self' | 'none';
  cooldown?: 0 | 1 | 2 | 3;
  oncePerCombat?: boolean;
  overheatBonus?: EffectAtom[];
  exclusiveTo?: CharacterId;
  upgrade?: SkillUpgradeDef;
}
```

공격 태그가 있는 스킬은 `single-enemy` 또는 `all-enemies` 대상이어야 한다. 자기 셋업 스킬은 `utility`로 분류한다.

### 3.2 장전형

```ts
interface FlipSkillDef extends SkillDefBase {
  type: 'flip';
  cost: number;
  base: EffectAtom[];
  heads?: { mode: 'any' | 'per'; effects: EffectAtom[] };
  tails?: { mode: 'any' | 'per'; effects: EffectAtom[] };
  elementFaces?: {
    element: Element;
    face: Face;
    effects: EffectAtom[];
  }[];
}
```

- 비용은 1~5다.
- 비용 5는 희귀이면서 `oncePerCombat` 또는 `ultimate`여야 한다.
- `any`: 해당 면이 하나 이상이면 한 번 발동한다.
- `per`: 해당 면 개수만큼 발동한다.
- `elementFaces`는 특정 속성 코인의 특정 면에 추가 보너스를 줄 때만 쓴다.
- 스킬 본체 효과가 끝난 뒤 코인 자체의 면 효과가 발동한다.

`any`와 `per`를 카드 문구에서 추측하게 하지 말고 데이터에 명시한다.

```ts
heads: { mode: 'any', effects: [{ kind: 'damage', amount: 4 }] }
// 앞면이 1개든 3개든 +4 한 번

heads: { mode: 'per', effects: [{ kind: 'damage', amount: 3 }] }
// 앞면 3개면 +9
```

### 3.3 소비형

```ts
interface ConsumeSkillDef extends SkillDefBase {
  type: 'consume';
  consume: { element: Element; count: number };
  effects: EffectAtom[];
}
```

- 소비 개수는 1~3이다.
- 손패에서 요구 속성을 만족하는 서로 다른 코인을 선택한다.
- 임시 부여 속성(`grants`)도 연료 조건으로 인정한다.
- 선택 코인은 소모 영역으로 가며 이번 전투 리셔플에서 제외된다.
- 플립이 없으므로 앞·뒷면 보너스와 속성 코인 면 효과가 발동하지 않는다.
- 소비형의 가치는 확정성, 과열 진입, 상태 전환, 큰 빌드 보상에서 나온다.

### 3.4 4+비용 표준

비용 4 이상은 한 턴의 대부분을 투자한다. 다음을 모두 만족시킨다.

1. 기본 효과만으로도 투자 가치가 분명하다.
2. 드로우, 다음 턴 드로우, 임시 코인, 의미 있는 상태, 빌드 전환 중 하나 이상의 추가 리턴이 있다.
3. 앞면이 나와야만 쓸 만한 구조로 만들지 않는다.
4. 현재 손 5개와 드로우·생성 수단으로 실제 구성 가능한지 확인한다.

### 3.5 지원 스킬 표준

버프·소환·쿨다운 조작처럼 즉시 피해가 없는 지원 스킬은 다음 중 하나 이상을 동반한다.

- 즉시 드로우
- 임시 코인
- 다음 턴 드로우
- 즉시 방어·회복
- 즉시 소환 행동처럼 눈에 보이는 리턴

셋업만 하고 그 턴에 아무 변화도 느끼지 못하는 스킬은 피한다.

### 3.6 임시 코인 강도

같은 수량이면 즉시성이 높을수록 강하다.

```text
손 생성 > 뽑을 더미 삽입 > 버림 더미 생성
```

- 손 생성: 바로 사용할 수 있으므로 수량을 줄이거나 쿨다운을 높인다.
- 뽑을 더미: 무작위 위치에 삽입한다. 맨 위 확정 삽입은 사용하지 않는다.
- 버림 더미: 리셔플 이후 가치라 가장 저렴하다.

### 3.7 과열

과열은 화염 격투가의 보조 아키타입이다.

- `enterOverheat`로 진입한다.
- 불리언이므로 중첩되지 않고 턴을 넘어 유지된다.
- `overheatBonus`가 있는 스킬이 성공적으로 해결될 때만 소비한다.
- 비과열 스킬은 과열을 건드리지 않는다.
- 과열 보너스가 피해만이라면 기본 피해와 한 타격으로 합쳐진다.

과열 진입 스킬은 `requiredElement: 'fire'`로 화염 동전을 장전해 플립한다. 소비형과 달리 면·속성 효과가 정상 발동한다.

### 3.8 스킬 템플릿

```text
[스킬명]
캐릭터 전용 여부:
등급: common / advanced / rare
타입: flip / consume
태그:
대상:
비용 또는 소비 조건:
쿨다운: 0 / 1 / 2 / 3 / 전투당 1회
기본 효과:
앞면 효과와 모드:
뒷면 효과와 모드:
특정 속성·면 효과:
과열 보너스:
드로우·임시 코인·상태 리턴:
강화:
빌드 역할:
저점에서의 가치:
고점에서의 가치:
다른 스킬을 대체하지 않는 이유:
```

### 3.9 현재 기준점

아래는 법칙이 아니라 회귀 비교용 기준점이다.

| 스킬 | 비용·제한 | 현재 효과 | 용도 |
|---|---|---|---|
| 베기/잽 | 1, 반복 | 피해 4, 앞면 +3 | 공격 바닥 행동 |
| 방어/주먹막기 | 1, 반복 | 방어 4, 뒷면 +3 | 방어 바닥 행동 |
| 불타는 일격 | 2, 기본 쿨다운 | 피해 8, 임시 화염→버림, 앞면당 +3 | 생성형 주력기 |
| 내면의 열정 | 화염 1 장전, 쿨3 | 과열 진입, 앞면 피해 5 / 뒷면 방어 3 | 과열 셋업 |
| 화격권 | 2, 쿨1 | 임시 화염→뽑기, 피해 10, 과열 +4 | 과열 소비 주력기 |
| 낙성권 | 4, 쿨2 | 피해 16, 화상 3, 임시 화염→손 | 4비용 표준 예시 |

신규 수치는 반드시 현재 콘텐츠와 시뮬레이션을 함께 비교하고, 사람 테스트 전에는 확정 수치로 표현하지 않는다.

## 4. 효과 원자 선택

| 목적 | 원자 |
|---|---|
| 직접 수치 | `damage`, `block`, `selfDamage`, `heal` |
| 상태 | `applyStatus` |
| 순환 | `draw`, `nextTurnDraw`, `addCoin` |
| 속성 취급 | `grantElement` |
| 행동 주기 | `reduceCooldown` |
| 과열 | `enterOverheat`, 스킬의 `overheatBonus` |
| 턴 훅 | `addTurnTrigger` |
| 빌드 참조 피해 | `damagePerTargetBurn`, `damagePerBlock` |
| 소환 | `summonEquipment`, `commandChosenSummon`, `empowerSummons` |

새 원자를 추가하기 전에 기존 원자 조합으로 표현할 수 있는지 확인한다. 새 원자가 필요하면 다음을 같은 PR에서 처리한다.

- `EffectAtom` 타입
- 인터프리터 `applyEffectAtom`
- 이벤트와 UI 설명
- 프리뷰 집계 축
- 콘텐츠 검증
- 단위·골든·시뮬 회귀 테스트
- [`current-implementation.md`](./current-implementation.md)

## 5. 캐릭터 설계

### 5.1 현재 기본 형상

- 장착 가능 슬롯: 8
- 시작 스킬: 4
- 시작 주머니 기준: 기본 8 + 대표 속성 2
- 고유 특성: 1개
- 특성 훅: `combatStart` 또는 `turnStart`
- 특성 효과는 자동 발동했을 때 대상 선택이 필요 없는 안전한 효과로 구성한다.

현재 캐릭터는 화염 격투가, 수호자, 술사, 냉기 기사, 마도기사다. 캐릭터 전용 스킬은 `exclusiveTo`로 공용 보상 풀과 경계를 명시한다.

### 5.2 캐릭터 템플릿

```text
[캐릭터명]
ID:
대표 속성:
최대 HP:
시작 주머니: 기본 8 + 대표 속성 2를 바꾸는 이유
시작 스킬 4개:
  - 반복 기본 공격
  - 반복 기본 방어
  - 캐릭터 핵심 행동 1
  - 캐릭터 핵심 행동 2
고유 특성:
특성 훅: combatStart / turnStart
특성 효과:
주요 빌드 2개 이상:
공용·비시그니처 코인을 집을 이유:
약점과 방어 수단:
8슬롯 완성 시 교체 우선순위:
```

시작 스킬은 캐릭터 문법을 보여줘야 하지만 완성 빌드를 시작부터 제공해서는 안 된다. 빈 4슬롯이 보상과 상점의 성장 공간이다.

## 6. 몬스터 설계

### 6.1 의도

적은 `EnemyIntent[]`를 순서대로 순환하며 현재 행동과 수치를 항상 공개한다.

```ts
{
  id: 'intent-id',
  actions: [
    { kind: 'attack', damage: 8 },
    { kind: 'block', amount: 4 }
  ]
}
```

지원 행동은 `nextDrawPenalty`, `applyStatus`, `heal`, `buffNextAttack`을 사용할 수 있다. 다단 공격은 `hits`로 표현한다.

### 6.2 패시브

적 패시브는 의도로 예고되지 않으므로 자기 대상 행동만 허용한다.

- 허용: `heal`, `block`, `buffNextAttack`
- 금지: 플레이어 피해, 상태이상, 드로우 감소 등
- 자동 조건은 최대 1개로 유지한다.
- 전장에 이름과 설명을 항상 노출한다.

### 6.3 몬스터 템플릿

```text
[몬스터명]
등급: 일반 / 엘리트 / 보스
역할:
최대 HP:
등장 막·방문 범위:
동시 등장 조합:
의도 루프:
패시브(선택):
강한 턴의 사전 신호:
플레이어 대응법:
반복 기본 방어로 무한 정체되지 않는 장치:
```

HP·피해 기준은 런 길이, 막별 적 배율, 반복 방어의 정체 가능성과 함께 시뮬레이션한다. 기존 표를 고정 밴드로 간주하지 않는다.

## 7. 패시브와 소환 장비

### 7.1 획득 패시브

획득 패시브는 시작 특성과 구분한다.

- 중복 획득하지 않는다.
- `combatStart` 또는 `turnStart` 훅을 사용한다.
- 자동 플레이어 문맥에서 안전한 원자만 사용한다.
- 현재 허용 범위는 방어, 임시 코인, 턴 트리거, 소환·강화, 속성 부여 중심이다.
- 가격은 양의 정수이며 상점·보스·보물 공급 경로를 고려한다.

### 7.2 소환 장비

소환 장비는 유닛이 아니라 슬롯형 자동 행동이다.

- 최대 3개
- 소환 순서대로 플레이어 턴 종료 시 행동
- `strike`는 첫 살아 있는 적에게 피해
- `ward`는 플레이어에게 방어
- 행동 뒤 지속시간 감소, 0이면 소멸
- 적 공격 대상이 되지 않는다.

## 8. 스킬 강화

현재 강화는 스킬당 1개이며 런에서 슬롯 단위로 한 번 적용한다.

허용 패치:

- `baseAmount`
- `addFaceEffect`
- `addCoinOnUse`
- `costDelta`
- `removeOncePerCombat`

강화는 기존 역할을 선명하게 해야 한다. 별도 스킬처럼 문법을 바꾸거나 부작용을 삭제해 선택 의미를 없애지 않는다. 스킬 교체 시 해당 슬롯의 강화는 초기화된다.

## 9. 성장·보상과의 연결

- 일반 전투: 코인 선택
- 엘리트: 코인 선택 + 스킬 제안
- 비최종 보스: 코인 선택 + 패시브 3택
- 상점: 코인·스킬·패시브 구매, 코인 제거
- 휴식: 최대 HP 30% 회복 또는 장착 스킬 강화
- 보물: 골드 + 패시브
- 이벤트: 전투·HP·골드·코인 중 명시적 위험을 지불

코인 보상은 대표 속성, 이미 보유한 속성, 기본 코인의 가중치를 사용한다. 새 코인을 추가하면 보상 풀 가중과 상점 가격, 저장 검증, UI 설명도 함께 확인한다.

## 10. 제출 전 체크리스트

### 코인

- [ ] 기본 코인이 아닌 경우 앞·뒷면 효과가 모두 존재한다.
- [ ] 코인 proc 허용 원자만 사용한다.
- [ ] 공격형·우호형 대상 규칙이 명확하다.
- [ ] 비시그니처 캐릭터도 집을 이유가 있다.

### 스킬

- [ ] 기본 효과가 존재하고 저점에서도 가치가 있다.
- [ ] 쿨다운과 전투당 1회가 충돌하지 않는다.
- [ ] 비용 1~5, 소비 1~3 규칙을 지킨다.
- [ ] 비용 5라면 희귀 + 전투당 1회 또는 궁극이다.
- [ ] 공격 태그는 적 대상 타입을 사용한다.
- [ ] 지원 스킬에 즉시 체감 리턴이 있다.
- [ ] 4+비용에 강한 기본치와 추가 리턴이 있다.
- [ ] 임시 코인 목적지의 강도를 반영했다.
- [ ] 강화가 유효하고 기존 역할을 보존한다.

### 캐릭터·적·패시브

- [ ] 시작 스킬은 4개이며 8슬롯 성장 공간이 있다.
- [ ] 캐릭터 전용 경계는 `exclusiveTo`로 표현한다.
- [ ] 적 의도는 수치까지 공개 가능하다.
- [ ] 적 패시브는 자기 대상 효과만 사용한다.
- [ ] 자동 훅에 선택이 필요한 효과가 없다.

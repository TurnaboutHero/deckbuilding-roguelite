# P5 제품화 설계 결정 (2026-07-12) — 무인(humanless) 항목 정본

Phase 5 계획(`.omx/plans/phase3-plus-engineering-autonomy.md` §P5)을 무인 실행 가능한
라운드로 분해한다. 사람 취향/손맛 게이트는 전부 유보(experience-unverified 유지) —
여기 항목은 공학 판정만으로 닫을 수 있는 것들이다. 최종 상태 표기는
**engineering-complete / experience-unverified**.

## P5.0 릴리스 기준·예산 (거버넌스)

- **번들 예산**: dist ≤ 2.6MiB (P4.6 마감 2.40MiB — 잔여 여유 ~205KiB, 사운드는
  별도 예산 아래 지연 로드). 초과 시 릴리스 차단.
- **성능 예산(provisional)**: 초기 로드 JS 파싱 후 첫 상호작용 가능까지 로컬 프리뷰
  기준 < 3s, 전투 커맨드 응답 < 100ms (계측 스크립트로 측정·기록, CI 비차단 report).
- **사운드 provenance 규칙**: 생성(도구·세션 기록)/원작(제작 기록)/명시 라이선스만.
  `docs/audio-provenance.md`에 파일별 출처·라이선스 필수. 미기록 오디오 = 릴리스 차단.
- 라벨 규약 유지: engineering-safe / balance-provisional / experience-unverified.

## P5.1 모바일/터치

- 대상 뷰포트(provisional): 세로 모바일 390×844(iPhone 12 기준), 가로 844×390,
  태블릿 1024×768. 1024 미만에서 카드 레일·유닛 배치 재배열(세로 스택/스크롤 없는
  압축 레이아웃), 지도/상점/이벤트 화면 responsive.
- **터치 대체 경로**: 기존 마우스 경로 전부에 터치 등가 — 코인 장전은 탭-탭(드래그
  불요 경로 이미 존재), 대상 지정 탭, 툴팁은 탭-토글(hover 대체). 완료·취소 경로 필수.
- E2E: S28 모바일 뷰포트 스위트(부팅·전투 1턴·보상·상점·이벤트·갈림길 각 화면
  클리핑/겹침 검사 + 터치 이벤트 시뮬).

## P5.2 사운드 (생성 자산만)

- MVP 제외였던 사운드를 최소 셋으로: 플립 결과 2(앞/뒤), 타격 1, 승리/패배 2, 상점
  구매 1, 버튼 1 — 7개 내외 짧은 SFX. **생성 경로**: 코드 합성(Web Audio 오실레이터
  기반 결정론 생성 스크립트 → wav 아웃풋 커밋) — 외부 라이선스 리스크 0, provenance =
  생성 스크립트 자체. BGM은 릴리스 범위 외(후속, 사람 취향 게이트).
- 음소거 토글(로컬 저장, 기본 ON=소리 없음 — 자동재생 정책 회피), reduced-motion과
  독립. 오디오 로드 실패는 무음 폴백(게임 진행 차단 금지).

## P5.3 접근성

- 키보드 완주: 전투·보상·갈림길·상점·이벤트·재시작 전 경로 Tab/Enter/Escape 도달
  가능 검증(S29). 포커스 트랩·포커스 이동 규칙 명시.
- aria: 라이브 리전(결산 티켓·거부 사유), 모든 아이콘 버튼 라벨, 상태 칩 라벨(기존
  유지 검증). 대비: 텍스트 대비 자동 검사 스크립트(AA 기준, report — 픽셀 폰트 특성상
  차단 아님·미달 목록 기록).
- reduced-motion: 전 신규 화면(지도·상점·이벤트) 포함 재검증.

## P5.4 성능·복구

- 로딩 실패 복구: 청크/이미지 로드 실패 시 재시도 UI(전체 백지 금지), 저장 파싱 실패
  시 "손상 저장 — 새 런 시작" 명시 UX(조용한 삭제 금지). 저장 쓰기 실패(쿼터) 무시하되
  경고 배지.
- 성능 계측: 부팅 TTI·전투 커맨드 라운드트립·리플레이 100커맨드 시간 측정 스크립트
  (`sim:perf` — report-only).

## P5.5 릴리스 파이프라인

- CI에 추가(차단): 콘텐츠 lint(validateContentDb — 이미 테스트로 존재, 명시 스텝화),
  자산 provenance 검사 스크립트(스프라이트 run 디렉토리 필수 파일·킷 결과 ok 검증,
  오디오 provenance 문서 대조), 번들 예산 검사(≤2.6MiB).
- CI에 추가(비차단 report): sim:economy 소표본, 성능 계측.
- 태그 릴리스 워크플로(vX.Y.Z → Pages 배포 + 릴리스 노트 자동 생성).

## P5.6 최종 마감

- CONTENT_VERSION 1.0.0-rc 승격(콘텐츠 변경 없으면 라벨만 — 골든 재고정 불요 확인),
  README/PLAYTEST_KIT 갱신, PROGRESS 최종 표, **[FABLE_DONE] 보고**(마일스톤 표·증거·
  라이브 URL·사람 전용 미결 항목 목록).
- 사람 전용 유보 목록(명시): 손맛/재미 판정(M2·M4·M5·M6), 밸런스 확정(guardian 지배·
  turtle 우세·상점 가격), BGM/사운드 취향, 난이도 곡선.

## 라운드 실행 순서

P5.0(문서·예산 스크립트) → P5.1(모바일+S28) → P5.2(사운드) → P5.3(접근성+S29) →
P5.4(복구·계측) → P5.5(파이프라인) → P5.6(마감·[FABLE_DONE]). 각 라운드 전체 게이트
+ 커밋/푸시/CI/live 후 진행. Pumasi는 quota 가용 시 병렬 레인(모바일 CSS·사운드 합성),
검증은 독립 워커 또는 페이블 직접.

# 코인플립 로그라이크 — Phase 분리 계획

> 마일스톤 상세(태스크·완료 기준·의존 관계)는 [docs/implementation-plan.md §7](../docs/implementation-plan.md)의 M0~M6이 원본.
> 이 문서는 그것을 바이브코딩용 3-Phase로 접어 재구성한 것이다. Phase 경계마다 **사람이 플레이해 보는 게이트**가 있다.

---

## Phase 1: MVP 전투 검증 (M0~M4, 약 5~7주)

### 목표
브라우저에서 전사 vs 적 1종 전투를 완주하며 **"배치 후 플립" 문법이 재미있는지** 검증. MVP 스킬 6종 전부 동작.

### 기능
- [ ] 모노레포 + 시드 RNG(스트림 분리) + CI (M0)
- [ ] 헤드리스 전투 코어 — 리듀서/이벤트, 턴 상태기계, CLI 1판 완주, 퍼즈 1,000판 (M1)
- [ ] 전투 UI — PRD §15.1 레이아웃, 드래그 장전, **픽셀 플립 연출**, 예상 효과 프리뷰 (M2)
- [ ] **게이트 ①: 손맛 플레이테스트** — "플립 전에 궁금했나" (M2 종료)
- [ ] 화염 코인 + 화상 + 임시 코인 + 전사 고유 특성 "불씨 주머니" (M3)
- [ ] 소비형(점화 검술) + 취급 태그(화염 폭주) + 일회성 스킬 지원 (M4)
- [ ] **게이트 ②: 플립 vs 소비 갈등 관찰** (M4 종료)

### 데이터
CoinDef(basic/fire), SkillDef 6종, EnemyDef 1종(약탈자), CombatState 전체

### 인증
없음 (로컬 게임, 계정 없음 — 확정)

### "진짜 제품" 체크리스트 (게임 버전)
- [ ] 실제 게임 규칙 (목업 화면 X — 코어 리듀서가 판정)
- [ ] 코어에 `Math.random` 0개 — 시드로 같은 전투 재현 가능 (URL 시드 파라미터)
- [ ] 실제 배포 URL — 테스터가 링크만 받고 플레이 가능 (GitHub Pages 확정, base path 설정)
- [ ] 골든 테스트(스킬 8케이스) + 코인 총량 불변식 통과

## Phase 2: 런 루프 + 밸런스 (M5~M6, 약 3~4주)

### 전제 조건
Phase 1의 게이트 ①② 통과 (실패 시 루프 재설계가 우선)

### 목표
5전투 런을 완주하며 성장(코인 깎기/스킬 세팅)이 동기부여가 되는지 확인. 수치를 시뮬레이터로 튜닝.

### 기능
- [ ] 적 3종 (수문장·주술사 추가) + 런 상태(HP 이월) (M5)
- [ ] 보상 — 코인 3택1 + 제거 1회 + 스킬 2택1(슬롯 만석 시 교체) (M5)
- [ ] localStorage 저장 (전투 경계) + 런 결과 화면(시드 표시)
- [ ] 밸런스 시뮬레이터 — 봇 4종, §16.2 지표 리포트, 튜닝 라운드 (M6)
- [ ] **게이트 ③: 사람 플레이테스트 N≥5** — 정량 지표 판정 (M6)

### 추가 데이터
EnemyDef 2종, RunState/RunSave, 보상 풀(강타·화염 주입, 마나 코인)

### 통합 테스트
Phase 1 골든 테스트 전부 유지 + 런 전체 시드 재현 골든 추가

## Phase 3: 확장 (기간 미정 — 검증 결과에 따라)

### 전제 조건
Phase 2 지표가 목표 대역 진입, "더 하고 싶다"는 테스터 반응

### 목표
콘텐츠 폭 확장 — 빌드 다양성

### 기능 (우선순위 순)
- [ ] 마나 빌드 완성 + 수호자 캐릭터 (뒷면 활용 컨셉)
- [ ] 일회성 희귀 스킬 콘텐츠, 턴 버프 스킬(화염검·불의 심장 — 예약된 트리거 2종 구현)
- [ ] 동상/감전 (지속형 % 상태이상) + 냉기/전기 캐릭터
- [ ] 다중 적 전투 콘텐츠(엔진은 이미 지원), 보스, 상점/이벤트 노드
- [ ] 모바일 대응, 사운드

### 주의사항
- 2코인+ 스킬이나 손패 생성 스킬을 추가할 때마다 **폭주 콤보 상한 재계산** (docs 계획 §9.4)
- 영구 변환은 보상/이벤트 전용 — 스킬로 절대 제공 금지 (PRD §10)

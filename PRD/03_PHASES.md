# [역사 문서] 코인플립 로그라이크 — 초기 Phase 계획

> 최초 작성: 2026-07-10
>
> ⚠️ 이 문서는 M0~M6를 시작할 때 만든 단계 계획이다. 현재 백로그나 규칙 정본이 아니다. 이미 구현된 항목과 당시 제외 범위가 섞여 있으므로 신규 작업 범위는 [`../docs/README.md`](../docs/README.md), [`../docs/current-implementation.md`](../docs/current-implementation.md), 최신 결정 로그를 기준으로 정한다.

## 1. 이 문서의 현재 역할

이 문서는 다음 용도로만 보존한다.

- 헤드리스 코어 → 최소 UI → 속성·소비 → 런 → 시뮬레이션 순으로 진행한 이유
- 사람 손맛 게이트를 자동 테스트와 구분한 근거
- 초기 MVP가 어디서 시작해 현재 P7까지 확장됐는지 보는 개발 이력

아래 Phase의 체크박스를 현재 미완성 목록으로 해석하지 않는다.

## 2. 초기 Phase 1 — MVP 전투 검증 (M0~M4)

### 당시 목표

브라우저에서 전사와 적 1종의 전투를 완주하며 **장전 후 플립** 문법을 검증한다.

### 당시 범위

- pnpm 모노레포, 시드 RNG 스트림, CI
- 순수 리듀서와 이벤트 기반 헤드리스 전투 코어
- CLI 전투와 코인 총량 불변식 퍼즈
- React 전투 UI, 드래그 장전, 플립 연출, 예상 효과 프리뷰
- 화염 코인, 화상, 임시 코인
- 소비형 스킬, 속성 취급 태그, 전투당 1회

### 현재 상태

공학 구현과 자동 검증은 완료됐다. 사람에게서 확인해야 하는 손맛과 플립·소비 갈등은 통과로 간주하지 않고 `experience-unverified`로 유지한다. 관련 보류는 [`PHASE1_HOLDS.md`](./PHASE1_HOLDS.md)를 본다.

## 3. 초기 Phase 2 — 런 루프와 시뮬레이션 (M5~M6)

### 당시 목표

5전투 선형 런, 보상, 저장, 정책별 자동 플레이를 통해 성장 구조와 밸런스를 사전 검증한다.

### 당시 범위

- 적 3종과 HP 이월
- 코인·스킬 보상
- 전투 경계 localStorage 저장
- 런 결과와 시드 노출
- 다수 정책 봇, 대량 시뮬레이션, 결정론 비교

### 현재 상태

M5~M6 자동화는 완료됐고, 당시 5전투 구조는 이후 그래프·경제·콘텐츠 확장에서 대체됐다. 사람 N≥5 판정은 여전히 별도 게이트다. 당시 수치와 결과는 [`M6_BALANCE_REPORT.md`](./M6_BALANCE_REPORT.md)에 역사적 증거로 보존한다.

## 4. 초기 Phase 3 — 확장

### 당시 계획

- 마나 빌드와 수호자
- 턴 트리거 스킬
- 냉기·전기와 추가 캐릭터
- 다중 적, 보스, 상점, 이벤트
- 모바일과 사운드

### 이후 진행

2026-07-11 오너 오버라이드로 사람 게이트를 개발 선행 조건에서 분리하고, 공학 증거를 통과하는 확장 트랙을 진행했다. 현재 구현에는 다음이 포함된다.

- 5캐릭터와 화염·마나·냉기·전기·혈액 코인
- 다중 적, 엘리트, 막 보스
- 상점, 이벤트, 휴식, 보물, 패시브, 강화, 골드
- 3막 × 막당 10방문 그래프
- 모바일 플레이, 런타임 합성 SFX, 접근성·성능 게이트
- P7 쿨다운, 8슬롯, 양면 속성 코인, 과열, 드로우 표준

이는 재미 검증 완료를 뜻하지 않는다. 공학 완료와 경험 검증은 계속 분리한다.

## 5. 현재 작업 시작 절차

초기 문서 세트만 AI에 제공하던 아래 방식은 폐기한다.

```text
@PRD/01_PRD.md
@PRD/02_DATA_MODEL.md
@PRD/04_PROJECT_SPEC.md
@docs/implementation-plan.md
```

현재 작업은 최소한 다음 문서를 함께 읽는다.

```text
@docs/README.md
@docs/PRD.md
@PRD/P7_NEW_DESIGN_DECISIONS.md
@docs/current-implementation.md
@PRD/04_PROJECT_SPEC.md
@docs/content-design-guide.md   # 콘텐츠 작업일 때
```

작업 순서는 더 이상 M0~M6를 반복하지 않고, 변경 유형에 따라 다음처럼 진행한다.

1. 제품 규칙과 최신 결정 로그 확인
2. 현재 코드와 테스트에서 실제 계약 확인
3. 회귀 테스트를 먼저 추가하거나 갱신
4. 코어·콘텐츠·UI 중 필요한 최소 계층 수정
5. 현재 구현 문서와 콘텐츠 가이드 동기화
6. `pnpm release:verify`와 관련 브라우저 게이트 실행
7. 사람 체감이 필요한 결론은 `experience-unverified`로 남김

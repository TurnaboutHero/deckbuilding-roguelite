## 1. 제품 개요

### 한 줄 요약
카드가 아닌 **동전**으로 덱을 짜는 턴제 로그라이크 — 고정 스킬 셋(상수)에 매 턴 무작위 동전(변수)을 장전해 최적 효율을 찾는 퍼즐형 전투 게임.

### 해결하는 문제 (기획 의도)
족보식 코인 게임은 "좋은 조합을 뽑는 운"에 재미가 쏠린다. 이 게임은 **한정된 자원을 어디에 걸지 스스로 결정하고 결과를 감수하는 재미**를 목표로, 동전을 스킬에 직접 장전하는 방식을 택했다.

### 핵심 가치
- **스킬은 실패하지 않는다** — 기본 효과는 항상 발동, 앞/뒷면은 추가 효과일 뿐 (저점/고점 구조)
- 속성 동전 하나를 두고 "플립해서 고점을 노릴까, 소비형 스킬의 확정 연료로 쓸까" 갈등
- 전투 후 보상으로 코인 주머니의 확률 구조를 직접 깎는 성장

---

## 2. 사용자

- **누구**: StS류 덱빌딩 로그라이크 경험자 (MVP 플레이테스터 3~5명)
- **상황**: 데스크톱 브라우저, 마우스로 동전을 드래그해 장전
- **목표**: 전투 1판(4~7턴)에서 "어디에 걸지 고민되는" 긴장감을 느낀다

### 사용자 시나리오
1. 테스터가 URL을 열고 전사로 런을 시작한다
2. 매 턴 동전 5개를 뽑아 스킬 카드의 노란 소켓에 장전하고 플립한다
3. 전투 승리 후 코인/스킬 보상으로 주머니를 성장시켜 5전투 런을 완주한다

## 4. 사용자 흐름

```
런 시작(전사) → [전투: 드로우 5 → 장전/소비 → 플립 → 효과 → 턴 종료 ⟲] → 승리
→ 보상(코인/스킬/제거) → 다음 전투 … ×5 → 런 결과(시드 표시) → 재시작
```

상세: docs/PRD.md §5(전투 루프), docs/implementation-plan.md §5.1(턴 구조 확정판)

## 5. 성공 기준 (MVP)

- [ ] 플레이테스터 다수가 "플립 전에 결과가 궁금했다"고 답한다 (M2 손맛 게이트)
- [ ] "속성 코인을 플립할지 소비할지 고민됐다"는 반응 관찰 (M4 게이트)
- [ ] 1전투 평균 4~7턴, 완전 무효 행동 5% 이하 (사람 플레이 로그 기준 — docs 계획 §8.3)
- [ ] 시드 하나로 런 전체가 재현된다 (버그 리포트 가능)

## 6. 안 만드는 것 (Out of Scope — Phase 1)

- 계정/서버/클라우드 저장 — 이유: 로컬 localStorage로 충분, MVP는 재미 검증이 목적
- 상점·이벤트·보스·맵 — 이유: PRD §14.2가 명시적으로 제외
- 모바일 터치 대응 — 이유: 데스크톱 웹 우선 확정, 검증 후 대응
- 냉기/전기/혈액 속성, 전사 외 캐릭터 — 이유: Phase 3
- 사운드/BGM — 이유: 손맛 검증은 시각 연출 우선 (가정 원장 #3)

# D23 — v4.5 즉시 전투 롤백 및 메타 UI 확정안

> 확정일: 2026-07-22  
> 규칙 정본: Google Drive `코어 전투·캐릭터 통합 PRD v4.5 최종본`  
> 우선순위: 본 문서의 즉시 사용 정정 > v4.5의 예약·일괄 실행 문단 > 이전 D22 핫픽스

## 1. 변경 목적

예약 행동을 여러 개 만든 뒤 한 번에 확정하는 구조는 코인 플립의 즉각적인 손맛을 늦추고, 자동 배치와 자동 실행이 플레이어의 선택을 대신했다. 전투는 다시 **코인을 직접 고르고, 스킬에 배팅하고, 즉시 결과를 확인한 뒤 다음 행동을 선택하는 구조**로 변경한다.

## 2. 확정 전투 계약

플립형 스킬의 한 행동은 다음 순서로 끝까지 해결한다.

`스킬 선택 → 대상 선택 → 손패 코인 선택 → 코인 걸기 → 즉시 사용 → RNG 플립 → 고정 성공면 개수 판정 → 코인·스킬·상태 효과 해결 → 결과 확인`

- 행동 예약, 실행 순서 레일, 행동 확정 버튼, 자동 코인 배치, 자동 행동 실행을 제거한다.
- 한 행동의 결과를 확인한 뒤에만 다음 행동을 선택한다.
- 전역 행동 횟수 제한은 없다. 손패와 합법적인 스킬이 남아 있으면 계속 행동할 수 있다.
- 반복 스킬은 같은 턴에 여러 번 즉시 사용할 수 있다.
- 쿨타임, 턴당 1회, 전투당 1회, 일회성, 명시적 반복 불가는 사용 직후 잠긴다.
- 턴 종료는 플레이어가 명시적으로 선택한다. 남은 코인 때문에 자동으로 턴을 종료하지 않는다.
- 소비형 스킬은 지정 속성 코인을 선택해 즉시 소비하며 플립·면 효과가 발생하지 않는다.
- 준비형·보험형은 v4.5 고유 해결 규칙을 유지하되 예약 큐를 만들지 않는다.
- 일반 플립형 스킬은 코인별 앞면·뒷면을 선언하지 않는다. 각 스킬 데이터의 고정 `successFace`와 실제 RNG 결과가 일치한 개수로 성공 단계를 판정한다.
- 면 선언은 추후 명시적인 전용 태그를 가진 특수 스킬에만 허용하며 현재 일반 전투 UI에는 노출하지 않는다.

## 3. 코인과 상태이상

| 코인 | 앞면 | 뒷면 |
|---|---|---|
| 기본 | 피해 4 | 방어 4 |
| 화염 | 피해 3, 화상 2 | 방어 3, 대상이 화상이면 피해 2 |
| 마나 | 버림 더미에 임시 기본 코인 1 | 다음 턴 드로우 +1 |
| 전기 | 고정 피해 3 | 감전 2 |
| 냉기 | 동상 2 | 방어 3, 다음 턴 방어 2 |
| 혈액 | 체력 2 지불, 피해 7 | 출혈 2 |

- 혈액 앞면은 현재 체력이 2 이하이면 코인 효과만 발동하지 않는다. 플립 판정과 스킬 효과는 정상 해결한다.
- 무속성 예측형·보험형, 제한이 없는 준비형과 캐릭터 예측형에는 모든 속성 코인을 사용할 수 있다.
- 소비형은 스킬에 명시된 속성 코인만 사용한다.
- 화상은 대상 턴 시작에 수치만큼 체력을 잃고 1 감소한다.
- 동상은 대상이 가하는 공격 피해를 25% 낮추고 대상 턴 종료에 1 감소한다.
- 출혈은 대상이 받는 공격 피해를 50% 높이고 대상 턴 종료에 1 감소한다.
- 감전은 직접 피해를 받을 때 현재 수치만큼 방전 피해를 주고 1 감소한다.
- 예외와 피해 묶음 순서는 v4.5 11~12장의 정의를 따른다.

## 4. 화면 구조

### 4.1 공통 프레임

- PC와 모바일 가로를 동일한 16:9 게임 프레임으로 설계한다.
- 기준 좌표계는 1280×720이다.
- 데스크톱 초과 공간은 레터박스로 처리한다.
- 모바일 가로는 동적 뷰포트 높이에 맞추고 전체 페이지 세로 스크롤을 막는다.
- 세로 화면은 게임을 축소하지 않고 가로 회전 안내를 표시한다.
- 조작 대상은 최소 44×44 CSS px을 보장한다.

### 4.2 타이틀

순서는 `이어서 하기 → 새 시작 → 튜토리얼 → 설정`으로 고정한다. 저장이 없으면 이어서 하기는 비활성화한다.

### 4.3 캐릭터 선택

- 캐릭터를 가로로 배열하고 초상화를 카드의 주 시각 요소로 사용한다.
- 카드 아래에 시작 동전, 시작 스킬, 고유 패시브를 표시한다.
- 카드 선택과 실제 런 시작을 분리하고 별도 출발 버튼을 둔다.

### 4.4 원정 지도

- 기존 `RunGraph.layers`를 사용해 3막 노드 선택 지도를 표시한다.
- 현재 선택 가능한 노드만 활성화하고 지나온 길·현재 위치·미래 노드를 구분한다.
- 1차 구현은 다음 층의 모든 후보로 연결되는 기존 그래프 계약을 유지한다.

### 4.5 패시브 인벤토리

- 런 중 언제든 획득 패시브 목록을 열 수 있다.
- 이름, 등급, 속성, 발동 시점, 전체 효과를 `ContentDb.passives`에서 표시한다.
- 전투에서는 드로어, 그 외 화면에서는 모달로 제공한다.

### 4.6 전투 기록

- 누적 전투 기록 UI와 저장 상태를 제거한다.
- 방금 해결된 행동의 코인 면, 피해, 방어, 상태이상 결과 피드백은 유지한다.
- 코어 이벤트는 테스트·리플레이·분석을 위해 유지한다.

## 5. 튜토리얼

튜토리얼은 일반 런 저장·보상과 분리된 결정론적 연습 모드다.

1. 게임 목표와 노드 진행
2. 기본 코인과 속성 코인
3. 스킬 비용·유형·효과
4. 코인 선택, 스킬에 걸기, 즉시 사용과 RNG 플립
5. 뽑을 더미·손패·버림 더미
6. 공격 1회, 방어 1회, 턴 종료를 직접 수행하는 고정 전투

각 단계는 필요한 직접 행동을 완료해야 진행된다. 건너뛰기와 다시 보기 기능을 제공하며, 튜토리얼은 일반 런을 덮어쓰지 않는다.

## 6. 저장 및 호환성

- 전투 상태는 저장하지 않으므로 구형 예약 큐 마이그레이션은 만들지 않는다.
- 진행 중 전투 재개는 새 규칙으로 해당 전투를 다시 시작한다.
- 런 메타 저장은 기존 마이그레이션 경계를 유지한다.
- 콘텐츠·규칙 버전은 v4.5 적용 버전으로 올리고, 호환할 수 없는 구형 전투 UI 설정은 읽기만 하고 다시 저장하지 않는다.

## 7. 완료 조건

- DOM과 코어 명령에서 행동 확정·예약 큐·자동 배치·자동 실행이 사라진다.
- 같은 반복 스킬을 한 턴에 두 번 이상 즉시 사용할 수 있다.
- 모든 코인의 양면 효과와 네 상태이상이 v4.5 테스트를 통과한다.
- 모든 속성 코인이 무속성 예측형 스킬에서 사용 가능하다.
- 전투 기록 UI가 없고 직전 행동 피드백은 보인다.
- 타이틀 4개 메뉴, 가로 캐릭터 선택, 지도, 패시브 인벤토리, 튜토리얼이 키보드와 터치로 동작한다.
- 1600×900, 1366×768, 1280×720, 932×430, 844×390, 740×360에서 주요 조작이 프레임 안에 들어온다.
- 타입 검사, 린트, 단위·통합 테스트, 시뮬레이션, 프로덕션 빌드, 브라우저 플레이테스트가 모두 통과한다.

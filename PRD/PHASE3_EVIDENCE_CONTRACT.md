# Phase 3+ 자동 증거 계약 (2026-07-11 고정)

> 목적: 사람 게이트 차단이 waived된 상태에서(오너 오버라이드 — `PHASE1_HOLDS.md`) 무엇이
> "증명된 것"이고 무엇이 "미검증"인지 구분을 잃지 않기 위한 고정 계약. P3.0에서 확정하며,
> 이후 마일스톤은 이 계약을 완화할 수 없다 (강화만 허용).

## 1. 3축 결론 라벨 (모든 마일스톤 보고 의무)

| 라벨 | 의미 | 판정 수단 |
|------|------|-----------|
| `engineering-safe` | 규칙 정확성·결정론·불변식·크래시·접근성·배포가 증명됨 | 아래 §3 게이트 전부 green |
| `balance-provisional` | 수치는 동작하지만 방향 근거가 시뮬레이션 한정 | §4 밸런스 대체 게이트 + 리포트 |
| `experience-unverified` | 재미·손맛·긴장감·욕구 — 자동화로 판정 불가 | 영구 미검증. 사람 데이터만 닫을 수 있다 |

`passed`/`통과` 표기는 사람 게이트에 절대 사용하지 않는다.

## 2. 골든·스키마 버전 규칙

- **전투 골든**: 기존 골든(§5.8 전수표, seed42 전투 9턴)은 동결. 규칙 확장으로 기존 골든 값이
  바뀌어야 하면 **버전된 마이그레이션 + 결정 기록** 없이는 금지. 신규 규칙은 신규 골든을 추가한다.
- **런 골든**: `seed42 run` 골든(5전투 4/3/4/4/5턴·HP 41·최종 가방)은 콘텐츠 풀이 바뀌는 마일스톤마다
  **의도된 변경인지 diff 리뷰 후** 재고정하고, 재고정 사유를 PROGRESS에 남긴다. 무언의 골든 갱신 금지.
- **이벤트 스키마**: 확장은 **가산적(additive) 우선** — 기존 이벤트에 선택 필드 추가는 허용,
  기존 필드 의미 변경·제거는 trace 스키마 버전 승격(`m6-trace-v1` → `p3-trace-v2`) + sim fold·
  `sim:human` 리플레이 양쪽의 호환 테스트 필수.
- **저장 스키마**: RunSave 형태 변경은 처음부터(P3.2 캐릭터 선택 포함) `saveVersion` 승격 +
  마이그레이션 함수 + round-trip/구버전 로드 테스트를 동반한다. 마이그레이션 없는 덮어쓰기 금지.

## 3. 공통 엔진 게이트 (CI 차단 — 모든 마일스톤)

```
pnpm typecheck · lint · test · build · ci:sim · 브라우저 playtest (CI 필수 잡) · feedback-check
```

- terminal 100% / crash 0 / invariant 위반 0 / seed 골든 불변(또는 §2 절차로 재고정).
- 같은 contentVersion+시드+커맨드 → 상태·이벤트 바이트 동일 (A=A).
- 신규 콘텐츠마다: 도달 불가능 효과 0, 무한 루프 0, 음수 자원 0, 중복 UID 0, 코인 증발 0.
- core 순수성: Math.random/Date.now 0 (ESLint 규칙), 이벤트 로그가 UI·sim의 단일 진실.

## 4. 밸런스 대체 게이트 (report-only가 원칙, 수정 방향 특정 시에만 변경)

- 정책 풀: Random/Aggro/Turtle/GreedyEV + **빌드 정책**(화염/마나, P3.4부터 냉기/전기) — CRN 쌍 비교.
- 신규 스킬·코인·보상마다 **지배 옵션/사장 옵션 탐지** 리포트: "합리적 정책에서 항상 선택" 또는
  "절대 선택 안 됨"이 관측되면 수정 대상(방향이 특정된 경우다).
- 신규 2코인+/손패 생성/트리거/변환 콘텐츠마다 **§9.4 콤보 상한 표 재계산** 의무.
- 지표 분해: 턴 피해 p50/p95/max, 전투 턴, HP 손실, 상태 기여, 속성 활용, 소비/플립 기회, 보상 선택률
  — 정책×적×빌드. 스키마는 `p3-balance-report-v1`(m6 스키마의 가산 확장)로 명명.
- 수치 변경 기록 의무: 무엇을·어느 방향으로·어떤 결정론 증거가 특정했는지. 전부 `balance-provisional`.

## 5. UX 대체 게이트 (모든 신규 상호작용)

- 마우스·키보드·터치 대체 경로에서 완료와 취소가 전부 가능. Escape 취소 일관.
- 카드/코인/상태/패시브/적 의도는 hover·focus·touch 상세 중 1개 이상으로 설명 (키워드 사전 확장).
- 플립·상태·대상 변경·보상·상점 거래는 원인→변화→최종 결과를 이벤트 기반으로 표시 (결산 티켓 확장).
- 1024×720 / 1280×720 / 1920×1080 겹침·클리핑·콘솔 오류 0, visual-verdict ≥90 (시각 반복마다).
- reduced-motion 전면 지원, (P5부터) 음소거.
- 금지: 유령 DOM/테스트 심, 인덱스 결합 셀렉터, UI 게임 규칙 중복, 설명 없는 매직 넘버.

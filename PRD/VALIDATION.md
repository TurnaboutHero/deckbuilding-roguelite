# VALIDATION — 코인플립 로그라이크 (Phase 1: M0~M4)

## 필수 검증

골 완료로 마크하기 전 다음 명령을 반드시 실행한다.

```bash
pnpm typecheck            # 전 패키지 tsc --noEmit
pnpm lint                 # ESLint — core의 Math.random/Date.now 금지 규칙 포함
pnpm test                 # Vitest 전체: 골든 테스트 + 코인 총량 불변식 퍼즈 + 콘텐츠 lint(코스트 상한 A18)
pnpm build                # 전 패키지 빌드 (UI 프로덕션 빌드 포함)
pnpm sim play --seed 42 --auto   # 헤드리스 CLI 전투 1판 완주 (M1 이후)
```

## 마일스톤별 검증

각 마일스톤 종료 시 실행한다.

```bash
# M0: pnpm test --filter @game/core  → 동일 시드 1,000회 플립 재현 + 스트림 독립성 테스트 통과, CI green
# M1: pnpm sim play --seed 42 --auto → 전투 1판 완주 / pnpm sim fuzz --games 1000 → 불변식 위반 0
#     골든: 베기 10/6, 방어 5/8 (docs/implementation-plan.md §5.8)
# M2: pnpm build + 전투 화면 스크린샷 캡처 + 같은 시드 URL 재접속 시 동일 전투 재현
#     + GitHub Pages 배포 워크플로 통과 (URL 접속 가능)
# M3: 골든: 화염 코인 개별 판정(화염 2 HH → 화상 2), 화상 틱(방어 관통 후 -1), 불타는 일격 HH=14,
#     불씨 주머니(전투 시작 시 뽑을 더미 11개), 임시 코인 생성·순환 원장 불변식 (전투 간 정산은 M5)
# M4: MVP 6종 전수 골든(§5.8 표) + 폭주→점화 검술 콤보(취급 연료 인정) + 소모 영역 리셔플 격리
#     + 일회성(화염 폭주 전투당 1회 거부) + 턴당 3회 캡 거부 테스트
```

## 수동 확인 절차

1. `pnpm dev`로 브라우저 전투 실행 — 코인 드래그 장전 → 소켓 채움 → 사용 선언 → 플립 연출 → 피해 적용이 끊김 없이 이어지는지 확인 (M2 이후)
2. PRD/01_PRD.md §7의 UI 레이아웃(주머니 회색 원 / 노란 소켓 / 필드 스프라이트 + HP바)이 docs/PRD.md §15.1과 일치하는지 확인
3. 적 의도가 전투 1턴부터 표시되는지 확인 (D8)
4. 화염 폭주 사용 후 카드가 전투 종료까지 잠기는지(일회성 배지) 확인 (M4)
5. 주머니·버림·소모 인스펙터가 종류/매수·영구/임시·취급 속성을 정확히 표시하고, 버림 리셔플·소모 격리 설명이 실제 규칙과 일치하는지 확인

## 완료로 보지 않는 조건

- 필수 검증 중 하나라도 실패
- PLAN.md 밖의 scope로 변경됨
- 명시적 승인 없이 public API가 변경됨
- 수동 재현(CLI 시뮬 완주)이 여전히 실패함
- 산출물이 생성됐지만 검토되지 않음
- 검증을 통과시키기 위해 테스트가 삭제·skip됨
- 진단 없이 에러가 침묵 처리됨
- 스킬 효과가 효과 원자 데이터가 아닌 custom 코드 함수로 구현됨 (04_PROJECT_SPEC 위반)
- UI 코드에 게임 규칙 분기가 들어감 (04_PROJECT_SPEC 위반)
- 사람 게이트 항목을 자동 통과로 표기함 — 반드시 "사람 대기"로 남긴다

## 1. 다중 적 전투 — 타겟팅 UI 규칙

엔진은 이미 다중 적을 지원하며(A7·B17) `pre-human-backlog` 잡의 `multi-enemy.test.ts`가 엔진 동작을
증거화한다. UI 규칙(확정):

- **대상 지정**: `single-enemy` 스킬이 무장(코스트 완충)되고 생존 적이 2 이상이면, 사용 활성화가
  **대상 지정 모드**로 진입한다 — 생존 적 스프라이트/HP 플레이트가 하이라이트되고 클릭/좌우 방향키+Enter로
  대상 확정. 생존 적이 1이면 지금처럼 즉시 발동(모드 생략).
- **기본 대상**: 마지막으로 공격한 생존 적 → 없으면 왼쪽 첫 생존 적 (C4 순서와 일치). Escape 취소.
- **A8 폴백 시각화**: 무대상 스킬의 공격성 proc이 첫 생존 적에게 귀속될 때, 피해/화상 플로트는 해당
  적 위에 뜬다(이미 이벤트가 대상을 지정하므로 UI 추론 금지).
- **의도 표시**: 적마다 자기 플레이트에 의도 배지. 전원 공격 턴 금지 원칙(content-design-guide §5)은
  콘텐츠 규칙이며 UI는 검증하지 않는다.
- **legalCommands 선행 확장 (Phase 3 진입 필수 전제)**: 현재 `legalCommands`는 `useFlipSkill`을
  target 0(첫 생존 적 고정)으로만 열거한다 — `multi-enemy.test.ts`의 해당 어서션은 **현재 기준선의
  한계를 기록한 것이지 미래 계약이 아니다**. 타겟팅 UI를 구현하기 **전에** `legalCommands`가 모든
  생존 적 인덱스를 target 후보로 열거하도록 확장하고(코어 변경 + 퍼즈/골든 회귀), 그 뒤 UI는
  열거된 합법 target 집합만 하이라이트한다. 이 확장 자체도 Phase 3 hold 아래 있으며 지금 구현하지 않는다.

## 2. 지속형 상태이상 컨테이너 (C1 — 동상·감전)

- 스키마: `statuses`를 `{ kind: 'stack', stacks } | { kind: 'duration', turns }` 2형 컨테이너로 확장.
  화상은 stack형 유지(D2 문법 불변). 동상 = n턴 동안 **가하는** 공격 피해 25%↓(출발 측),
  감전 = n턴 동안 **받는** 피해 50%↑(수신 측).
- 적용 순서·반올림(확정, implementation-plan §12): 최종 피해 = floor(피해 × 동상 배율 × 감전 배율).
  화상 틱은 수정자 미적용 고정 피해 유지.
- 만료: 대상 턴 종료 시 잔여 턴 −1, 재부여는 턴 수 가산. 이벤트 어휘는 기존
  `statusApplied/statusTicked`에 duration 필드를 추가하는 방향(이벤트 스키마 버전 주의).
- 골든 선행: 동상 1턴 중 공격 12 → 9(floor), 감전 중 수신 10 → 15, 동상+감전 동시 12 → floor(12×0.75×1.5)=13.

## 3. 용광로 (`grantElement chooseBasicInHand`)

- 손패의 기본 코인 1개를 플레이어가 **직접 지정**해 이번 턴 화염 취급. UX는 이번 잡의 연료 지정
  모드와 동일 문법(하이라이트→선택→확정, Escape 취소)을 재사용한다 — 새 상호작용 어휘를 만들지 않는다.
- 대상 0개(기본 코인 없음)면 해당 라인만 불발, 스킬은 발동(§5.4와 동일).

## 4. 소비 연료 직접 선택 — E2E 전제

- 규칙과 구현·단위 테스트는 `pre-human-backlog` 잡에서 확정(count 1 = A15 자동 유지, count≥2 =
  지정 모드, 실행 coins[]는 손패 순서). **브라우저 E2E는 첫 cost≥2 소비 콘텐츠와 함께** 추가한다 —
  그 콘텐츠 PR의 게이트에 "연료 지정 모드 playtest 시나리오"를 필수로 포함할 것.

## 5. 콘텐츠 게이트 재계산 의무 (기존 규칙 재확인)

- 2코인+ 스킬·손패 생성 스킬 추가 시 §9.4 폭주 상한 표 재계산.
- 신규 콘텐츠는 A18 lint(cost≤4, 5=rare+일회성/궁극, 소비 1~3)를 통과해야 하며 수치는
  content-design-guide §1 기준표 대조 후 시뮬 재실행(ci:sim + sim:balance)으로 회귀 확인.

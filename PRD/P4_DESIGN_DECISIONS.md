# P4 설계 결정 기록 — 런 그래프·상점·이벤트·보스 (2026-07-12)

오너 지시(인간 승인 없이 결정 가능한 범위 전체 착수)에 따라, SSoT가 확정한 스펙은 그대로 잇고
미확정 지점은 **게임공학 기준의 provisional 결정**으로 기록한다. 모든 신규 수치는
`balance-provisional`이며, 결정론 시뮬 증거가 방향을 특정할 때만 변경한다.

SSoT 출처: `docs/implementation-plan.md` §12 부록(예약 지점)·B1, `docs/content-design-guide.md`
§5(몬스터 기준표·예시 5종)·§6(상점·이벤트), PRD §10(영구 변환은 이벤트 전용), P3.4 §825
가중 함수(`weightedCoinOptions` — 상점 공유 의무).

## D1. 런 그래프 — 레이어 노드 리스트 (결정론, 분기 선택형)

- **형태**: 선형 `combatIndex`를 **레이어드 노드 리스트**로 일반화한다. 레이어 10개,
  각 레이어는 노드 1~2개. 노드 2개 레이어에서 플레이어가 다음 노드를 선택한다(분기 = 선택,
  스킵 없음 — 모든 레이어를 정확히 1회 통과).
- **생성**: 런 시드의 전용 `graph` 스트림으로 분기 레이어의 노드 종류·이벤트/조우 배정을
  롤한다. 같은 시드 → 같은 그래프 (결정론, CRN 스트림 격리 유지).
- **레이어 계획 (provisional)**:
  1. 전투(단일 일반) 2. 전투(2마리) 3. **상점|이벤트** 4. 전투(2마리) 5. 엘리트
  6. **상점|이벤트** 7. 전투(3마리) 8. **이벤트|전투(2마리)** 9. 상점(고정 — 보스 준비)
  10. **보스**
- **근거**: B1의 "HP 이월·중간 회복 없음" 긴장을 유지하면서(회복 노드 없음), 상점 제거/구매가
  실질 회복 대체 자원이 된다. 전투 수 5→6~7로 증가하나 골드 경제가 상쇄 — Monte Carlo로 검증(D7).
- **기각**: 자유 DAG(같은 레이어 노드 수 3+·스킵 경로)는 legalCommands 열거·시뮬 정책 축이
  기하급수로 늘어 P4 범위를 넘는다. 노드 리스트 일반화가 SSoT 문언("노드 리스트로 일반화")과 일치.

## D2. 조우 테이블·신규 몬스터·보스

- **정식 다중 적**: content-design-guide §5.2 예시 5종(고블린 32·도적 26·구울 38·마도사 22·
  슬라임 16)을 정식 콘텐츠로 승격. §5.2 개체 수치는 무변경, 조합만 §5.1 대역으로 선별:
  - 2마리 풀(대역 65~85): **고블린+구울(70)** 정합. **도적+고블린(58)**은 하한 7 미달이나
    도적 감전(받는 피해 +50%)이 실효 압박을 보상한다는 판단으로 **예외 승인** — 두 조합
    모두 provisional, Monte Carlo 승률 이상치면 재조합. (그 외 2마리 조합은 전부 대역 밖.)
  - 3마리 풀(대역 75~95): **구울+고블린+슬라임(86)** — §5.1 예시 그대로. 1개로 시작.
  - 엘리트: 기존 raider-plus·gatekeeper-plus 재사용 (검증된 수치).
- **보스 1종 (신규)**: "**잿불 마도왕**" — HP 150, 순서형 루프: ① **마력 증폭** ↑버프(다음
  공격 +8 — 사용자 확정에 따라 충전/특수 행동 타입은 없다, 명시적 buff 행동으로 구현하고
  UI 의도도 버프로 표시) ② 파멸의 화염구 ⚔20(증폭 시 28 — §5.1 강공 18~24 대역은 기본 20으로
  충족, 28은 패턴 순서가 예고하는 스파이크) ③ 잿불 보호막 🛡12 + 위축 1. "강공은 반드시
  예고"는 순서형 루프 자체가 보장(버프 턴 다음이 항상 강공). 수치 전부 balance-provisional.
- **상태 어휘**: 원 초안 매핑 유지 — 취약→감전, 약화→동상 (§5.2 각주).
- **행동 타입 규약 (사용자 확정)**: 몬스터 행동 타입에 충전/특수는 **없다**. §5.2 마도사의
  "주문 충전 ⚡"을 포함해 모든 충전류 행동은 명시적 **buff**(다음 공격 +N)로 구현하고 UI
  의도도 버프로 표시한다. 강공 예고는 순서형 패턴 순서가 담당한다.

## D3. 골드 (결정론 고정 수급)

- 일반 전투 **35**, 엘리트 **70**, 보스 **100** (§6 대역 30~45/60~80 안의 고정값 — 수급
  RNG를 없애 시뮬 분산을 줄이고 CRN 스트림 추가를 회피). 시작 골드 0. 이벤트 골드 증감은
  이벤트 데이터가 명시. RunState.gold 필드는 이미 예약됨(구현만).

## D4. 상점 — §6 3행동 그대로

- 동전 제거: 75 → 구매마다 +25 (런 전역 누진 카운터 `shopRemovals`).
- 스킬 진열 5: 일반 3 + 고급 1 + 무작위 1, `rewardEligibleSkillIds` 공유(전용 풀 경계 준수),
  가격 일반 50/고급 80/희귀 120.
- 동전 진열 3: **`weightedCoinOptions` 공유 의무**(§825 — P3.4에서 도입한 정본 그대로,
  상점 전용 시드 스트림). 가격 기본 25/대표 속성 50/타 속성 70.
- 행동은 골드가 허용하는 한 반복 가능, `leaveShop` 명시 커맨드로 종료. 재진열 없음(1방문 1롤).

## D5. 이벤트 — 위험요소 4유형 각 1종 (공짜 금지)

배정은 그래프 생성 시 롤, 해결은 선택에 대해 결정론(해결 시점 주사위 없음). 전부 거절
("지나친다") 가능. provisional 4종:
1. **전투 위험**: 엘리트 전투 수락 → 승리 시 희귀 스킬 1 선택 (거절 시 무).
2. **체력 감소**: HP 5 잃고 대표 속성 코인 1개 획득.
3. **재화 소모 = 영구 변환 (이벤트 전용, PRD §10)**: 100골드로 기본 코인 1개 → 대표 속성
   코인 영구 교체.
4. **자원 소모**: 기본 코인 1개 제거 → 대표 속성 코인 1개 획득 (주머니 압축 트레이드).

## D6. 저장 v3 + 마이그레이션

- RUN_SAVE_VERSION 2→**3**: `graph`(노드 리스트·현재 위치)·`gold` 활성화·`shopRemovals`·
  노드별 pending 상태. v2(선형 5전투) 저장은 **레거시 5전투 그래프**(전투×5, 분기 없음)로
  감싸 마이그레이션 — 진행 중 런 보존, 의미 변화 없음. v1→v2→v3 체인, 미지 버전 거부.
  라운드트립·역호환 테스트 의무(증거 계약 §2).
- CONTENT_VERSION은 몬스터/보스 콘텐츠가 실리는 커밋에서 **0.9.0-p4**로 승격, seed42 골든
  재고정을 버전 승격에 결속.

## D7. 시뮬 — 경로 정책 축 + 경제 Monte Carlo

- 노드 선택 정책 2종(상충): **fight-first**(분기에서 전투/이벤트-전투 우선) vs
  **economy-first**(상점/이벤트 우선). 기존 전투 정책 4종·빌드 정책과 직교 조합.
- 상점 구매 정책(provisional): 제거 1회 우선(기본 코인) → 대표 속성 코인 → 스킬(미보유 중
  최고 등급) — 골드 잔량 규칙. 이벤트는 EV 계산 없이 정책별 고정 선택(fight-first 수락 /
  economy-first는 재화 보존 선택).
- **경제 Monte Carlo**(report-only): 캐릭터 4 × 경로 정책 2 × 500런 — 파산률(구매 의도 시
  골드 부족), 구매 횟수 분포, 제거 비용 누진 도달 단계, 속성 노출률(상점 진열 포함),
  이벤트 선택률·기대값, 보스 도달률·승률. `p4-economy-report-v1` 스키마.
- 기존 게이트 불변: terminal 100%·crash 0·invariant 0·A=A·CRN 스트림 격리(그래프 스트림 추가).

## D8. UI 표면

- **지도 화면**: 전투 사이 노드 리스트/분기 선택(키보드·마우스, 현재 위치·종류 아이콘·툴팁).
- **상점 화면**: 3행동 카드형 진열, 가격·잔여 골드, 구매 확정/불가 사유 피드백(기존 거부 사유
  패턴 재사용), 나가기.
- **이벤트 화면**: 제시문·위험/보상 명시·수락/거절.
- HUD에 골드 표시(주머니 옆). 테스트 전용 진입로: `?node=shop` 등 (정식 콘텐츠 무접촉 원칙 유지).
- 신규 스프라이트: 도적·구울·마도사·슬라임·보스 5종 — sprite-gen component-row + 킷 선검증
  (자산 라운드에서 일괄). 그 전 라운드는 기존 스프라이트 조합(고블린 기존 자산)으로 진행 가능한
  범위만 배포.

## D9. P4.2b/P4.3 실행 편차 (2026-07-12, 통합 감사 반영)

- **진행 SSoT = 완료 전투 수**: bag/skill 보상 상한은 레이어 인덱스가 아니라 "통과한
  combat/elite/boss 노드 수"로 계산한다(코어 `completedCombatCount` 단일 정본을 저장
  검증기가 공유). 비전투 노드(상점/이벤트)는 보상 상한을 절대 늘리지 않는다. 상점
  구매·제거는 별도 카운터(`shopPurchasedCoins`·`shopPurchasedSkills`·`shopRemovals`)로
  상한/하한에 반영한다.
- **save v4**: 상점 상태(pendingShop)·구매 카운터 추가 — v3(P4.1)와 형태가 달라
  RUN_SAVE_VERSION 3→4, v1→v2→v3→v4 명시 체인. 저장 검증의 그래프 길이는 5 고정이
  아니라 저장된 그래프 자체가 기준(레거시 마이그레이션 저장만 5).
- **이벤트 노드 유예**: D1의 이벤트 레이어(3·6·8)는 이벤트 시스템(P4.4) 전까지
  상점|전투 분기·전투로 대체한 10레이어 계획을 활성화한다. P4.4에서 이벤트 옵션 복원
  시 그래프 계획 개정 + 골든 재고정을 다시 결속한다. 활성 레이어(provisional):
  1 전투(단일: 약탈자|주술사|수문장) 2 전투 2마리 3 **상점|전투 2마리** 4 전투 2마리
  5 엘리트(약탈자+|수문장+) 6 **상점|전투 3마리** 7 전투 3마리 8 전투 2마리 9 상점(고정)
  10 보스(잿불 마도왕).
- **그래프 전용 RNG**: `derive(runSeed, 'graph')` 스트림만 소비 — 전투/보상/상점 스트림
  격리 유지(CRN 계약). 10레이어 활성화로 seed42 런 골든은 의도 변경 — 재고정은 이
  라운드 커밋에 결속(0.9.0-p4 시대).
- **이동 규칙**: 선택은 항상 "다음 레이어의 노드"만, 스킵 없음. 분기 레이어(노드 2)는
  `choose-node` 페이즈에서 명시 커맨드, 단일 노드는 자동 진입. 불법 payload·범위 밖
  선택은 코어·저장 양쪽에서 거부.

## D10. P4.4 이벤트 시스템 (2026-07-12, provisional)

- **원칙(§6 그대로)**: 공짜 금지 — 모든 이벤트는 위험요소 4유형 중 하나. 전부 "지나친다"
  선택 가능. 해결은 선택에 대해 결정론(해결 시점 주사위 없음).
- **이벤트 4종 (각 유형 1종, 수치 전부 balance-provisional)**:
  1. `ambush-bounty` **매복 현상금** (전투 위험): 수락 → 엘리트 전투(raider-plus|gatekeeper-plus,
     이벤트 스트림 롤). 승리 시 보상의 스킬 단계가 **희귀 전용 2진열**로 대체(미보유 희귀 풀,
     부족 시 fallback 규칙 재사용). 패배는 일반 패배.
  2. `blood-offering` **피의 제물** (체력 감소): HP 5 지불(생존 하한 — currentHp ≤ 5면 수락
     불가) → 대표 속성 코인 1 영구 획득.
  3. `transmute-altar` **변환 제단** (재화 소모 = **영구 변환, 이벤트 전용 — PRD §10**):
     100골드 지불 → 기본 코인 1개(직접 선택)를 대표 속성 코인으로 영구 교체. 기본 코인
     0개면 수락 불가.
  4. `coin-sacrifice` **동전 희생** (자원 소모): 기본 코인 1개(직접 선택) 제거 → 대표 속성
     코인 1 획득 (주머니 압축 트레이드). bag 최소 1 가드.
- **배정·결정론**: 이벤트 노드 진입 시 `derive(runSeed, 'event-<layerIndex>')` 전용 스트림으로
  이벤트 1종 롤 — **기존 graph/combat/reward/shop 스트림 불변** (스트림 격리 계약 유지).
- **그래프 계획 v2 (D9 유예 해제)**: 레이어 3·6 = 상점|**이벤트**, 8 = **이벤트**|전투 2마리
  — D1 원안 복원. generateRunGraph 의도 변경 → seed42 골든 재고정을 이 라운드 커밋에 결속.
- **상태 기계**: RunPhase + `'event'`. 진입 시 `pendingEvent { eventId }`. 커맨드:
  `acceptEvent(run, db, choice?)`(제단·희생은 bag 인덱스 선택 필요) / `declineEvent(run, db)`.
  매복 수락 → 즉시 엘리트 전투 시작 상태로 전이(`pendingEventCombat` 표시), 승리 정산 시
  희귀 스킬 진열·골드는 엘리트 70.
- **진행 SSoT 확장**: 이벤트 전투는 combat 노드가 아니므로 `eventCombats: number` 카운터를
  save에 추가해 보상 상한 산정에 합산(비전투 이벤트는 상한 불증가 — 코인 증감은
  `eventCoinGains`/`eventCoinLosses` 카운터로 bag 경계에 반영).
- **save v5**: pendingEvent·pendingEventCombat·이벤트 카운터 3종. v1→…→v4→v5 체인,
  이벤트 payload 계약(pendingEvent는 event 페이즈에만, eventId는 콘텐츠 사전 대조).
- **telemetry**: path 사실에 `{ layer, type: 'event', action: 'accept'|'decline', choice? }`
